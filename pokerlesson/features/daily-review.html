DailyReview（今日の1問）

1日1問・1分で完結する習慣化導線。START LESSON の手前に置く「静かなゲートウェイ」。

機能ID: FEAT-01 Phase 1

概要

DailyReview は、5〜15分かかる本格セッション（START LESSON）の入口として、 「1分で完結する1問だけ」をホーム画面に常設する機能。 Duolingo 的なストリーク・XPボーナスによる強制感を意図的に排除し、 PokerLesson らしい「静かな余韻」のトーンを維持したまま日次接触を促す。

  • ユーザー価値: 忙しい日でも「1問だけ」で学習習慣を維持できる
  • ビジネス価値: DAU 向上・セッション開始率向上（30日KPIで測定）
  • フリー枠との関係: DailyReview はフリー枠（30問）の消費対象外。習慣化を阻害しない設計

関連画面

  • ホーム画面 (HOME-01) — TODAY'S NOTE カードを配置
  • CPU対戦セットアップ (SETUP-01)

問題選定ロジック

以下の優先順位でその日の1問を決定する。

優先順位条件source値
1. 前回の続き 直近24時間以内の不正解問題（本日すでに正解済みは除外） continuation
2. 弱点補強 PlayerTendency.recommendedQuestionIDs から1問 weakness
3. ランダム 未回答優先、なければ全体からランダム random

処理フロー

sequenceDiagram participant U as User participant HomeView participant DailyReview as DailyReview(Model) participant QuizVM as QuizViewModel participant QuizView HomeView->>DailyReview: isCompletedToday() DailyReview-->>HomeView: false → カード表示(アクティブ) U->>HomeView: TODAY'S NOTE カードをタップ HomeView->>DailyReview: select() DailyReview-->>HomeView: DailyReviewSelection(question, source) HomeView->>QuizVM: isDailyReview = true, dailyReviewSource = source HomeView->>QuizView: navigate QuizView->>U: 1問出題 U->>QuizView: 回答 QuizView->>QuizVM: 正誤判定 QuizVM->>DailyReview: 完了タイムスタンプ保存(@AppStorage) QuizVM-->>QuizView: isDailyReview=true → DailyReview完了画面へ QuizView->>U: 静かな完了画面（比較1行 + ホームに戻るボタンのみ）

ホーム画面上のカード仕様

状態表示操作
アクティブ 「TODAY'S NOTE」ラベル + 「1問だけ」案内テキスト。START LESSON より控えめなビジュアル（サイズ・コントラスト共に下げる） タップで DailyReview クイズへ遷移
完了済み グレーアウト + チェックマーク + 「今日のノートは終わっています。」テキスト タップ不可（または軽いフィードバックのみ）

完了画面の設計思想

完了画面は意図的に「何も煽らない」設計とする。

  • メッセージ: 「お疲れさまでした。」のみ
  • 比較1行: 直近のパフォーマンスとの変化だけを表現（上達・成長などの断定語は避ける）
    • 「昨日より安定しています」
    • 「昨日と同じペースです」
    • 「今日は少しブレています」
  • 表示しないもの: 広告・XPボーナス・継続催促・「次の問題へ」誘導・ストリーク表示
  • ボタン: 「ホームに戻る」のみ

完了状態の管理

// Models/DailyReview.swift
struct DailyReviewSelection {
    let question: Question
    let source: DailyReviewSource
}

enum DailyReviewSource: String {
    case continuation  // 前回の続き
    case weakness      // 弱点補強
    case random        // ランダム
}

enum DailyReview {
    /// その日の1問を選定して返す
    static func select(from questions: [Question], tendency: PlayerTendency) -> DailyReviewSelection

    /// 直近のパフォーマンスと比較して1行コメントを返す
    static func quietComparison(history: [QuizResult]) -> String

    /// 本日完了済みかどうか (@AppStorage("daily_review_last_completed_at") から判定)
    static func isCompletedToday() -> Bool
}

// 完了タイムスタンプ保存キー
// @AppStorage("daily_review_last_completed_at") var lastCompletedAt: Double = 0
// 判定: Calendar.current.isDateInToday(Date(timeIntervalSince1970: lastCompletedAt))

ビジネスルール

  • DailyReview の1問はフリー枠（30問）の消費にカウントしない
  • 1日1問のみ（完了後は当日中カードを非アクティブ化）。翌日0時にリセット（Calendar.isDateInToday で判定）
  • 完了画面から START LESSON への誘導リンクは設置しない（10分以内の自然な遷移のみKPI計測）
  • 問題選定の優先順位はサーバー側で変更できるようにしない（Phase 1 はクライアントサイドのみ）

エラー処理

発生条件対応
問題データが0件カードを非表示。エラーUIは出さない（START LESSON を前面に出す）
@AppStorage 読み込み失敗完了済みとして扱わない（アクティブ表示を維持）。再度完了できても問題なし
PlayerTendency が未計算source を random にフォールバック

実装メモ

  • Models/DailyReview.swift（新規）: DailyReviewSelection 構造体、DailyReviewSource enum、DailyReview enum を定義
  • Views/HomeView.swift: dailyReviewCard / dailyReviewCardActive / dailyReviewCardCompleted サブビューを追加。START LESSON カード直下に配置
  • Views/QuizView.swift: QuizResultView に isDailyReview 分岐を追加。trueの場合は完了画面へ
  • ViewModels/QuizViewModel.swift: isDailyReview: Bool / dailyReviewSource: DailyReviewSource? プロパティを追加
  • TODAY'S NOTE カードは START LESSON より視覚的に控えめにする（フォントサイズ・背景色・パディングで差をつける）
  • 比較1行は直近7日間の正答率を基準に3パターンから選択。比較データが存在しない場合は比較行を非表示

KPI 観測ポイント（リリース後30日）

指標算出方法目標値判断
DailyReview 開始率 daily_review_started / DAU 30% 以上 未達なら導線・カードビジュアルを見直す
DailyReview 完了率 daily_review_completed / daily_review_started 70% 以上 未達なら問題難易度・完了画面UXを見直す
完了後 LESSON 進行率 lesson_started_after_daily / daily_review_completed 5% 以上 5% 以上あれば「ゲートウェイ」として機能
source 別分布 daily_review_started の source パラメータ 参考値 continuation が多い = 不正解が多い傾向の把握に使う
