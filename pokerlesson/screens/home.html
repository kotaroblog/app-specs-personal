ホーム画面

アプリ起動後の最初の画面。学習セッション・CPU対戦・DailyReview への導線を提供する。

画面ID: HOME-01 Phase 1

目的・役割

ユーザーが「今日何をするか」を素早く選択できる起点画面。 Phase 1 では DailyReview（TODAY'S NOTE）カードを追加し、 本格セッション前の1分完結の習慣化導線を設ける。 画面構成はシンプルさを最優先とし、余計な情報・通知・ストリーク表示は置かない。

UI 構成

要素種類説明
START LESSON カード Button / NavigationLink 本格セッション（5〜15分）への主導線。最も目立つビジュアル
TODAY'S NOTE カード（DailyReview） Button / NavigationLink START LESSON 直下に配置。1日1問の習慣化導線。START LESSON より視覚的に控えめ（フォントサイズ・背景色・パディングを下げる）
CPU対戦カード Button / NavigationLink CPU対戦セットアップへの導線
設定ボタン Toolbar Button 右上。設定画面へ遷移

状態

状態TODAY'S NOTE カードの表示
アクティブ（未完了） 「TODAY'S NOTE」ラベル + 「1問だけ」案内テキスト。タップ可能
完了済み（当日中） グレーアウト + チェックマーク + 「今日のノートは終わっています。」テキスト。タップ不可（または軽いフィードバックのみ）
問題データなし TODAY'S NOTE カードを非表示にする。エラーUIは出さない

DailyReview カードの状態遷移

条件カード状態判定ロジック
当日未完了 かつ 問題あり アクティブ DailyReview.isCompletedToday() == false かつ 問題データが存在
当日完了済み 完了済み（グレーアウト） DailyReview.isCompletedToday() == true。@AppStorage("daily_review_last_completed_at") を Calendar.isDateInToday で判定
翌日 アクティブ（リセット） 日付が変わると isCompletedToday() が false を返す

遷移

操作遷移先条件
START LESSON カードをタップ レッスン選択画面 常時
TODAY'S NOTE カードをタップ QuizView（isDailyReview=true） アクティブ状態のみ
CPU対戦カードをタップ CPUMatchSetupView 常時
設定ボタンをタップ 設定画面 常時

使用するデータモデル

// DailyReview 状態判定
DailyReview.isCompletedToday()  // @AppStorage から読み取り

// DailyReview 問題選定（カードタップ時）
DailyReview.select(from: questions, tendency: playerTendency)
// → DailyReviewSelection(question: Question, source: DailyReviewSource)

// KPI ロギング（カード表示時）
EventLogger.shared.log(.dailyReviewCardShown(source: selection.source))

実装メモ

  • TODAY'S NOTE カードは START LESSON カード直下に配置する（CardGrid や VStack で順序固定）
  • カードのビジュアル差分: TODAY'S NOTE は背景色を .systemGray6、パディングを 12pt（START LESSON は 20pt）など控えめに設定する
  • dailyReviewCard は @ViewBuilder のサブビューとして切り出し、HomeView の本体を 200行以下に保つ
  • ホーム画面の onAppear で DailyReview.select() を呼び出し、選定結果を @State に保持する
  • lesson_started_after_daily の判定: START LESSON タップ時に @AppStorage("daily_review_last_completed_at") との差分を確認し、600秒以内なら EventLogger に送信
