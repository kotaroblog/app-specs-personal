CPU対戦セットアップ

CPU対戦のテーブルサイズ・スタック等を設定してマッチを開始する画面。

画面ID: SETUP-01 v1.1 リデザイン反映

目的・役割

CPU対戦を開始する前に、難易度・テーブルサイズ・初期スタック・テンポ・学習サポート（GTOヒント）をユーザーが選択する。 Phase 1 で 6max をデフォルト化、v1.1 でヘッダーの装飾（"CPU MATCH" セリフ大文字 + スート飾り + 真鍮ライン）を全廃し、 各項目を goldCard() セクションカードに統一した。

UI 構成（v1.1）

すべて ScrollView 内の VStack(spacing: 16)。各セクションは goldCard()（1ptボーダー + surface 塗り）。 navigationTitle("対戦設定") をインライン表示。

セクション要素説明
CPU難易度 縦並びの選択行（3行） easy / normal / hard。各行に色付きインジケータ + 難易度名 + 説明 + 学習目標。選択行はアクセント色ボーダー + checkmark
テーブルサイズ 2chip（横並び） 6max / 9max。6max には「推奨」バッジ（accent サブカラー）+ サブタイトル「学習向け・現代主流」
初期スタック 4chip（横並び） 25bb / 50bb / 100bb / 200bb。monospaced 数値
テンポ 3chip（横並び） 遅め / 標準 / 速め。各chipに displayName + shortDescription を縦に表示
学習サポート Toggle行 GTO推奨ヒントON/OFF。eye.fill / eye.slash アイコン + 説明 + SwiftUI Toggle（accent tint）
対戦開始ボタン Primary Button（CTA） 「対戦開始」 + 右に arrow.forward。アクセント色塗り、白文字

v1.1 リデザインで削除・刷新した要素

  • ヘッダー装飾：上部の miniDivider + ♠♥♦♣ スートアイコン + セリフ大文字 "CPU MATCH"（tracking 2.5）+ サブテキスト「9人MTTシミュレーション」を全削除
  • CTA：「DEAL THE CARDS」セリフ大文字 + サブテキスト「対戦開始」 → 通常CTA「対戦開始」
  • 選択中chipの AppTheme.Gradients.goldShine グラデ塗り → accentSubtle 単色 + accent ボーダー
  • 難易度ラベル levelLabel のモノスペース tracking 装飾を縮小
  • 2max オプションは元々非対応（実装上は 6max / 9max の2択）。仕様書側の記述ずれを v1.1 で訂正

6max デフォルト化（Phase 1）

項目値
テーブルサイズ初期値6（setupTableSize = 6）
6max チップの表示「推奨」バッジ + サブタイトル「学習向け・現代主流」
9max チップの表示サブタイトル「MTT・トーナメント向け」

変更の意図: 6人テーブルは9人より手数が増え学習密度が上がる。 また現代のキャッシュゲームの主流であり、席が空きすぎないため対戦UI的にも自然。

状態

状態表示内容
初期表示 テーブルサイズ = 6max 選択済み（デフォルト）、推奨バッジ表示
6max 選択時 チップがハイライト。推奨バッジが目立つ形で表示
9max / 2max 選択時 選択チップがハイライト。推奨バッジは 6max チップに引き続き表示

遷移

操作遷移先
「対戦開始」ボタンをタップ CPU対戦画面（setupTableSize を引き継ぐ）
戻る ホーム画面

使用するデータモデル

// Views/CPUMatch/CPUMatchView.swift
// テーブルサイズの初期値
@State private var setupTableSize: Int = 6  // Phase 1 で 9 → 6 に変更

// KPI ロギング（画面表示時）
EventLogger.shared.log(.cpuSetupShown)

// KPI ロギング（対戦開始時）
EventLogger.shared.log(.cpuSetupStartedMatch)
EventLogger.shared.log(.cpuMatchStarted(tableSize: setupTableSize))

実装メモ

  • Views/CPUMatch/CPUMatchView.swift: setupTableSize の初期値を 6 に変更
  • Views/CPUMatch/CPUMatchSetupView.swift: tableSizeChip コンポーネントに推奨バッジを追加。バッジは .overlay または ZStack で実装。サブタイトルは Text("学習向け・現代主流")
  • 推奨バッジのスタイルは既存の「おすすめ」等バッジがあればそれを流用。なければ Capsule + アクセントカラーで実装
  • 画面表示時（onAppear）に EventLogger.shared.log(.cpuSetupShown) を呼ぶ
  • 「対戦開始」ボタンの action 内で .cpuSetupStartedMatch と .cpuMatchStarted(tableSize:) を順に送信
