ルーティン機能

曜日固定のトレーニングテンプレートを「今日のルーティン」として自動提案する機能（v1.1.0）。v1.1.1 でピラミッドセット対応（TemplateSetEntry）を追加。v1.1.2 でルーティン画面から直接新規作成できる CreateTemplateView を追加。

v1.2 機能ID: WD-FEAT-ROUTINE

概要

「ルーティン機能」は WorkoutTemplate（旧称: テンプレート）に曜日割り当て（assignedWeekdays）を付与し、その日の曜日に対応したルーティンをトレーニング記録画面・カレンダー画面に自動提案する機能。ユーザーはワークアウトを開始する際に何をすべきかを迷わず、1タップでルーティンを適用できる。

目的・ユーザー価値

  • 「今日は何をやるべきか」迷いをなくし、記録開始までの摩擦を最小化する
  • スプリット（胸の日/背中の日）を維持しやすくする
  • 週間トレーニング計画をカレンダーで視覚的に確認できる（予定ドット）

想定ユーザー

  • 週3〜4回以上トレーニングし、部位別スプリットを組んでいるユーザー
  • 特定の曜日に固定メニューを行うルーティンワーカー

データモデル

@Model
final class WorkoutTemplate {
    var id: UUID = UUID()
    var name: String = ""
    var createdAt: Date = Date()

    /// 割り当て曜日（v1.1.0）: 0=日曜, 1=月曜, ..., 6=土曜
    /// 空配列の場合は曜日固定なし（テンプレートとして手動適用のみ）
    var assignedWeekdays: [Int] = []

    @Relationship(deleteRule: .cascade, inverse: \TemplateEntry.template)
    var entries: [TemplateEntry] = []

    var sortedEntries: [TemplateEntry] { entries.sorted { $0.order < $1.order } }

    /// 指定日の曜日がこのルーティンの担当曜日かどうかを返す
    func appliesTo(date: Date) -> Bool {
        guard !assignedWeekdays.isEmpty else { return false }
        let weekday = Calendar.current.component(.weekday, from: date) - 1
        return assignedWeekdays.contains(weekday)
    }
}

/// v1.1.1 新規: セットごとの値を持つ（ピラミッドセット等）
@Model
final class TemplateSetEntry {
    var id: UUID = UUID()
    var weight: Double = 0    // ウエイト種目: kg
    var reps: Int = 0         // ウエイト種目: 回
    var duration: Double = 0  // 有酸素種目: 分
    var distance: Double = 0  // 有酸素種目: km
    var order: Int = 0
    var entry: TemplateEntry?
}
マイグレーション
assignedWeekdays: [Int] = [] はデフォルト空配列のため、SwiftData lightweight migration で既存データへの影響なし。

v1.1.1 追記: TemplateEntry.setEntries: [TemplateSetEntry] = [] もデフォルト空配列で lightweight migration に対応。既存データは TemplateMigrationService が起動時に自動展開する（詳細は後述）。

TemplateSetEntry とピラミッドセット（v1.1.1）

v1.1.0 まで TemplateEntry は「全セット同じ値 × N セット（weight × reps × sets）」しか表現できなかった。v1.1.1 で TemplateSetEntry を導入し、セットごとに重量・回数を独立して設定できるようになった。

シナリオ例v1.1.0 までv1.1.1 以降
均等セット60kg × 10 rep × 3セットweight=60, reps=10, sets=3TemplateSetEntry を 3 件（同値）
ピラミッドセット20kg×10 / 40kg×8 / 60kg×6表現不可TemplateSetEntry を 3 件（異なる値）
有酸素 1 セットトレッドミル 30 分duration=30, sets=1TemplateSetEntry 1 件（duration=30）

関連画面

  • ルーティン選択（TemplateListView） — ルーティン一覧・適用・曜日バッジ表示
  • トレーニング記録画面（WorkoutSessionView） — エンプティCTA「今日のルーティン」プライマリ提案
  • カレンダー画面（CalendarView） — 未来日の予定ドット・BottomSheet ルーティンセクション

SaveTemplateView / EditTemplateView でも曜日選択 UI（WeekdayPicker）を表示する（別途仕様書なし、TemplateListView 仕様書参照）。

TemplateMigrationService（v1.1.1 新規）

/// レガシー TemplateEntry の weight × reps × sets を setEntries 形式に展開する（v1.1.1）
/// 冪等: setEntries が空でない entry はスキップ
@MainActor
enum TemplateMigrationService {

    /// 指定 TemplateEntry が未移行（setEntries が空）ならレガシー値を展開する
    static func migrateIfNeeded(_ entry: TemplateEntry, modelContext: ModelContext)

    /// 全 TemplateEntry の未移行分を一括展開
    /// 呼び出し元: WorkoutDiaryApp.onAppear, EditTemplateView.onAppear
    static func migrateAllIfNeeded(modelContext: ModelContext)
}
移行後の挙動
ウエイト種目: sets 件の TemplateSetEntry を生成（全て同じ weight/reps）
有酸素種目: 1 件の TemplateSetEntry を生成（duration/distance を保持）
sets=0 の場合は min(sets, 1)=1 件として展開する

EditTemplateView（v1.1.1 全面リライト）

v1.1.0 までは全種目のセット数・重量・回数を1行で編集していたが、v1.1.1 でピラミッドセット対応のため全面リライト。

要素説明
Section per 種目各種目を独立 Section で表示。Section ヘッダーは部位カラー左ボーダー + 種目名 + 種目削除ボタン

AddTemplateEntrySheet（v1.1.1 新規）

EditTemplateView から「種目を追加」でシート表示される。既存の Exercise 一覧を表示し、種目を選択すると TemplateEntry + デフォルト TemplateSetEntry を作成して追加する。

要素説明
種目一覧部位カラー左ボーダー Section で Exercise をグループ表示
検索バー種目名で絞り込み
デフォルト値ウエイト種目: 3 セット分の TemplateSetEntry（weight=0, reps=0）。有酸素種目: 1 セット（duration=0）

SaveTemplateView 保存ロジック改善（v1.1.1）

ワークアウトから「ルーティン保存」時、各 WorkoutSet を個別の TemplateSetEntry として保存するよう改善。均等セットでも変則セットでも実データをそのまま反映できる。

RoutineSuggestionService（v1.1.1 改修）

/// 指定日のルーティン提案・適用を行うサービス（v1.1.0 / v1.1.1 改修）
@MainActor
enum RoutineSuggestionService {

    /// 指定日に割り当てられたルーティンを全て返す
    static func routines(for date: Date, in templates: [WorkoutTemplate]) -> [WorkoutTemplate]

    /// 指定日に該当する先頭1件のルーティンを返す（なければ nil）
    static func todaysRoutine(for date: Date, in templates: [WorkoutTemplate]) -> WorkoutTemplate?

    /// ルーティンの全エントリを WorkoutSession に展開する（v1.1.1 改修）
    /// - setEntries が空でなければ優先使用（v1.1.1）
    /// - setEntries が空（レガシー）の場合は従来通り sets ループ
    /// - existingSession があれば末尾に追加、なければ新規に WorkoutSession を作成
    /// - Returns: 挿入したセット数
    @discardableResult
    static func applyRoutine(
        _ routine: WorkoutTemplate,
        to date: Date,
        existingSession: WorkoutSession?,
        modelContext: ModelContext,
        exercises: [Exercise]
    ) -> Int
}

WeekdayPicker UI コンポーネント

SaveTemplateView / EditTemplateView で使用する曜日選択 UI。

項目内容
ファイルViews/Components/WeekdayPicker.swift
表示日〜土の7ボタンを横並びで表示。選択状態は AppTheme.accent で塗りつぶし
バインディング@Binding var selectedWeekdays: [Int]
複数選択可。同一ルーティンに月・水・金など複数曜日を割り当て可能
表示形式「日」「月」「火」「水」「木」「金」「土」（1文字ラベル）

ビジネスルール

  • 1つのルーティンに複数曜日を割り当て可能（例: 月・水・金）
  • 同一曜日に複数のルーティンが割り当てられている場合、RoutineSuggestionService.todaysRoutine は先頭1件のみを返す（作成日降順クエリ結果の先頭）
  • assignedWeekdays が空配列のルーティンは「曜日固定なし」として、自動提案の対象外（TemplateListView から手動適用のみ）
  • ルーティン適用は「末尾追加」。既存セッションがあれば既存セットの後ろに挿入し、上書きはしない
  • TemplateEntry.exerciseName と一致する Exercise が見つからない場合、そのエントリはスキップ（名前変更・削除されたケースへの対応）
  • セット数上限: TemplateListView の loadTemplate では min(entry.sets, 20) で上限制御

外部連携

連携先用途備考
SwiftData ModelContextWorkoutSession / WorkoutSet の INSERT@MainActor 必須
Calendar.current曜日判定（component(.weekday, from:)）1=Sunday を 0 に正規化して使用

エラー処理

発生条件対応
TemplateEntry に対応する Exercise が存在しない（名前変更・削除）そのエントリをスキップして処理を継続。エラー表示なし
assignedWeekdays に無効値（0〜6 外）が含まれるappliesTo(date:) 内で contains 判定が false になるため影響なし
ルーティン適用後のセット数が多すぎる（テンプレート由来）TemplateListView: min(entry.sets, 20) で上限制御

実装メモ

  • RoutineSuggestionService は @MainActor enum（状態なし、静的メソッドのみ）
  • WorkoutTemplate.appliesTo(date:) は Calendar.weekday から 1 を引いて 0-based に正規化（0=日曜）
  • WorkoutSessionView の todaysRoutine は @Query から取得した templates を RoutineSuggestionService に渡す computed property
  • エンプティCTA のプライマリカードは AppTheme.accent 背景 + 黒文字 + shadow(accent.opacity(0.4), radius: 10)
  • WeekdayPicker は SaveTemplateView / EditTemplateView の「曜日選択」セクションに配置
  • CalendarView の予定ドットは枠線スタイル（stroke）で実績ドット（fill）と区別
  • 通知リマインダー（v1.2.0 実装予定）: ルーティン当日の朝に「今日は〇〇の日です」プッシュ通知
