シェアカード機能

5テーマ + Instagram Stories 対応のSNS共有（v1.0.8: 写真保存・ウォーターマーク・解像度 Premium 差分化）

v1.2機能ID: WD-FEAT-SHARE

概要

WorkoutSession や StatsView の結果を画像化してSNSに共有する機能。5テーマから選択可能、Instagram Storiesへの直接シェア対応。v1.0.7 でカード種類に「月次サマリー」「部位別ダイジェスト」を追加、Instagram Story 遷移バグを修正。v1.0.8 でシェアシートに「写真に保存」ボタンを追加し、無料ユーザー向けウォーターマークと解像度 Premium 差分化を実装。

月次サマリーカード（Monthly）詳細

  • 集計期間: 閲覧月の1日〜末日
  • 最多部位: 6部位（chest / back / shoulders / arms / legs / abs）+ cardio / other を含む全カテゴリ対象
  • 月間TOP種目: 最大5件をボリューム降順で表示
  • 実装ファイル: MonthlySummaryCardView.swift, MonthlySummaryCalculator.swift, MonthlySummaryData.swift

部位別ダイジェストカード（Body Part）詳細

  • 集計対象: 通常ウエイト6部位のみ（chest / back / shoulders / arms / legs / abs）。cardio と other は除外
  • 期間: 月（デフォルト）/ 週 切替可能
  • 表示: 6部位それぞれのセット数を横棒グラフで可視化
  • 部位別カラー: chest=赤、back=青、shoulders=黄、arms=紫、legs=緑、abs=ピンク
  • 実装ファイル: BodyPartDigestCardView.swift, BodyPartDigestCalculator.swift, BodyPartDigestData.swift

写真に保存機能（v1.0.8追加）

PhotoLibrarySaver（Services層）

  • 写真ライブラリへの書き込み専用権限 NSPhotoLibraryAddUsageDescription を .addOnly で要求
  • 大きな UIImage を直接渡すとメモリ起因でクラッシュするため、PNG として一時ファイル（tmp/workout-share-{UUID}.png）に書き出し後、PHAssetCreationRequest.addResource(with: .photo, fileURL:) で保存
  • 保存後に defer で一時ファイルを削除
  • 権限エラー / エンコードエラー / ライブラリエラーをそれぞれ SaveError enum で区別して呼び出し元に throw
// 権限取得 → 一時ファイル書き出し → PHAssetCreationRequest
let status = await PHPhotoLibrary.requestAuthorization(for: .addOnly)
// ...
let request = PHAssetCreationRequest.forAsset()
request.addResource(with: .photo, fileURL: fileURL, options: nil)

UX フロー

  1. 「写真に保存」ボタンをタップ → isSavingToPhotos = true（ProgressView 表示）
  2. PhotoLibrarySaver.save(image) を非同期実行
  3. 成功 → 成功トーストを表示（無料: 4秒 / Premium: 1.5秒）
  4. 無料ユーザーのみ、トースト下に「✨ Premiumで透かしなし・高解像度に →」ボタンを表示してPaywallシートへ誘導
  5. 権限拒否 → 「設定アプリを開く」アラート表示
  6. 保存失敗 → 「保存に失敗しました」アラート表示

ウォーターマーク（v1.0.8追加）

  • 共通コンポーネント WatermarkView（Views/Components/）を実装
  • フォント: system size 10 / semibold、前景色 white.opacity(0.95)
  • 背景: black.opacity(0.35) の Capsule シェイプ
  • 合成: ShareViewModel.renderSingle 内で StoreManager.shared.isPremium を判定し、非 Premium 時のみ overlay（bottomTrailing）で追加
  • Instagram Story フォーマット時も同様に適用される（overlay は共通パス）

処理フロー

  1. ShareViewModel が選択テーマ・カード種類でViewを構築
  2. renderSingle 内で isPremium 判定 → 非 Premium 時は WatermarkView を overlay で合成
  3. ImageRenderer で UIImage 生成（scale = renderScale(isPremium:)）
  4. 標準シェア: UIActivityViewController で共有
  5. 写真保存: PhotoLibrarySaver.save（一時ファイル経由）
  6. Instagram Stories: JPEG変換(quality 0.9) → UIPasteboard(localOnly:true) → instagram-stories://share で直接Instagram Story編集画面へ遷移

Instagram Stories 対応（v1.0.7改善後）

// JPEG変換でサイズ削減
let imageData = uiImage.jpegData(compressionQuality: 0.9)
// localOnly指定でセキュリティ向上
UIPasteboard.general.setItems(
    [["com.instagram.sharedSticker.backgroundImage": imageData]],
    options: [.localOnly: true]
)
// completionHandlerでログ出力・安定遷移
UIApplication.shared.open(
    URL(string: "instagram-stories://share")!,
    options: [:],
    completionHandler: { success in /* Logger出力 */ }
)

ビジネスルール

  • プレミアムテーマ・Monthly・Body Part カードは isPremium=true ユーザーのみ選択可
  • ウォーターマーク・解像度差分は isPremium=false の場合に自動適用（ユーザー操作不要）
  • 写真保存後の Premium 訴求トーストは saveToPhotos 完了時のみ表示（SNSシェアフローでは非表示）
  • Instagram 未インストール時は通常 ShareSheet にフォールバック
  • 画像内にApp Storeリンク（QRコード or テキスト）含む
