Undo Toast システム

削除操作を即座に反映しつつ 5 秒間 Undo を可能にするトースト通知システム（v1.1.1）

v1.0 機能ID: WD-FEAT-UNDO-TOAST

概要・目的

従来の削除確認ダイアログ（Alert）は操作の流れを止め、UX の摩擦となっていた。Undo Toast システムは「先に削除してから取り消せる」パターンを採用し、スムーズな操作感を実現する。

ユーザー価値

  • 削除確認のタップ 2 回が不要になり、1 スワイプ で削除できる
  • 誤削除した場合でも 5 秒以内に「元に戻す」で復元できる
  • 削除結果がリストから即座に消えるため、操作が成功したことが視覚的に分かる

適用範囲（v1.1.1）

操作対象画面
種目削除ExerciseListView（左スワイプ）
セット削除（個別）WorkoutSessionView のセット行
種目グループ全削除WorkoutSessionView の種目ヘッダー
ルーティン削除TemplateListView（🗑 アイコンタップ）

構成要素

ToastMessage（struct）

struct ToastMessage: Identifiable, Equatable {
    let id: UUID
    let title: String
    let undoHandler: (() -> Void)?    // nil なら通常メッセージ Toast
    let commitHandler: (() -> Void)?  // 5秒満了 or 上書き時に実行
}
  • Equatable は id のみで比較（クロージャは比較不可のため除外）
  • undoHandler が nil の場合は「元に戻す」ボタンを非表示にした通常 Toast として利用できる

ToastManager（@Observable @MainActor singleton）

@MainActor
@Observable
final class ToastManager {
    static let shared = ToastManager()
    private(set) var currentToast: ToastMessage?
    private var commitTask: Task<Void, Never>?

    static let displayDuration: TimeInterval = 5.0

    func showUndoToast(title: String, undo: @escaping () -> Void, commit: @escaping () -> Void)
    func undo()
    func commitCurrent()
}
メソッド処理
showUndoToast(title:undo:commit:)既存トーストを commitCurrent() で確定してから新トーストをセット。5 秒タイマーをリセット
undo()タイマーをキャンセルし undoHandler を実行。currentToast を nil にする
commitCurrent()タイマーをキャンセルし commitHandler を実行。currentToast を nil にする
タイマーキャンセルの重要性
commitTask は Task { [weak self] in ... } で保持し、currentToast?.id で一致確認してから commitCurrent() を呼ぶ。先に undo/commit が呼ばれた場合は id 不一致でスキップされ、二重 commit を防ぐ。

ToastOverlay（View）

ContentView の .overlay(alignment: .bottom) に配置するオーバーレイ View。

項目値・説明
背景Color(.secondarySystemGroupedBackground)（不透明・コンテンツが透けない）
枠線AppTheme.accent.opacity(0.45)
影shadow(color: .black.opacity(0.45), radius: 18)
「元に戻す」ボタンAppTheme.accent 背景・黒文字・Capsule 形状
底辺余白padding(.bottom, 130): タブバー + 広告 + ボトムアクションバーの上に表示
アニメーション.spring(duration: 0.35) で下から滑り込む

保留中削除パターン（各 View 側の実装）

// View の @State
@State private var pendingDeletionIDs: Set<UUID> = []

// 削除アクション
func deleteSomething(_ item: SomeModel) {
    pendingDeletionIDs.insert(item.id)  // ① リストから即除外
    ToastManager.shared.showUndoToast(
        title: "「\(item.name)」を削除しました",
        undo: {
            pendingDeletionIDs.remove(item.id)  // ② Undo: 復元
        },
        commit: {
            modelContext.delete(item)            // ③ 確定: 実際に削除
        }
    )
}

// @Query フィルター or List 描画でフィルタリング
var displayItems: [SomeModel] {
    allItems.filter { !pendingDeletionIDs.contains($0.id) }
}
注意
pendingDeletionIDs は View の @State で管理する（ToastManager に持たせない）。View が破棄された際に自動クリアされ、リーク・不整合を防ぐ。
ただし View が破棄される前に commitTask が実行される場合は modelContext が有効であることを確認すること。

処理フロー: 削除 → Toast → Undo / Commit

sequenceDiagram participant U as ユーザー participant V as View participant TM as ToastManager participant DB as SwiftData U->>V: 削除操作（スワイプ / タップ） V->>V: pendingDeletionIDs に ID を追加 V->>V: リストから除外（即時反映） V->>TM: showUndoToast(title:undo:commit:) TM->>TM: 既存トーストを commitCurrent() TM->>TM: 新 ToastMessage をセット TM->>TM: 5秒タイマー開始 alt 5秒以内に「元に戻す」をタップ U->>TM: undo() TM->>TM: タイマーキャンセル TM->>V: undoHandler() 実行 V->>V: pendingDeletionIDs から ID を除外（復元） TM->>TM: currentToast = nil（Toast 消去） else 5秒経過 or 別削除でトーストが上書き TM->>TM: commitCurrent() TM->>V: commitHandler() 実行 V->>DB: modelContext.delete(item) TM->>TM: currentToast = nil（Toast 消去） end

ビジネスルール

  • Toast の表示時間は 5 秒固定（ToastManager.displayDuration）
  • 複数の削除を連続して行った場合、前の Toast は即座に確定されてから新 Toast が表示される
  • 「元に戻す」は最後の 1 件のみ取り消せる（複数 Undo キューは持たない）
  • Toast はアプリ全体で 1 件のみ（ToastManager.shared で管理）
  • Toast の「元に戻す」ボタンはアクセシビリティラベルを設定すること（VoiceOver 対応）

エラー処理

発生条件対応
commitHandler 内の modelContext.delete() が失敗SwiftData はデフォルトで try? を使用。エラーは Logger でログ出力し UI 通知はしない
View が破棄された後に commitTask が発火[weak self] パターン + id 一致確認でスキップ。pendingDeletionIDs は @State のため View 破棄時にクリアされるが、commitHandler のクロージャは独立して保持されるため DB 削除は実行される

実装メモ

  • ToastManager は @Observable singleton（static let shared）
  • ToastOverlay は ContentView の .overlay(alignment: .bottom) に配置
  • 各 View の @State pendingDeletionIDs: Set<UUID> でリスト除外を管理
  • commitTask は [weak self] + id 一致確認で二重 commit を防止
  • ExerciseListView: 左スワイプ削除で Undo Toast
  • TemplateListView: 🗑 アイコンタップで Undo Toast（確認ダイアログ廃止）
  • WorkoutSessionView: セット削除・種目グループ削除で Undo Toast

リスク・トレードオフ

リスク内容対策
既存 Alert との一貫性一部の削除が Alert、一部が Toast になると操作が混在するv1.1.1 以降は削除操作をすべて Toast に移行する方針。Alert は他の操作（ログアウト等）のみ使用
5 秒の短さ5 秒は短すぎる可能性がある（特に低速ユーザー）将来的に設定から変更できるようにすることも検討。現時点は 5 秒固定
pendingDeletionIDs リークView が破棄されず長期保持された場合、不要な ID が残り続けるcommitHandler が呼ばれれば DB 削除は完了する。次回起動時には @Query から自動排除されるため実害なし

将来拡張

  • 種目追加の Undo（追加直後に「追加を取り消す」Toast を表示）
  • ワークアウトセッション全体の Undo（セッション開始直後の取り消し）
  • 複数 Undo キュー（現状は 1 件のみ）
  • Toast 表示時間のユーザー設定（3 秒 / 5 秒 / 8 秒）
