ExerciseStatsCalculator

種目ごとの統計値（PR・最終使用日・累計セット数）をメモ化して提供するサービス（v1.1.1）

v1.0 機能ID: WD-FEAT-EXERCISE-STATS

概要・目的

ExerciseListView の各行では「PR 100kg · 先週 · 計47セット」形式のメタ情報を表示する。このメタ情報を算出するには全 WorkoutSession × 全 WorkoutSet を種目ごとにフィルタリングする必要があり、素朴な実装だと View body の再描画のたびに sessions.flatMap { ... }.filter { ... } を全種目分繰り返すことになる（種目数 × セット数の計算量爆発）。

ExerciseStatsCalculator は算出結果を [Exercise.id: ExerciseStats] のキャッシュに保持し、セッション状態が変化したときのみキャッシュを破棄することでこの問題を解決する。

関連画面

計算する統計値の定義

struct ExerciseStats {
    let pr: Double?           // 最大重量（kg）。1セットも記録がない or 全て weight=0 の場合は nil
    let prReps: Int?          // PR 時の回数。pr が nil の場合は nil
    let lastUsedDate: Date?   // 最終使用日（WorkoutSet.session.date の最大値）
    let totalSets: Int        // 累計セット数（種目に紐づく WorkoutSet の総数）
    let totalSessions: Int    // 累計セッション数（種目を含むユニーク日数）

    static let empty: ExerciseStats  // 全 nil / 全 0（記録なし時に返す）
}
フィールド表示例算出方法
pr「PR 100 kg」matchingSets.max(by: { $0.weight < $1.weight })?.weight。0 の場合は nil 扱い
prReps「PR 100 kg × 5 回」（詳細画面用）最大 weight のセットの reps
lastUsedDate「先週」「3日前」（相対表示）matchingSets.compactMap { $0.session?.date }.max()
totalSets「計 47 セット」matchingSets.count
totalSessions（詳細画面用）Set(matchingSets.compactMap { $0.session?.id }).count

メモ化シグネチャの設計

private struct SessionsSignature: Equatable {
    let sessionCount: Int
    let totalSets: Int

    init(sessions: [WorkoutSession]) {
        self.sessionCount = sessions.count
        self.totalSets = sessions.reduce(0) { $0 + $1.sets.count }
    }
}

セッション数と総セット数の2値でセッション状態の変化を検知する。セット追加・削除・セッション追加・削除のいずれかが発生すると必ず値が変化する。ただし「同じセッション数・同じセット数で値だけが変更された場合」はキャッシュが破棄されない（値変更時は totalSets は変わらないため検出できない）。

制約: セット値変更時のキャッシュ非無効化
既存セットの weight や reps を変更した場合（追加・削除なし）、シグネチャが変化しないためキャッシュが残る。
ExerciseListView では PR や最終使用日を表示するにすぎないため、セット値変更後に再表示されるまでの遅延は許容する設計判断。
確実な再描画が必要な場合は invalidate() を呼ぶこと。

クラス定義

@MainActor
@Observable
final class ExerciseStatsCalculator {
    private var cache: [UUID: ExerciseStats] = [:]
    private var lastSignature: SessionsSignature = .init(sessionCount: -1, totalSets: -1)

    /// 指定種目の統計値を返す。キャッシュヒットしない場合は計算して保存する
    func stats(for exercise: Exercise, sessions: [WorkoutSession]) -> ExerciseStats

    /// テスト用 / 強制再計算用（セット値変更後に確実に再計算したい場合）
    func invalidate()
}

処理フロー: stats(for:sessions:) の呼び出し

sequenceDiagram participant V as ExerciseListView participant C as ExerciseStatsCalculator participant Cache as cache[Exercise.id] V->>C: stats(for: exercise, sessions: sessions) C->>C: SessionsSignature(sessions: sessions) を計算 alt シグネチャが変化（セッション追加/削除 等） C->>Cache: cache.removeAll() C->>C: lastSignature を更新 end alt キャッシュヒット Cache-->>C: ExerciseStats を返す else キャッシュミス C->>C: compute(for: exercise, sessions: sessions) C->>Cache: 結果を保存 Cache-->>C: ExerciseStats を返す end C-->>V: ExerciseStats

View body 再描画との関係

ExerciseListView では @State private var statsCalculator = ExerciseStatsCalculator() で保持し、各行の描画時に statsCalculator.stats(for: exercise, sessions: sessions) を呼ぶ。

  • 初回描画時: 全種目分を順次計算してキャッシュに保存（N 種目 × M セットの計算は初回のみ）
  • 2 回目以降（シグネチャ変化なし）: キャッシュヒットで O(1) 返却
  • セッション追加・削除後（シグネチャ変化）: キャッシュ全破棄 → 再計算
@Observable との相性
ExerciseStatsCalculator は @Observable だが、cache と lastSignature は private なため View は直接観察しない。View は sessions の変化（@Query による自動再描画）をトリガーに stats(for:sessions:) を再呼び出しする。

ビジネスルール

  • weight = 0 のセット（デフォルト値や有酸素種目）は PR 計算から除外する（pr = nil）
  • 有酸素種目（useCardioInput = true）は weight = 0 のため pr は常に nil になる
  • totalSessions はユニークなセッション ID 数。同一セッションに同じ種目を複数回行った場合もカウントは 1
  • キャッシュのスコープはアプリのメモリ上のみ。アプリ終了でリセット

エラー処理

発生条件対応
Exercise に対応する WorkoutSet が 0 件ExerciseStats.empty を返す（pr=nil, lastUsedDate=nil, totalSets=0）
WorkoutSet.session が nil（孤立セット）compactMap で除外。totalSessions・lastUsedDate の計算から自動除外される

実装メモ

  • ExerciseStatsCalculator は @MainActor @Observable final class
  • ExerciseListView の @State で保持（View 生存中はキャッシュが維持される）
  • compute() は sessions.flatMap で全セットを取得してから exercise.id でフィルタリング
  • SessionsSignature は sessionCount + totalSets の 2 値で変化を検知
  • invalidate() はテスト / 強制再計算用（通常の使用では不要）

将来拡張

  • 種目詳細画面のグラフ表示: ExerciseStatsCalculator を拡張して「週別最大重量の時系列データ」を追加し、折れ線グラフで表示
  • 種目別 PR タイムライン: PR 更新の履歴（日付・重量・回数）を算出してタイムライン表示
  • ボリューム（重量 × 回数 × セット数）の追跡: 週次・月次ボリュームの集計
  • シグネチャの精緻化: セット値変更時もキャッシュ無効化するため、totalWeight などのハッシュ値を追加（計算コストと相談）
