v1.0.8 機能ロードマップ企画書

product-strategist評価（2026-05-17）の Sランク4項目を v1.0.8 で実装する

draft 企画ID: WD-PLAN-V108

背景

v1.0.7 リリース後の戦略評価で、現アプリは「日本版 Hevy-lite × 記録最速UX」路線で勝ち筋が見えていると判定された。スコア上の最大ボトルネックはマネタイズ B+ と SNS 拡散導線（Instagram Story 一時停止中）。v1.0.8 ではこの2軸を補強し、年間¥143,000の収益目標達成を後押しする。

v1.0.8 で実装する Sランク4項目

① 体重記録機能（最優先・実装コスト低）

項目内容
目的ダイエット層を獲得し、継続率と通知効率を上げる
主要UI統計タブに「体重」セクションを追加（折れ線グラフ + 入力ボタン）。設定からゴール体重を設定可能
データモデル新規 BodyWeightEntry（id / date / value(kg) / note）を SwiftData @Model で追加
通知朝の体重記録リマインダー（オプション）。ストリーク危機通知と独立
シェア連携月次サマリーカードに「体重推移」を1行追加（前月比 +/- kg）
Free/Premium記録は Free、過去30日以上のグラフ表示は Premium（ライト感あるロック）
見積もり10〜15時間（モデル・グラフ・通知・ローカライズ込み）

② Instagram Story 復活（Facebook App ID 取得後）

項目内容
目的筋トレ界隈のスクショ/ストーリー文化に乗り、SNS拡散を取り戻す
前提作業Facebook for Developers でアプリ登録、App ID 取得（スマホで再挑戦）
主要変更InstagramShareService に com.instagram.sharedSticker.sourceApplication を追加、URL に ?source_application=<APP_ID> を付与
UI復活ShareCardFormat.availableCases から filter を外すだけ（実装は v1.0.7 で残してある）
追加対応TikTok / Threads 縦長フォーマット用に別 case を追加（Story と画像サイズ共通化）
見積もり3〜5時間（App ID取得が別タスク）

③ PRタイムライン（SNS相性最強）

項目内容
目的「ベンチ100kg達成」「スクワット更新」のような自動カードでSNS拡散を促進
主要UI統計タブに「PR履歴」セクションを新設。時系列で過去のPR達成日・種目・記録を一覧表示。各PRをタップでシェアカード起動
データソース既存の PRDetector を毎セッション保存時に呼び出して PersonalRecordLog（@Model）に記録
マイグレーション過去セッションを起動時に1度だけ走査して PR ログを生成（重い処理は async）
シェアカード追加「PR Milestone」カード — 該当種目の初回100kg到達などキリ番達成時に特別デザイン
Free/PremiumPR履歴閲覧は Free、PRシェアカードは Premium（既存通り）
見積もり15〜20時間（マイグレーション・グラフ・シェア統合）

④ 週次レポート強化（Paywall導線兼用）

項目内容
目的毎週日曜にプッシュで「今週の成果」を要約し、Paywall への自然な誘導点にする
主要UI毎週日曜21:00 にローカル通知 →タップで WeeklyReportView（フルスクリーン）→「シェア・保存はPremium」CTA
レポート内容総セット数 / 総ボリューム(kg) / PR更新数 / 継続日数 / 部位別バランス / 来週の提案（任意）
共有形式既存の WeeklySummaryCardView を全画面プレビュー化、Premium で画像保存 / SNSシェア
通知設定既存 NotificationService に scheduleWeeklyReport(weekday: 1, hour: 21) 追加
見積もり8〜12時間（通知・全画面UI・Paywall導線）

実装順序とマイルストーン

週作業累積工数

除外する機能（後回しの判断）

  • AI トレ提案 — コスト高・期待値高すぎ・個人開発で保守重い
  • SNSフィード / コミュニティ — Hevy が既に占有、差別化困難
  • Advanced Stats（高度分析）— 大半のユーザーは見ない、Premium価値としては弱い
