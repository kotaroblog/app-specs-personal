シェア画面（WorkoutShareSheet）

ワークアウト結果のSNS共有用カード生成（v1.0.8: 写真保存・ウォーターマーク・Premium訴求トースト追加）

v1.2画面ID: WD-SHARE-01

目的・役割

WorkoutSession や StatsView の結果を画像化してSNSにシェアする画面。5テーマ + Instagram Stories 対応。v1.0.7 よりカード種類に「月次サマリー」「部位別ダイジェスト」を追加、Instagram Story シェアの安定性を改善。v1.0.8 で写真への直接保存に対応、無料ユーザー向けウォーターマーク・高解像度差分で課金訴求を強化。

UI 構成

要素種類説明
テーマ選択PickerDefault / Dark / Minimal / Ocean / Sunset
カード種類選択PickerDaily / PR / Weekly / Progress / Monthly / Body Part（後3種はPremium限定）
期間切替ボタンSegmentedControlカード種類「Body Part」選択時のみ表示。月 / 週 を切替
プレビューShareCardPreviewView生成された画像のプレビュー（無料時はウォーターマーク付き）
「シェア」ボタンButton（プライマリ）UIActivityViewController 起動
「写真に保存」ボタンButton（セカンダリ）v1.0.8追加PhotoLibrarySaver.save を呼び出し写真ライブラリへ直接保存。保存中は ProgressView を表示
「Instagram Stories」Button専用フォーマットで Instagram に直接遷移
保存成功トーストoverlay（top alignment）v1.0.8追加保存完了時に画面上部にカプセル型トーストを表示。無料ユーザーには Premium 訴求リンクを追加表示

アラート・通知一覧

トリガー種類内容

実装メモ

  • ShareViewModel が画像生成を統括
  • ImageRenderer で UIImage 生成
  • Instagram Stories 用は別フォーマット（縦長）
  • Instagram Story シェア: PNG→JPEG(quality 0.9)変換でサイズ削減（v1.0.7）
  • UIPasteboard に localOnly: true を指定（v1.0.7）
  • UIApplication.shared.open に completionHandler を追加してログ出力、Story編集画面への正常遷移を実現（v1.0.7）
  • Monthly / Body Part カードは isPremium=true ユーザーのみ選択可
  • 写真保存: PhotoLibrarySaver.save（一時ファイル経由でメモリ起因クラッシュを回避）（v1.0.8）
  • 写真権限: NSPhotoLibraryAddUsageDescription を .addOnly で要求（v1.0.8）
  • WatermarkView を renderSingle の overlay（bottomTrailing）で合成 — isPremium=false 時のみ表示（v1.0.8）
  • ShareCardFormat.renderScale(isPremium:) で無料@2x / Premium@3x を切替（v1.0.8）
  • 保存成功トーストの表示時間: 無料1.5秒→4秒に延長し、Premium 訴求リンクのタップ時間を確保（v1.0.8）
