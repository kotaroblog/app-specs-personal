種目一覧画面（ExerciseListView）

種目マスタの一覧・追加・編集

v1.1画面ID: WD-EXERCISE-01

目的・役割

登録済みの種目（Exercise）を一覧表示し、追加・編集・削除・ワークアウト追加を行う画面。カテゴリ別フィルタ対応。v1.1.0 で行の操作 UI を「➕・✏️・🗑」アイコン常時表示に刷新し、スワイプアクションを廃止した。

UI 構成（v1.1.0 以降）

要素種類説明
カテゴリフィルタPicker（segmented 等）胸/背中/肩/腕/脚/腹筋/有酸素/その他 でフィルタ
種目リストList名前・カテゴリ・有酸素フラグ表示
➕（ワークアウト追加）アイコンButton（各行右端）タップで AddWorkoutView を該当種目プリセット状態で起動。行タップと同機能
✏️（編集）アイコンButton（各行右端）ExerciseFormView（編集モード）を起動
🗑（削除）アイコンButton（各行右端）確認ダイアログ後に削除
追加ボタンToolbarItem / NavigationLinkExerciseFormView（新規）を起動
v1.1.0 変更点
- スワイプアクション（削除）を廃止
- 各行に「➕（ワークアウト追加）・✏️（編集）・🗑（削除）」アイコンを常時表示
- 行タップ / ➕アイコンタップ = AddWorkoutView（該当種目プリセット）に統一
- セルタップ = ExerciseFormView（編集）だった旧仕様を廃止

遷移

操作遷移先
行タップ / ➕アイコンタップAddWorkoutView（該当種目プリセット状態、sheet）
✏️アイコンタップExerciseFormView（編集モード）
追加ボタンExerciseFormView（新規）
🗑アイコンタップ確認ダイアログ → 削除 or キャンセル

使用するデータモデル

@Model final class Exercise {
    var id: UUID
    var name: String
    var category: String  // ExerciseCategory.rawValue
    var memo: String
    var createdAt: Date
    var useCardioInput: Bool = false
}

// v1.1.0 追加: 部位カラー
enum ExerciseCategory: String, CaseIterable {
    case chest, back, shoulders, arms, legs, abs, cardio, other

    var color: Color { ... }  // 8部位のカラー定義（カレンダーヒートマップ等で使用）
    var displayName: String   // 日本語表示名
}

実装メモ

  • 初回起動時に insertDefaultExercises() でデフォルト種目を挿入
  • 削除時は @Relationship(deleteRule: .cascade) で関連 WorkoutSet も削除
  • v1.1.0: スワイプ削除廃止。「➕・✏️・🗑」アイコンを各行右端に常時表示
  • v1.1.0: 行タップ = ➕アイコンタップ = AddWorkoutView（プリセット）に統一
  • v1.1.0: ExerciseCategory に color プロパティ追加（カレンダーヒートマップ用）
