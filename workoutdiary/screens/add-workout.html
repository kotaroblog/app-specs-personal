セット追加画面（AddWorkoutView）

新しいセットの種目・重量・レップを入力

v1.4画面ID: WD-ADD-01

目的・役割

WorkoutSessionView から FAB タップで sheet 起動。種目を選択し、重量・レップ（または時間・距離）を入力する画面。v1.0.7 でクイック調整ボタンと最近使った種目セクションを追加。v1.1.0 で「前回をコピー」ボタンを追加し、QuickWeightField / QuickRepsField の縦書き問題を修正。

UI 構成

要素種類説明
最近使った種目横スクロールリスト直近5件のユニーク種目チップ。タップで部位選択をスキップしてセット入力シートへ直接遷移。中触覚フィードバック（UIImpactFeedbackGenerator.medium）付き
部位選択カテゴリ選択グリッド胸/背中/肩/腕/脚/腹筋/有酸素/その他
種目選択部位フィルタ済みリスト部位選択後に該当種目を表示
前回をコピーボタン（v1.1.0）Button（種目名直下）「doc.on.clipboard.fill」アイコン + 前回セッションのこの種目のセット数と重量サマリを表示。タップで重量・回数・セット数を一括復元。前回値がない場合は非表示
重量クイック調整QuickAdjustFields有酸素以外で表示。−5 / −2.5 / [TextField] / +2.5 / +5 のボタン群。横並び2段（1段目: 重量行、2段目: Reps行）
Reps クイック調整QuickAdjustFields−5 / −1 / [TextField] / +1 / +5。軽い触覚フィードバック（light）付き
時間・距離入力NumberField有酸素時に表示（分・km）
1RM 推定プレビューTextEpley 式で推定した1RMを参考表示
前回値表示Text同種目の前回 weight × reps を参考表示
「追加」ボタンToolbarItem（confirmationAction）WorkoutSet INSERT。入力未完了時は disabled

前回をコピーボタン（v1.1.0）

項目内容
表示位置セット入力シート内、種目名直下
表示条件同種目の前回セッション記録が1件以上ある場合のみ表示
動作前回セッションの同種目のセット（重量・回数・セット数）を入力欄に一括コピー
プレビュー表示最大3セットのサマリ（例「60.0kg×10rep, 65.0kg×8rep ...+2 more」）をボタン内に小文字で表示
関連ファイルAddWorkoutView.swift（lastSetsForSelectedExercise / applyPreviousSets）
v1.1.0 修正: QuickWeightField / QuickRepsField の縦書き問題
−2.5 / +2.5 のボタンラベルが縦書きになっていた問題（lineLimit(1) + fixedSize() の未設定）を修正。1行横並びで正常表示されるようになった。

セット入力行レイアウト（v1.0.7〜）

各セット行を縦2段構成に変更。

段内容
1段目（重量行）−5 / −2.5 / [重量TextField] / +2.5 / +5
2段目（Reps行）−5 / −1 / [RepsTextField] / +1 / +5

種目インライン新規登録（v1.1.2）

部位選択後の種目リスト末尾に「+ 新しい種目を追加」行を常時表示し、種目タブに移動しなくてもワークアウト記録中に新規種目を登録できる。

要素説明
「+ 新しい種目を追加」行種目リストの末尾に固定表示。AppTheme.accent カラー + sf symbol「plus.circle」アイコン
ExerciseFormView（sheet）タップで sheet 表示。initialCategory に現在選択中の部位カテゴリをプリセットして渡す
onSaved コールバック保存完了時に新規 Exercise を受け取り、0.3 秒後にその種目でセット入力シートを自動表示
ExerciseFormView の変更点（v1.1.2）
initialCategory: ExerciseCategory? プロパティを追加。呼び出し元から部位カテゴリを渡せるようになった（省略時は .chest など固定値でも可）。
onSaved: (Exercise) -> Void クロージャを追加。保存後にコールバックで新規 Exercise を受け渡す。

状態

状態表示内容
種目未選択部位選択グリッド表示
部位選択済（種目リスト）該当種目リスト + 末尾「+ 新しい種目を追加」行（v1.1.2）
種目選択済（セット入力シート）「前回をコピー」ボタン（前回値あり時） + セット入力フォーム
未入力追加ボタン disabled

追加成功 + PR祝賀演出（WorkoutSessionView overlay）
追加成功（PRなし）sheet を閉じる

実装メモ

  • exercise.useCardioInput で入力フィールド切替（有酸素 / 筋力）
  • 前回値は sessions から同種目の最新 WorkoutSet を取得して表示
  • 追加後 PRDetector でPR検出
  • 最近使った種目: sessions から直近5件のユニーク種目を抽出
  • クイック調整ボタン: Views/Components/QuickAdjustFields.swift
  • 触覚フィードバック: 重量/Repsボタン=light、最近使った種目タップ=medium
  • v1.1.0: 「前回をコピー」ボタン（copyPreviousButton）追加。lastSetsForSelectedExercise → applyPreviousSets で入力欄を一括更新
  • v1.1.0: QuickWeightField / QuickRepsField の縦書き問題を修正（lineLimit(1) + fixedSize()）
  • preSelectedExercise: Exercise? 引数で種目プリセット起動に対応（ExerciseListView / CalendarView から利用）
  • v1.1.2: 種目リスト末尾「+ 新しい種目を追加」行を追加。ExerciseFormView を sheet 起動し initialCategory でカテゴリをプリセット。onSaved コールバックで新規 Exercise を受け取り 0.3 秒後にセット入力シートを自動表示
