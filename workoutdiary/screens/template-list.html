ルーティン選択（TemplateListView）

保存済みのルーティン（ワークアウトテンプレート）を一覧・適用・編集する

v1.3画面ID: WD-TEMPLATE-01

目的・役割

頻繁に行うメニュー（例: 胸の日 / 脚の日 / プッシュデー）をルーティンとして保存・適用する画面。v1.1.0 でラベルを「テンプレート」から「ルーティン」へ変更し、曜日バッジ表示・「➕/✏️/🗑」アイコン常時表示を追加した。v1.1.1 でルーティン削除が Undo Toast 対応（softDelete）に変更された。v1.1.2 でナビゲーションバー「+」ボタンと空状態 CTA「New Routine」ボタンを追加し、ルーティン画面単体で新規作成を完結できるようになった。内部モデル名 WorkoutTemplate は互換維持のため変更なし。

UI 構成（v1.1.2 以降）

要素種類説明
ナビゲーションバー「+」ボタンToolbarItem（.navigationBarTrailing）v1.1.2: タップで CreateTemplateView を sheet 表示。ルーティン 1 件以上あっても常時表示
ルーティンリストList（insetGrouped）ルーティン名・曜日バッジ・エントリプレビュー（種目名・重量×回数×セット数）を表示
曜日バッジCapsule TextassignedWeekdays が空でない場合、ルーティン名横に「月・水・金」形式で表示（AppTheme.accent）
➕（ワークアウト追加）アイコンButton（各行右端）タップで loadTemplate() を実行し dismiss。行タップと同機能
✏️（編集）アイコンButton（各行右端）EditTemplateView を sheet 表示
🗑（削除）アイコンButton（各行右端）v1.1.1: 確認ダイアログを廃止し Undo Toast（保留中削除）に変更
空状態「New Routine」CTAButton（中央）v1.1.2: ルーティン 0 件時にリストの代わりに中央表示。タップで CreateTemplateView を sheet 表示
v1.1.2 変更点（ルーティン新規作成導線追加）
これまでルーティンの新規作成は「ワークアウト記録 → 保存（SaveTemplateView）」経路しかなかった。v1.1.2 で以下 2 か所に新規作成の導線を追加し、ルーティン画面で完結できるようになった。
- ナビゲーションバー右上「+」ボタン（常時表示）
- 空状態の中央 CTA「New Routine」ボタン
いずれも CreateTemplateView を sheet 表示する。
v1.1.0 変更点
- UIラベル「テンプレート」→「ルーティン」に全置換（navigationTitle / ボタンラベル等）
- スワイプアクション削除を廃止し、「➕・✏️・🗑」アイコンを各行右端に常時表示
- assignedWeekdays をもとに曜日バッジを表示
- 行タップ / ➕アイコンタップ = ルーティン適用して dismiss（確認ダイアログ廃止）
内部モデル名（WorkoutTemplate）・localstoreキー等は変更なし（SwiftData互換維持）
v1.1.1 変更点
- 削除 UI 変更: 🗑アイコンタップ時の確認ダイアログを廃止し Undo Toast（保留中削除パターン）に変更
- pendingDeletionIDs: Set<UUID> で保留中削除を管理。リスト表示から即座に除外し、5秒後に確定
- 「元に戻す」タップで削除キャンセル
詳細は Undo Toast システム機能仕様書 を参照

使用するデータモデル

@Model final class WorkoutTemplate {
    var id: UUID
    var name: String
    var createdAt: Date
    // v1.1.0 追加：曜日固定ルーティン
    var assignedWeekdays: [Int] = []   // 0=日, 1=月, ..., 6=土。空=曜日固定なし
    @Relationship(deleteRule: .cascade, inverse: \TemplateEntry.template)
    var entries: [TemplateEntry] = []

    var sortedEntries: [TemplateEntry] { ... }
    func appliesTo(date: Date) -> Bool  // 曜日判定
}

@Model final class TemplateEntry {
    var exerciseName: String
    var exerciseCategory: String
    var weight: Double          // レガシー（後方互換用。setEntries が正源）
    var reps: Int               // レガシー（後方互換用）
    var sets: Int               // レガシー（後方互換用）
    var order: Int
    var duration: Double        // 有酸素: 分（レガシー）
    var distance: Double        // 有酸素: km（レガシー）
    var isCardio: Bool = false
    // v1.1.1 追加：セットごとの値を保持
    @Relationship(deleteRule: .cascade, inverse: \TemplateSetEntry.entry)
    var setEntries: [TemplateSetEntry] = []
}

// v1.1.1 新規モデル
@Model final class TemplateSetEntry {
    var weight: Double = 0
    var reps: Int = 0
    var duration: Double = 0
    var distance: Double = 0
    var order: Int = 0
    var entry: TemplateEntry?
}

実装メモ

  • 適用時（applyRoutine）は setEntries が空でなければ setEntries を優先使用（v1.1.1）
  • レガシー（setEntries 空）の場合は従来通り sets ループで WorkoutSet INSERT
  • 経験・目標で重量・レップを自動補正（UserProfile.goal / UserProfile.experience 参照）
  • 初回起動時に insertDefaultTemplates() でデフォルト挿入
  • v1.1.0: スワイプ削除を廃止し「➕・✏️・🗑」アイコン常時表示に変更
  • v1.1.0: 曜日バッジ weekdayBadge(_ weekdays:) で「月・水・金」形式に変換
  • v1.1.0: 行タップ = ➕アイコンタップ（loadTemplate → dismiss）に統一
  • v1.1.1: 🗑タップ → pendingDeletionIDs に追加 → Undo Toast → 5秒後 modelContext.delete()
  • TemplateListView は WorkoutSessionView から sheet 表示（date: Date を引数で受け取る）
  • v1.1.2: toolbar「+」ボタン + 空状態 CTA から CreateTemplateView を sheet 起動（@State showCreateSheet: Bool で管理）
  • v1.1.2: CreateTemplateView で保存完了後に自動的にリストが更新される（@Query で自動反映）
