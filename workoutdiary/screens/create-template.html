新規ルーティン作成（CreateTemplateView）

ルーティン画面から直接、名前・曜日・種目・セット詳細を入力してルーティンを作成する画面（v1.1.2 新規）

v1.1 画面ID: WD-CREATE-TEMPLATE-01

目的・役割

v1.1.1 まで新規ルーティン作成はワークアウト記録後の「保存（SaveTemplateView）」経路のみだった。v1.1.2 でルーティン一覧画面（TemplateListView）から直接ルーティンを作成できる画面として新規追加。ルーティン名・割り当て曜日・種目セットを一画面で入力し、SwiftData に一括書き込みする。キャンセル時はデータが永続化されないため、ゴミデータは残らない。

UI 構成

要素種類説明
ナビゲーションタイトルNavigationTitle「新しいルーティン」
キャンセルボタンToolbarItem（.cancellationAction）sheet を dismiss。State を破棄するためゴミデータは残らない

DraftEntrySectionView（サブView）

種目ごとのセット詳細入力を担当するサブ View。永続化前のローカル State（ドラフト）を操作するため、SwiftData には触れない。

要素説明
Section ヘッダー部位カラー左ボーダー + 種目名 + 削除ボタン（ゴミ箱アイコン）
セット行「Set N」+ 重量（kg）TextField + 回数（rep）TextField。有酸素種目の場合は 時間（分）+ 距離（km）
「+ セットを追加」ボタン直前セットの値を引き継いで新規セット行を追加

CreateTemplatePickerSheet（サブView）

種目を選択するためのシート。既存 Exercise 一覧を部位グループで表示し、種目を選択するとドラフトに追加する。AddTemplateEntrySheet（EditTemplateView で使うシート）とは独立した実装。

要素説明
種目一覧部位カラー左ボーダー Section で Exercise をグループ表示
検索バー種目名で絞り込み
「+」ボタン（ナビゲーションバー）ExerciseFormView を sheet 表示して種目自体を新規登録できる。登録後に自動で選択状態に

状態表示内容

使用するデータモデル

// 保存時に以下の順で INSERT
@Model final class WorkoutTemplate {
    var name: String
    var assignedWeekdays: [Int] = []   // WeekdayPicker の選択値
    var entries: [TemplateEntry] = []
}

@Model final class TemplateEntry {
    var exerciseName: String
    var exerciseCategory: String
    var order: Int
    var isCardio: Bool
    var setEntries: [TemplateSetEntry] = []
    // レガシーフィールドも同期（後方互換）
    var weight: Double
    var reps: Int
    var sets: Int
}

@Model final class TemplateSetEntry {
    var weight: Double = 0
    var reps: Int = 0
    var duration: Double = 0
    var distance: Double = 0
    var order: Int = 0
}
設計方針: ローカル State で組み立ててから一括 INSERT
CreateTemplateView はフォーム入力中に SwiftData に書き込まない。キャンセル時のゴミデータ問題を避けるため、ドラフト用の struct（例: DraftTemplateEntry）を State で保持し、保存ボタン確定時に一括 INSERT する。EditTemplateView（既存データ直接編集）とは設計アプローチが異なる。

実装メモ

  • ルーティン名が空の場合は保存ボタンを disabled にする（.disabled(routineName.isEmpty)）
  • DraftEntrySectionView は @Binding で親の draftEntries 配列を受け取る。直接 modelContext には触れない
  • CreateTemplatePickerSheet から「+」で ExerciseFormView を sheet 起動する場合、SwiftUI の sheet 二重積みは iOS 16.4+ から安定動作
  • 保存時: WorkoutTemplate → TemplateEntry → TemplateSetEntry の順に INSERT し、各リレーションを設定してから modelContext.insert
  • レガシーフィールド（weight / reps / sets）は setEntries の先頭値・件数から同期（TemplateMigrationService の逆操作）
  • 曜日ピッカーは既存 WeekdayPicker コンポーネントを流用
