ロギング方針

os.log / Logger を用いたログ出力ルール

基本方針

print() は禁止。Logger（os.log）を使用する（CLAUDE.md準拠）。

Logger カテゴリの定義

import OSLog

extension Logger {
    private static let subsystem = Bundle.main.bundleIdentifier!

    static let app = Logger(subsystem: subsystem, category: "app")
    static let ui = Logger(subsystem: subsystem, category: "ui")
    static let data = Logger(subsystem: subsystem, category: "data")
    static let network = Logger(subsystem: subsystem, category: "network")
    static let purchase = Logger(subsystem: subsystem, category: "purchase")
    static let ad = Logger(subsystem: subsystem, category: "ad")
}

ログレベルの使い分け

レベル用途例
.debug開発時のみ確認したい情報VM初期化、画面遷移
.info通常のイベントログイン、購入完了
.notice注目すべきイベント初回起動、設定変更
.warning異常だが継続可能ネットワーク再試行、課金復元失敗
.errorエラー、機能継続不可SwiftData保存失敗
.criticalアプリ継続不可データベース破損

プライバシー対応

Logger 出力時、個人情報・APIキー・トークンを含めない。やむを得ず含む場合は privacy: .private を指定する。 
// OK
Logger.app.info("画面遷移: \(screenName, privacy: .public)")

// NG（メールアドレスを公開ログに出している）
Logger.app.info("ログイン: \(user.email)")

// OK（私的データ扱い）
Logger.app.info("ログイン: \(user.email, privacy: .private)")

禁止事項

  • print() の使用（CI/lint で検知）
  • APIキー・パスワード・課金トランザクション詳細の出力
  • 例外の詳細スタックトレースをユーザー向けログに含める

確認方法

macOS の Console.app でデバイスを選択し、subsystem でフィルタすると当該アプリのログのみ確認可能。
