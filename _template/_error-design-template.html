エラー設計・例外パターン一覧

例外の分類とユーザーへの提示方針

エラー分類の方針

  • システムエラー: 想定外、開発者が修正すべき
  • ビジネスエラー: 業務ルール違反、UIで分かりやすく案内
  • ネットワークエラー: 一時的、リトライ可能
  • 権限エラー: 設定アプリへの誘導が必要

エラー一覧

エラーID 分類 発生条件 ユーザー表示 復旧方法 ロギング
ERR-001 ビジネス 必須項目未入力 「[項目名]を入力してください」 入力促進 不要
ERR-002 ネットワーク API接続失敗 「通信エラーが発生しました。再試行しますか？」 リトライボタン warning
ERR-003 システム SwiftData書き込み失敗 「データ保存に失敗しました」 再起動を促す error
ERR-004 権限 通知許可がない 「通知を有効にすると…」 設定アプリ起動 info

Swift エラー型の方針

Result型 または throws を統一して使用。コールバックでのエラー返却は禁止（CLAUDE.md準拠）。 
enum [App]Error: LocalizedError {
    case invalidInput(field: String)
    case networkUnavailable
    case persistenceFailed(underlying: Error)
    case permissionDenied(type: PermissionType)

    var errorDescription: String? {
        switch self {
        case .invalidInput(let field): return "\(field)を入力してください"
        case .networkUnavailable: return "通信エラーが発生しました"
        case .persistenceFailed: return "データ保存に失敗しました"
        case .permissionDenied: return "権限が必要です"
        }
    }
}
