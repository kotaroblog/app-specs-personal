記録完了

収支保存後に「気になった1手」記録へ誘う最重要ファネル

設計確定・実装未着手 画面ID: PD-03

目的・役割

収支セッション保存の完了を伝える画面。だがこの画面の本質は本アプリ最重要ファネル＝「収支記録 → ハンド記録」への橋渡しにある。「気になった1手、残しますか？」を主誘導に据え、差別化行動（ハンド記録・弱点タグ言語化）へ最大確率で接続する。同時に「今夜はこのまま終える」も罰的にせず同等の軽さで並置し、記録継続の心理的ハードルを上げない。

UI 構成

要素種類説明
完了アイコン＋見出しIcon + Titleチェックマーク＋「記録しました」
保存サマリCaption店・形式・BB収支（リング）またはトナメ結果を1行表示（例: アキバギルド・リング2/5・+200 BB）
ハンド記録 主誘導カードHighlight Card「気になった1手、残しますか？」＋「30秒で記録。続けるほど、負け筋が言葉になる。」＋「ハンドを記録する」主ボタン
終了ボタンPlain Button（軽量）「今夜はこのまま終える」。罰的な表現・色を使わずホームへ戻す

状態

状態表示内容
リング保存後サマリにBB収支（勝ち＝緑／負け＝テラコッタ）を表示
トナメ保存後サマリにトナメ結果（バスト／ITM／優勝）を表示
共通ハンド記録主誘導カードを中心に配置し、終了ボタンを軽く並置

遷移

操作遷移先
「ハンドを記録する」タップハンド簡易記録（PD-04）
「今夜はこのまま終える」タップホーム（PD-01）

使用するデータモデル

Session   // 直前に保存したセッション（サマリ表示用: venue / gameTypeRaw / bbProfit / resultRaw）
// この画面自体は新規保存を行わない。ハンド記録へ進んだ場合、生成される Hand.session に本 Session を紐づける

KPI イベント

イベントパラメータ目的
hand_prompt_shownなし完了画面「1手残す?」の表示数（ファネル分母）
hand_prompt_acceptedなし承諾数（収支→ハンドの最重要ファネル分子）。shown との突合で承諾率を算出

実装メモ

  • 本画面は収支→ハンドの最重要ファネル。主誘導と終了を心理的に対等に見せ、承諾率を計測しながら文言・レイアウトを改善する
  • 「今夜はこのまま終える」は否定色・小さすぎる文字にせず、罰を感じさせない軽さで置く
  • ハンド記録へ進んだ場合、生成する Hand に本 Session を session として紐づける（has_session=true）
  • 離脱は直接イベント化せず、shown と accepted の差分で算出（ファネル設計思想）
