履歴

セッション／ハンドを月区切りで一覧する

設計確定・実装未着手 画面ID: PD-08

目的・役割

記録したセッションとハンドを時系列でたどる一覧画面。セグメントで「セッション／ハンド」を切り替え、月区切りヘッダとフィルタチップで目的の記録に素早く到達する。クイックメモ由来の下書きハンドもここに現れ、後から詳細化・セッション紐づけへ導く。

UI 構成

要素種類説明
セグメントSegmented Controlセッション／ハンドの切替
フィルタチップChip（横並び）すべて／店／タグ。ハンドセグメント時のみ「下書きのみ」を追加
月区切りヘッダSection Header「2026年 7月」等。月ごとに行をグルーピング
セッション行List Row（Card）後述の行仕様①
ハンド行／下書き行List Row（Card）後述の行仕様②③
タブバーTab Barホーム／履歴／分析／設定

行仕様3種

行構成要素
① セッション行店名・形式バッジ（リング／トナメ）・日付・右端にBB収支（リング／勝ち緑・負けテラコッタ）またはトナメ結果（例: 2E ITM・アンバー色）
② ハンド行ミニカード2枚・ポジション＋状況（例: CO ・ vs open）・弱点タグチップ・メモ先頭1行（省略記号付き）
③ 下書き行ミニカード2枚＋下書きバッジ＋メモのみ。ポジション／状況は未入力のため表示しない

状態

状態表示内容
セッションセグメント月区切りでセッション行を一覧。フィルタは すべて／店／タグ
ハンドセグメント月区切りでハンド行・下書き行を一覧。フィルタに「下書きのみ」を追加
下書き混在下書き行は下書きバッジで区別し、ポジション／状況を出さずメモのみ表示
データなしエンプティステート（記録の開始を促す）

遷移

操作遷移先
セッション行 タップセッション詳細（PD-09）
ハンド行 タップハンド詳細（PD-10・通常）
下書き行 タップハンド詳細（PD-10・下書き）

使用するデータモデル

Session // 店・形式・日付・BB収支/トナメ結果（行仕様①）
Hand    // entryModeRaw（simple/quick=下書き判定）/ holeCard1,2 / positionRaw / scenarioRaw
        // weaknessTagsRaw / note（行仕様②③）
Venue   // 店フィルタ・店名表示

実装メモ

  • 閲覧に制限を設けない。ハンドの無料制限は記録数（月10件）のみで、既存記録の閲覧は無制限
  • 下書き判定は entryModeRaw == "quick"。下書き行はポジション／状況を描画しない（未入力を空欄で見せない）
  • 月区切りは Session.date / Hand.createdAt を月単位でグルーピング
  • 「下書きのみ」フィルタはハンドセグメント時のみ表示
