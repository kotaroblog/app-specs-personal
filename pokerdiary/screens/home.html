ホーム

今月の成績ダッシュボードと記録への最短導線

設計確定・実装未着手 画面ID: PD-01

目的・役割

アプリ起動直後の起点画面。今月の成績を2レイヤー収支（実力＝リングBB／トナメ結果、お金＝娯楽費）で一望させつつ、最頻動線である「今夜を記録」へ最短で誘導する。加えて、前週レビューで決めた「次回の課題」を常に目に入る位置に置き、行動変容のループを閉じる。

UI 構成

要素種類説明
今月ヘッダTitle + Chip「今月の成績」見出し＋月セレクトチップ（例: 7月）
今月サマリ3点セットCard × 3（横並び）①リング（実力）＝BB収支（例: +240 BB）／②トナメ＝エントリー数＋ITM数（例: 5エントリー・2 ITM）／③支出（娯楽費）＝円（例: ¥18,000）。円のプラマイ収支は表示しない

「次回の課題」カードCard（左アクセントボーダー）最新 WeeklyReview.nextFocus を表示（例: 「COからの3betに、KJs以下は降りる」）。未設定時は非表示

週次レビュー誘導カードCard + Chevron「日曜の週次レビュー」。日曜〜週次レビュー未完了時に強調表示、タップでPD-06へ

状態

状態表示内容
初回起動（記録0件）エンプティステート。サマリ3点セットは出さず、「今夜を記録」CTAと価値説明のみを表示する
通常今月サマリ・次回の課題（あれば）・店別リスト・週次レビュー誘導カードを表示
日曜（レビュー誘導活性）週次レビューが未完了なら誘導カードを強調表示し、レビューへの着手を促す
次回の課題 未設定「次回の課題」カードを非表示にする（存在しないものを空欄で見せない）

使用するデータモデル

// 今月サマリ・店別成績は Session を月・店で集計（クエリ／computed で算出、非正規化しない）
Session   // date / gameTypeRaw / chipProfit / bbProfit / resultRaw / spentYen / venue
Venue     // name（店別リストの見出し）
WeeklyReview // nextFocus（「次回の課題」）/ completedAt（当週レビュー完了判定）

実装メモ

  • サマリ3点セット・店別成績は Session を月／店でクエリ集計する。Venue に成績を非正規化して持たせない（SSoT一元化）
  • リング＝BB収支の合算、トナメ＝エントリー数・ITM数のカウント、支出＝spentYen 合算で構成
  • 「次回の課題」は completedAt != nil の最新 WeeklyReview の nextFocus。空文字なら非表示
  • 週次レビュー誘導の強調は「曜日が日曜以降 かつ 当週 WeeklyReview 未完了」で判定
  • 負け表示色はテラコッタ（#D0876B）、勝ち表示色はグリーン（#3FB950）でBB収支の正負を表現

未決事項

主CTA「今夜を記録」の配置は、サマリ直下（画面上部）に置く案と、タブバー中央＋ボタン（親指圏固定）に置く案の2案がある。片手到達性（親指リーチ）が判断基準となるため、実装時に実機で両案を検証して決定する。
