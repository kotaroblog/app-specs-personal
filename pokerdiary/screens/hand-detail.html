ハンド詳細

1ハンドの全容。シェア・レンジ逆引き・答え合わせの起点

設計確定・実装未着手 画面ID: PD-10

目的・役割

1ハンドの内容を詳細表示し、シェアカード生成・レンジ逆引き・答え合わせといった従機能への起点となる画面。通常ハンドは全情報＋照合結果を集約し、クイックメモ由来の下書きハンドは「詳細を追加する」「セッションに紐づける」で完成へ導く。

UI 構成（2状態）

通常状態

要素種類説明
ホールカード（大）＋ボードCardホール2枚を大表示、ボードをミニカードで表示
ポジション／状況／アクションLabel例: CO ・ オープンに対して ・ コール
セッションリンクField Row紐づくセッション（店・日付・ステークス）へ遷移

状態

状態表示内容
通常（照合該当）全情報＋照合結果ブロックを表示。シェア・削除ボタンあり
通常（照合非該当）照合結果ブロックを非表示。その他は通常表示
下書き下書きバッジ・「詳細を追加する」主CTA・「セッションに紐づける」行を表示。シェアボタンは出さない
削除操作確認ダイアログを経て削除

遷移

操作遷移先
「レンジを見る」タップ（照合ブロック）レンジ逆引きビューア（PD-17・該当セルハイライト付き）
「似た問題を解く」タップ答え合わせ（PD-18）
「シェアカードを作る」タップシェアカード生成（PD-13）
セッションリンク タップセッション詳細（PD-09）
「詳細を追加する」タップ（下書き）ハンド簡易記録（PD-04）の続き入力
削除 タップ確認ダイアログ → 削除

使用するデータモデル

Hand // entryModeRaw（simple/quick）/ holeCard1,2 / boardCards / positionRaw / scenarioRaw
     // actionRaw / weaknessTagsRaw / note / session?
// 照合結果（HeroPreflopResult相当）は保存しない。position+ホール+scenario から都度再計算
Session // セッションリンク・下書き紐づけ先

実装メモ

  • 照合結果ブロックはPD-07と同型。都度再計算・保存しない
  • 削除は確認ダイアログ必須（誤削除防止）
  • 下書き状態はシェアボタンを出さない（情報不足のカードを共有させない）
  • 「レンジを見る」からPD-17へ遷移時は、該当セルのハイライト情報を渡す
