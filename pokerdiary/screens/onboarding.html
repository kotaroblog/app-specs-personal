オンボーディング

3頁で価値を伝え、通知許可とPaywallへ橋渡し

設計確定・実装未着手 画面ID: PD-15

目的・役割

初回起動時の3頁ウォークスルー。BB収支・弱点タグ・週次レビュー通知という3つの核となる価値を順に伝え、最終頁で通知許可（ソフトアスク→システムダイアログの2段構え）を得る。完了後、初回のみ source=onboarding のPaywallをモーダル提示（スキップ可）してホームへ送る。

UI 構成

頁テーマ内容
①BB収支の価値訴求「アミュ帰りの10秒で、今夜の収支を」。チップIN/OUTからBB収支を自動換算・円のプラマイに縛られない
②弱点タグ＝負け筋の言語化「気になった1手が、負け筋の言葉になる」。弱点タグで記録するほど負けパターンが見える
③週次レビュー通知「日曜夜、5分の振り返り」。ソフトアスク→「通知を許可する」でシステムダイアログ。「あとで」を並置
—共通ステップドット・スキップ（最終頁は非表示）

状態

状態表示内容
頁①②「次へ」で進行。スキップ常設
頁③（最終）「通知を許可する」主CTA＋「あとで（ホームへ）」。スキップは非表示
通知許可タップソフトアスク後にシステムダイアログを表示（2段構え）
完了（初回のみ）source=onboarding のPaywallをモーダル提示（スキップ可）→ ホーム

遷移

操作遷移先
「次へ」タップ次頁
スキップ タップオンボーディング完了処理へ
「通知を許可する」タップシステム通知ダイアログ
完了Paywall（source=onboarding・スキップ可）→ ホーム（PD-01）

使用するデータモデル

// SwiftDataモデルは持たない。初回起動フラグは UserDefaults（AppStorage）で管理
// 通知許可は UNUserNotificationCenter

KPI イベント

イベントパラメータ目的
onboarding_completedなし初回体験の到達率

実装メモ

  • 通知許可はソフトアスク→システムダイアログの2段構え（拒否率を下げる）。「あとで」を並置し強制しない
  • スキップは常設。最終頁のみ非表示
  • 完了後の source=onboarding Paywall は初回のみ・スキップ可
  • 週次レビュー通知の既定は日曜21:00（設定で変更可）
