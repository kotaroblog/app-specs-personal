店詳細

1店舗の通算成績と「いつものステークス」

設計確定・実装未着手 画面ID: PD-12

目的・役割

特定の店舗における通算成績を4指標で俯瞰し、その店でのセッション履歴をたどる画面。「いつものステークス」を最新セッションからクエリ導出し、店ごとの傾向把握を助ける。

UI 構成

要素種類説明
店名ヘッダTitle店名
いつものステークスCaption最新セッションから導出（例: 2/5）
通算4指標Stat群リングBB通算・トナメE/ITM・支出・訪問数
セッション履歴List（Card行）この店のセッション行。タップでセッション詳細（PD-09）

状態

状態表示内容
通常通算4指標＋いつものステークス＋セッション履歴
リング未プレイリングBBは0/空。トナメ・支出・訪問数のみ意味を持つ
セッションなし履歴なし（店マスタのみ存在）

遷移

操作遷移先
セッション行 タップセッション詳細（PD-09）
分析（店別成績）から本画面へ流入

使用するデータモデル

Venue   // name / sessions
Session // その店の全セッション。通算集計（bbProfit合算 / buyInCount・ITM / spentYen / 件数=訪問数）
        // 「いつものステークス」は最新Sessionの sbChips,bbChips からクエリ導出（非正規化しない）

実装メモ

  • 通算4指標・訪問数はその店のSessionをクエリ集計する（Venueに非正規化して持たせない）
  • 「いつものステークス」は最新Sessionから導出（data-model 設計判断「店の前回値は非正規化しない」に準拠）
  • リングBB＝bbProfitの合算、トナメ＝エントリー数・ITM数、支出＝spentYenの合算、訪問数＝セッション件数
