ハンド簡易記録

52セルグリッドピッカーで30秒。負け筋を言葉にする差別化の中核

設計確定・実装未着手 画面ID: PD-04

目的・役割

「気になった1手」を30秒で構造化記録する、本アプリの差別化中核画面。上から順にタップするだけの7ステップで、ホールカード・ボード・ポジション・状況・アクション・弱点タグ・メモを残す。弱点タグの言語化と状況（シナリオ）記録により、後の週次レビューとMTTレンジ照合（PD-07）を成立させる。

UI 構成（7ステップ）

#要素種類説明
1ホールカード52セルグリッドピッカー4スート行×13ランク列。スート色分け（赤=♥♦／黒=♠♣）。選択トグル。3枚目を選ぶと古い方を押し出す（常に最大2枚）。プレビューにカード表示＋「あと○枚」
2ボード52セルグリッドピッカー（任意）0〜5枚。同じピッカーで追加。プレビューに最大5枠
3ポジションChip 9択UTG／UTG+1／UTG+2／LJ／HJ／CO／BTN／SB／BB（9max）
4状況（シナリオ）Chip 4択自分が最初に参加=rfi／オープンに対して=vs_open／スクイーズに対して=cold_call_vs_squeeze／その他=other。「その他」はレンジ照合の対象外である旨を注記表示
5自分のアクションSegmented 3択フォールド／コール／レイズ
6弱点タグChip（複数選択）プリセット10種から最大3つ。選択数カウンタ（○/3）付き。負け色テラコッタで表現
7ひとことメモText Field（任意）1行メモ（例: リバーのオーバーベットにコール、バリュー濃かった）
—保存ボタンPrimary Button「このハンドを保存」

弱点タグ プリセット10種: レンジ外参加／ポジション軽視／コール過多／オーバーブラフ／バリュー逃し／ベットサイズ不適切／ティルト／スタック管理ミス／相手レンジ決め打ち／ドロー追いすぎ。

状態

状態表示内容
ホールカード未完（0〜1枚）プレビューに「あと○枚」を表示。空枠はダッシュ枠
状況=その他 選択時「『その他』はレンジ照合の対象外です。記録だけ残します」を注記表示
状況=照合可能3種 選択時「レンジ照合に使う局面です」を表示
弱点タグ3つ選択済みそれ以上の選択を受け付けない（カウンタ 3/3）
記録完了（PD-03）経由Hand.session に直前 Session を紐づけ（has_session=true）
月10件超（無料）保存時に Paywall（source=handLimit）を提示

遷移

操作遷移先
保存（scenarioRaw が照合可能3種 かつ ホール・ポジション入力済み）MTTレンジ照合バナー（PD-07）
保存（上記以外・その他・入力不足）そのまま完了（ホームまたは呼び出し元へ）
ハンド記録 月10件超Paywall（source=handLimit）
戻る／スキップ呼び出し元へ戻る

使用するデータモデル

Hand // entryModeRaw = "simple"（30秒簡易記録）
     // holeCard1 / holeCard2（"Ah" 形式）/ boardCards[0〜5]
     // positionRaw（9max）/ scenarioRaw（rfi/vs_open/cold_call_vs_squeeze/other）
     // actionRaw（fold/call/raise）/ weaknessTagsRaw[最大3]（enum rawValue）
     // note（1行）/ session?（PD-03経由なら紐づく）

KPI イベント

イベントパラメータ目的
hand_loggedentry_mode, tag_count, has_session差別化行動の量と質。カード内容・メモ本文は含めない

実装メモ

  • ホールカードは常に最大2枚。3枚目タップで最古の1枚を押し出すトグル挙動とする
  • 保存時の照合判定: scenarioRaw ∈ {rfi, vs_open, cold_call_vs_squeeze} かつ ホールカード2枚・ポジション入力済み かつ 該当レンジ表が存在 → PD-07 へ
  • 状況=other は照合対象外。誤って照合しないようガードする
  • カードは "Ah" 形式（Rank+Suit rawValue）で保存。旧PokerLesson の PokerCard / HandNotation と相互変換
  • 無料はハンド記録 月10件。11件目の保存操作で Paywall（source=handLimit）

未決事項

カード入力方式は2案ある。
① 52セルグリッドピッカー（一望性重視・セル幅約25pt）… 全カードを一画面で見渡せるが、タップターゲットが小さい。
② 「ランク→スート」2タップ方式（タップターゲット大・タップ数+2）… 押しやすいが手数が増える。
ボード入力を含む最大7枚選択時の誤タップ率を判断基準に、実機TestFlightで両方式を検証して決定する。
