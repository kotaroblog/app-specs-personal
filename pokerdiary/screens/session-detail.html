セッション詳細

1セッションの成績・ハンド一覧と下書き紐づけ動線

設計確定・実装未着手 画面ID: PD-09

目的・役割

1回のセッションの成績（BB収支／トナメ結果・支出・メモ）と、そのセッションに紐づくハンドを一覧する画面。加えて、プレイ中にクイックメモで残した未紐づけの下書きを、このセッションに一括で紐づける「下書き紐づけ提案」を提供し、記録の取りこぼしをサルベージする。

UI 構成

要素種類説明
ヘッダカードCard店名・形式バッジ・日付＋プレイ時間・BB収支またはトナメ結果・支出・メモ
下書き紐づけ提案カードHighlight Card「この日の下書きが N件 あります」＋「紐づける」ボタン。候補が無ければ非表示
ハンド一覧List（Card行）このセッションに紐づくハンド行。行タップでハンド詳細（PD-10）
「ハンドを追加」Plain Buttonハンド簡易記録（PD-04）を開き、本セッションを自動紐づけ
編集Nav Actionセッション内容の編集

下書き紐づけ提案の判定

候補=「セッションの date の前後24時間以内に作成された、未紐づけ（session == nil）の entryModeRaw == "quick" ハンド」。深夜0時跨ぎ対策として「同日」ではなく±24時間で判定する。承諾（「紐づける」）で候補を一括紐づけする。

状態

状態表示内容
リングセッションヘッダにBB収支（勝ち緑／負けテラコッタ）・支出・プレイ時間を表示
トナメセッションヘッダにトナメ結果（バスト／ITM／優勝）・支出を表示
下書き候補あり紐づけ提案カードを表示（件数付き）
下書き候補なし紐づけ提案カードを非表示
紐づけ承諾後候補ハンドを本セッションに紐づけ、ハンド一覧へ反映

遷移

操作遷移先
ハンド行 タップハンド詳細（PD-10）
「ハンドを追加」タップハンド簡易記録（PD-04・session自動紐づけ）
「紐づける」タップ同一画面内で下書きを一括紐づけ
編集 タップセッション編集

使用するデータモデル

Session // venue / gameTypeRaw / date / durationMinutes / bbProfit / resultRaw / spentYen / note
        // プレイ時間は date + durationMinutes から表示（開始終了時刻は保持しない）
Hand    // hands（紐づくハンド）/ session==nil & entryModeRaw=="quick" が下書き紐づけ候補
Venue   // 店名表示

KPI イベント

イベントパラメータ目的
draft_link_prompt_shownなし下書き紐づけ提案の表示数（サルベージのファネル分母）
draft_link_acceptedなし紐づけ承諾数（クイックメモ→セッション接続のファネル分子）

実装メモ

  • プレイ時間は date ＋ durationMinutes から算出表示する。開始／終了時刻は保持しない
  • 下書き候補は ±24時間・session==nil・entryMode=quick で抽出（0時跨ぎ対策）
  • 「ハンドを追加」から生成する Hand には本 Session を自動紐づけ（has_session=true）
  • KPIのパラメータに金額・店名・自由記述を含めない
