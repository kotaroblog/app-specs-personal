収支記録シート

リング／トナメ2モードの10秒収支入力（習慣の錨）

設計確定・実装未着手 画面ID: PD-02

目的・役割

アミュ帰りの10秒で「今夜の収支」を残すための中核入力シート。リング＝チップIN／OUTからBB収支をライブ換算、トナメ＝結果とプライズ内容を記録する。円は「支出（娯楽費）」のみを扱い、プラマイ収支・円換算・ROIは記録しない（2レイヤー収支・4+整合）。このシートは習慣の錨であり、どの経路でも永久無料でPaywallに阻まれない。

UI 構成

共通

要素種類説明
ナビバーNavigation「今夜を記録」タイトル・キャンセル・日付ピッカー
モードセグメントSegmented Controlリング／トナメの切替。前回使用モードを記憶して初期選択する
店選択チップChip（横スクロール）店名サジェスト（使用頻度順）＋「＋新規」。選択済みはアクセント塗り
保存ボタンPrimary ButtonSession を保存し記録完了画面（PD-03）へ

リングモード

要素種類説明
ステークスチップChipその店の最新Sessionから前回値をクエリ導出して先頭表示（例: 2/5）。他候補も併置
チップINField + Keypad持ち込みチップ（点）。店の前回値をデフォルト
チップOUTField + Keypad終了時チップ（点）。アクティブ入力対象
BB収支ライブ換算Result Card（緑/テラコッタ）chipProfit / bbChips を即時換算表示（例: +200 BB）。補助表記「+1,000点 @2/5（1BB=5点）」
参加費（円）Field（任意）娯楽費として任意入力（例: ¥3,000）→ spentYen

トナメモード

要素種類説明
エントリー費（円）Field1エントリーあたりの費用（娯楽費）
エントリー数Stepperリエントリーを含むエントリー数（buyInCount）
結果Segmented Controlバスト／ITM／優勝の3択（resultRaw = busted / itm / won）
プライズ内容Text Field入賞時のみ表示。テキスト記録（例: リングチップ50,000点）→ prizeNote
支出合計Auto Field（上書き可）エントリー費×エントリー数を自動計算。手動上書き可 → spentYen
注記Caption「アミューズメントのため賞金は出ません。円換算・ROIは記録しません」

状態

状態表示内容
リング初期前回使用モードがリングなら初期表示。店・ステークス・チップINは前回値をプリセット
トナメ初期前回使用モードがトナメなら初期表示。結果未選択・プライズ欄は非表示
入賞（ITM/優勝）選択時プライズ内容テキスト欄を表示する
BB収支換算中（リング）チップIN/OUTとステークスが揃った時点で緑（勝ち）／テラコッタ（負け）カードにライブ換算を表示
新規店「＋新規」から店名を追加し Venue を作成

使用するデータモデル

Session   // gameTypeRaw（ring/tournament）
          // リング: chipIn / chipOut / sbChips / bbChips / durationMinutes(任意)
          //         computed chipProfit = chipOut - chipIn / bbProfit = chipProfit / bbChips
          // トナメ: buyInCount / resultRaw(busted/itm/won) / prizeNote
          // 共通: spentYen（娯楽費）/ venue / date
Venue     // name（店選択・前回値クエリの起点）

KPI イベント

イベントパラメータ目的
session_loggedgame_type, has_duration, venue_is_new錨行動の頻度（最重要リテンション指標）。金額・店名は含めない

実装メモ

  • 10秒入力が守るべき錨 — 入力ステップ数・タップ数を最小化し、前回値プリセットで実質「確認＋OUT入力」で完結させる
  • 収支入力はPaywallに阻まれない — このシートは永久無料。課金導線をこの画面のコア入力に挟まない
  • 店の「前回ステークス・前回チップIN」はその店の最新Sessionからクエリ導出（Venueに非正規化しない）
  • BB収支換算は bbChips > 0 のときのみ算出（0除算回避、bbProfitはOptional）
  • トナメの円換算・ROIは意図的に非実装。プライズは金銭換算せずテキストのみ（4+・賭博性の否定）
  • KPIのパラメータに金額・店名・自由記述を含めない（件数・真偽値・enum値のみ）
