答え合わせ

遷移元ハンドに一致する130問から出題・正誤と解説

設計確定・実装未着手 画面ID: PD-18

目的・役割

記録したハンドのポジション＋状況に一致する問題を130問（流用questions JSON）から出題し、3択回答→正誤＋解説で学習を促す。従（受動）機能であり、照合と合算で無料1日3回の制限を持つ。

UI 構成

要素種類説明
出題ヘッダLabel + Chip「CO ・ オープンに対して の問題（130問）」＋残り回数（例: 残り 2/3）
問題カードCard状況説明＋ハンド（ホールカード）
回答3択Button × 3フォールド／コール／レイズ
結果表示Card正解／不正解＋推奨アクション＋解説文
次の問題へPrimary Button残り回数を消費して次へ。0回で本画面ロック
ロック画面Lock View回数使い切り時。「Proで無制限に」でPaywall（source=matchLimit）

状態

状態表示内容
出題中（quiz）問題カード＋3択。残り回数を表示
結果表示（result）不正解の自分の選択はテラコッタ、正解は青／緑で区別。推奨アクション＋解説を表示
ロック（locked）回数使い切り。ロック画面→Paywall（source=matchLimit）誘導

遷移

操作遷移先
ハンド詳細／照合バナーの「似た問題を解く」から本画面へ流入
3択タップ結果表示（result）
「次の問題へ」タップ次問（残0でロック）
回数使い切り→「Proで無制限に」Paywall（source=matchLimit）

使用するデータモデル

Question     // 130問マスタ（流用）。positionRaw × scenarioRaw 一致で抽選
AnswerRecord // 回答履歴を保存: question / selectedActionRaw / isCorrect / answeredAt / durationSeconds
// 照合・答え合わせの合算回数は日付ベースで管理（無料1日3回）

KPI イベント

イベントパラメータ目的
drill_answeredcorrect答え合わせ利用と正答率

実装メモ

  • 出題は遷移元ハンドの positionRaw ＋ 状況に一致する問題を130問（流用questions JSON）からランダム抽選
  • 回答は AnswerRecord（流用モデル）に保存
  • 残り回数は照合と合算（無料1日3回）。0回時はロック画面→Paywall（source=matchLimit）
  • 結果表示は不正解=テラコッタ／正解=青（緑）で明確に区別
