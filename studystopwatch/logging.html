ロギング方針

Logger（os.log）を用いたログ出力ルール

print() は禁止。Logger（os.log）を使用する。

共通設定

private let logger = Logger(
    subsystem: Bundle.main.bundleIdentifier ?? "StudyStopwatch",
    category: "[クラス名]"
)

ログレベル

レベル用途
.info通常イベント（タイマー開始・停止、セッション保存）
.warning異常だが継続可（Live Activity orphaned、AppGroup nil）
.error機能継続不可（StoreKit verify失敗、SwiftData save失敗）

プライバシー対応

StoreKitトランザクションID、志望校名、メールアドレス等はログに出さない。 識別子（科目ID、セッションID）は privacy: .public。

Firebase Analytics 計測イベント一覧

AnalyticsManager.shared.log(_:) 経由で送信する。イベント名は Firebase Analytics の制約（40文字以内・英数とアンダースコアのみ）に準拠。 ユーザー入力値（科目名・点数・志望校名）はパラメータに含めない。

ロック計測イベントの移行運用（v1.2〜）: ロック表示UIを PremiumLockOverlay に共通化したことで、ロックの表示・タップ計測は 発火元付きの premium_lock_shown / premium_lock_tapped に一本化する方針。 ただし移行期間中は、既存の grade_lock_shown / grade_lock_tapped / weekly_report_lock_tapped と新イベントを併送する。 Firebase側で新イベントへの集計移行を確認後、旧イベントを撤去する予定。
