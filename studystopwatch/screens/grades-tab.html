レポートタブ・成績セグメント

レポートタブ内に上部セグメントコントロール（学習時間 / 成績）を追加し、成績セグメントでテスト結果を記録・可視化する（v1.1.0）。タブ構成は5タブのまま維持（ホーム / 今日 / カレンダー / レポート / 設定）

v1.1.0 画面ID: SS-GRADES-01 無料（v1.2.0で件数制限予定）

目的・役割

学習時間の記録だけでなくテストの点数・偏差値も一元管理することで、 「勉強した結果が成績に反映されているか」を可視化する。 受験生にとって最大のKPIである「点数の推移」をグラフで見せることで、 StudyStopwatch を学習管理の中核アプリとして定着させる。

タブ統合方針: タブ数をこれ以上増やすUX劣化を避けるため、成績機能は新規タブとして追加せず、 既存のレポートタブ内に Picker(.segmented)（学習時間 / 成績）を配置して切り替える方式を採用する。 タブ構成は「ホーム / 今日 / カレンダー / レポート / 設定」の5タブのまま維持する。

セグメント切り替え UI: レポートタブ上部の SegmentedPicker で ReportSection enum（.time / .grade）を切り替える。 学習時間セグメント選択時のみ共有ボタン（toolbar trailing）を表示する。 ナビゲーションタイトルは常に「レポート」のまま（成績セグメント表示中も変わらない）。

画面構成（成績セグメント内の画面ツリー）

  • SS-GRADES-01: 成績セグメント メインビュー（GradeReportView、グラフ＋履歴一覧）
  • SS-GRADES-02: テスト記録入力フォーム（TestRecordEditView、新規作成・編集共用・sheet表示）
  • SS-GRADES-03: テスト記録詳細画面（TestRecordDetailView、sheet表示。「編集」ボタンから SS-GRADES-02 へ遷移）

使用するデータモデル

// TestRecord（v1.1.0 新規）
@Model final class TestRecord {
    var id: UUID = UUID()
    var date: Date = Date()
    var testTypeRaw: String = TestType.regular.rawValue  // TestType.rawValue を保存
    var subject: Subject?                                 // Subject との任意リレーション
    var score: Double = 0.0
    var maxScore: Double = 100.0
    var deviation: Double? = nil   // 模試のみ使用
    var rank: Int? = nil           // 任意
    var memo: String? = nil        // 任意
    var createdAt: Date = Date()

    // Computed property
    var scoreRate: Double { maxScore > 0 ? score / maxScore * 100 : 0 }
    var testType: TestType { TestType(rawValue: testTypeRaw) ?? .other }
}

enum TestType: String, Codable, CaseIterable {
    case regular = "regular"  // 定期テスト
    case mock    = "mock"     // 模試
    case quiz    = "quiz"     // 小テスト
    case other   = "other"    // その他

    var displayName: String {
        switch self {
        case .regular: return "定期テスト"
        case .mock:    return "模試"
        case .quiz:    return "小テスト"
        case .other:   return "その他"
        }
    }
}

// Subject（既存）— TestRecord.subject のリレーション先
@Model final class Subject {
    // ...
    @Relationship(deleteRule: .nullify, inverse: \TestRecord.subject)
    var testRecords: [TestRecord] = []
}
課金線引き（v1.1.0）: 全件保存可（課金壁なし）。 v1.2.0 で「無料=直近10件、Premium=無制限」の制限を有効化予定。 v1.1.0 時点では件数チェックロジックを実装しないこと。

実装メモ

  • グラフは Swift Charts（import Charts）を使用。iOS 16+ 標準
  • 折れ線グラフ: LineMark。複数教科のデータを .foregroundStyle(by: .value("科目", subjectName)) で色分け。Y軸は得点率（%）
  • 教科フィルタの複数選択状態は @State private var selectedSubjectIDs: Set<UUID> で管理
  • テスト種別フィルタは @State private var selectedTestType: TestType?（nil=すべて）
  • TestRecord の SwiftData クエリは @Query(sort: \TestRecord.date, order: .reverse) を使用
  • 入力フォームの点数・満点は String バインドで受け取り、保存時に Double? にパース
  • SS-GRADES-02 / SS-GRADES-03 はいずれも .sheet で表示する（.navigationDestination は使わない）
  • ReportView 上部に Picker("", selection: $section) { ... }.pickerStyle(.segmented) を配置し、ReportSection enum（.time / .grade）で切り替える
  • 学習時間セグメント時のみ toolbar trailing に共有ボタンを表示（.grade セグメント時は非表示）
  • TabView の変更は不要（タブ構成は5タブのまま。.grades ケースは追加しない）
  • Subject 削除時に紐づく TestRecord は deleteRule: .nullify で subject を nil にする（記録自体は保持）
