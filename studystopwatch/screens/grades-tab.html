成績タブ

テスト結果を記録・可視化する新規タブ。既存3タブ構成に第4タブとして追加（v1.1.0）

v1.1.0 画面ID: SS-GRADES-01 無料（v1.2.0で件数制限予定）

目的・役割

学習時間の記録だけでなくテストの点数・偏差値も一元管理することで、 「勉強した結果が成績に反映されているか」を可視化する。 受験生にとって最大のKPIである「点数の推移」をグラフで見せることで、 StudyStopwatch を学習管理の中核アプリとして定着させる。

タブ構成変更: v1.0.x まで「タイマー / 履歴 / 設定」の3タブ構成を、 v1.1.0 から「タイマー / 履歴 / 成績 / 設定」の4タブ構成に変更する。

画面構成（成績タブ内の画面ツリー）

  • SS-GRADES-01: 成績タブ メイン画面（グラフ＋履歴一覧）
  • SS-GRADES-02: テスト記録入力フォーム（新規作成・編集共用）
  • SS-GRADES-03: テスト記録詳細画面（編集・削除）

SS-GRADES-02：テスト記録入力フォーム

UI 構成

要素種類説明
日付DatePicker（コンパクト）必須。デフォルト今日
テスト種別Picker（インライン）必須。定期テスト / 模試 / 小テスト / その他

満点TextField（数値キーボード）必須。Double。例: 100

メモTextEditor任意。自由記述。最大500文字

キャンセルボタンNavigationBar leading Button「キャンセル」入力破棄して戻る

項目ルールエラー文言
点数0以上・満点以下・空欄不可「点数は0以上・満点以下の数値を入力してください」

SS-GRADES-03：テスト記録詳細画面

UI 構成

要素種類説明
得点率大表示Text（大フォント・科目カラー）xx.x% を大きく表示
基本情報Form（読み取り専用）日付・テスト種別・教科・点数/満点・偏差値（模試のみ）・順位・メモ

削除確認アラート

  • タイトル: 「記録を削除しますか？」
  • メッセージ: 「この操作は取り消せません。」
  • ボタン: 「削除」（破壊的）/ 「キャンセル」

状態（メイン画面）

状態表示内容
初期表示（データ0件）グラフ空状態View。「＋」ボタンで入力フォームへ誘導
データあり・フィルタ未設定全件の推移グラフ＋全件リスト
データあり・フィルタ適用中フィルタ条件に合致するデータのみグラフ＋リスト表示
フィルタ適用後0件「該当するテスト記録がありません」の空状態View
保存中（非同期）保存ボタンに ProgressIndicator を表示し、多重タップを防止

遷移

操作遷移先
「＋」ボタンをタップSS-GRADES-02（新規作成モード、.navigationDestination）
履歴リストの行をタップSS-GRADES-03（詳細画面）
詳細画面「編集」ボタンをタップSS-GRADES-02（編集モード）
詳細画面「削除」確認ボタンをタップ削除後 SS-GRADES-01 へ戻る
入力フォーム「保存」ボタンをタップ保存後 SS-GRADES-01 へ戻る
入力フォーム「キャンセル」ボタンをタップSS-GRADES-01 へ戻る（入力内容は破棄）

使用するデータモデル

// TestRecord（v1.1.0 新規）
@Model final class TestRecord {
    var id: UUID = UUID()
    var date: Date = Date()
    var testTypeRaw: String = TestType.regular.rawValue  // TestType.rawValue を保存
    var subject: Subject?                                 // Subject との任意リレーション
    var score: Double = 0.0
    var maxScore: Double = 100.0
    var deviation: Double? = nil   // 模試のみ使用
    var rank: Int? = nil           // 任意
    var memo: String? = nil        // 任意
    var createdAt: Date = Date()

    // Computed property
    var scoreRate: Double { maxScore > 0 ? score / maxScore * 100 : 0 }
    var testType: TestType { TestType(rawValue: testTypeRaw) ?? .other }
}

enum TestType: String, Codable, CaseIterable {
    case regular = "regular"  // 定期テスト
    case mock    = "mock"     // 模試
    case quiz    = "quiz"     // 小テスト
    case other   = "other"    // その他

    var displayName: String {
        switch self {
        case .regular: return "定期テスト"
        case .mock:    return "模試"
        case .quiz:    return "小テスト"
        case .other:   return "その他"
        }
    }
}

// Subject（既存）— TestRecord.subject のリレーション先
@Model final class Subject {
    // ...
    @Relationship(deleteRule: .nullify, inverse: \TestRecord.subject)
    var testRecords: [TestRecord] = []
}
課金線引き（v1.1.0）: 全件保存可（課金壁なし）。 v1.2.0 で「無料=直近10件、Premium=無制限」の制限を有効化予定。 v1.1.0 時点では件数チェックロジックを実装しないこと。

実装メモ

  • グラフは Swift Charts（import Charts）を使用。iOS 16+ 標準
  • 折れ線グラフ: LineMark。複数教科のデータを .foregroundStyle(by: .value("科目", subjectName)) で色分け
  • 教科フィルタの複数選択状態は @State private var selectedSubjectIDs: Set<UUID> で管理
  • テスト種別フィルタは @State private var selectedTestType: TestType?（nil=すべて）
  • TestRecord の SwiftData クエリは @Query(sort: \TestRecord.date, order: .reverse) を使用
  • 入力フォームの点数・満点は String バインドで受け取り、保存時に Double? にパース
  • タブアイコンは chart.bar.xaxis（SF Symbols）
  • TabView の変更は ContentView の TabView に .grades ケースを追加
  • Subject 削除時に紐づく TestRecord は deleteRule: .nullify で subject を nil にする（記録自体は保持）
