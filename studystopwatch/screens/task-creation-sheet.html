タスク作成シート（TaskCreationSheet）

やること（StudyTask）の新規作成・編集モーダルシート

v1.0 画面ID: SS-TASK-01

目的・役割

単体の StudyTask を新規作成または編集するモーダルシート。 タイトルと期限日のみを入力する軽量フォーム。 カレンダー画面（日ビューの「📅 期限付き」ボタン）や Today 画面の「期限付きで追加」ボタンから呼び出される。 編集モード時はタスク行のタイトルタップから呼び出される。

起動モード

モードenum初期値呼び出し元
新規作成Mode.create(defaultDueDate:)タイトル空・期限日 = defaultDueDateカレンダー日ビュー・Today 画面
編集Mode.edit(StudyTask)既存タイトル・既存期限日タスク行タイトルタップ

UI 構成

UI/UX改善: タイトル空欄時に「よく使う例」プリセット候補を表示。 プリセットボタンは minHeight: 44pt で HIG 準拠のタップ領域を確保。
要素説明
タイトル入力 TextField「例: 数学 p.20〜30」プレースホルダー。複数行対応（lineLimit 1...3）。自動フォーカス（@FocusState titleFocused）
プリセット候補（改善）タイトルが空欄の場合のみ表示。「英単語を覚える」「数学ワークを進める」「復習ノートを見直す」の3候補。タップでタイトルに入力。各ボタンは minHeight: 44pt
期限日 DatePickerグラフィカル表示（.graphical スタイル）。選択した日付が dueDate に反映
「保存」ボタン（toolbar trailing）タイトルが空でないとき（canSave）のみ有効。タップで保存 → dismiss
「キャンセル」ボタン（toolbar leading）変更を破棄して dismiss

状態

状態表示内容
タイトル空欄プリセット候補を表示。「保存」ボタン無効
タイトル入力済みプリセット候補を非表示。「保存」ボタン有効化
編集モード初期既存タイトル・期限日が入力済み状態で表示。「保存」ボタン有効

遷移

操作遷移先
「保存」タップ（新規）StudyTask を modelContext に insert して dismiss
「保存」タップ（編集）既存 StudyTask を更新して dismiss
「キャンセル」タップ変更破棄して dismiss

使用するデータモデル

@Model final class StudyTask {
    var id: UUID
    var title: String
    var isCompleted: Bool
    var dueDate: Date?
    var sortOrder: Int
    var createdAt: Date
}

実装メモ

  • Mode enum の associated value で新規/編集を切り替え。init(mode:) で State の初期値を分岐
  • プリセット候補は let titlePresets = ["英単語を覚える", "数学ワークを進める", "復習ノートを見直す"]
  • 各プリセットボタンは .frame(minHeight: 44) を付与（HIG 準拠）
  • タイトルの先頭・末尾空白をトリムして空文字チェック（canSave）
  • onAppear で titleFocused = true を設定してキーボード自動表示
