フォーカス科目モード

ホーム画面で「今日の1科目」をピン留めし、選ぶ疲れをなくして1科目に集中させる画面

v1.1.0 画面ID: SS-FOCUS-01 無料

目的・役割

「今日どの科目を勉強するか毎回選ぶのが面倒・迷う」という受験生の課題を解決する。 ホーム画面で1科目をピン留めすることで、他の科目を折りたたみ、ピン留めした科目のタイマーを 大画面で即時起動できる。複数科目を管理しながら、1日の「メイン科目」に集中できる体験を提供する。 The One Thing / 1 Big Thing と同コンセプトだが、StudyStopwatch の既存タイマー機能と シームレスに統合されている点で差別化する。

UI 構成

ピン留めなし（通常状態）

要素種類説明
フォーカスモード説明バナーInfoBanner初回のみ表示。「1科目をピン留めして集中しよう」のガイド文言。✕ボタンで非表示
科目リストList / LazyVStack既存の科目カード一覧。各カードにピン留めボタン（📌アイコン）が追加される
ピン留めボタンButton（各科目カード右端）タップで当該科目をフォーカス科目に設定

ピン留めあり（フォーカス状態）

要素種類説明
フォーカス科目ヒーローカードVStack（大カード）科目名・アイコン・カラーを大きく表示。本日の累計学習時間を表示
計測ボタンButton（大型・主要CTA）タイマー未起動時は「計測開始」、起動中は「計測中（停止）」。既存タイマー機能を呼び出す
今日の目標進捗バーProgressView日次目標（分）に対する当日学習時間の割合。目標未設定時は非表示
ピン留め解除ボタンButton（テキストスタイル）「ピン留めを解除」。タップで通常状態に戻る
科目変更ボタンButton（テキストスタイル）「科目を変更」。タップで科目選択シートを表示
折りたたまれた他科目DisclosureGroup / 折りたたみセクション「他の科目を表示 (n)」として折りたたみ表示。タップで展開し全科目を表示（ピン留めボタン付き）

科目選択シート（フォーカス科目変更時）

要素種類説明
シートタイトルNavigationTitle「フォーカス科目を選択」
科目リストList全科目を一覧表示。現在のフォーカス科目にはチェックマーク
キャンセルボタンButtonシートを閉じる

状態

状態表示内容
フォーカスモードOFF（設定で無効化）通常のホーム画面。ピン留めボタンは非表示。フォーカスモード機能自体が無効
フォーカスモードON・ピン留めなし全科目カードにピン留めボタンを表示。初回はガイドバナーを表示
フォーカスモードON・ピン留めあり・タイマー未起動ヒーローカード＋「計測開始」ボタン＋折りたたまれた他科目
フォーカスモードON・ピン留めあり・タイマー起動中（ピン留め科目）ヒーローカードにタイマー経過時間をリアルタイム表示。ボタンは「停止」表示
フォーカスモードON・ピン留めあり・タイマー起動中（別科目）ヒーローカードに「別の科目で計測中」の注記を表示。「切り替えて計測」ボタンを表示
科目0件「科目を追加してください」の空状態View。フォーカスモードは非表示

遷移

操作遷移先 / 動作
ピン留めボタンをタップフォーカス科目を設定。同画面内でヒーローカード表示に切り替わる（アニメーション付き）
「計測開始」ボタンをタップピン留め科目のタイマーを起動（既存タイマー機能と同一処理）。ボタン表示が「計測中（停止）」に変わる
「停止」ボタンをタップタイマーを停止しセッション保存（既存フロー）
「ピン留めを解除」をタップフォーカス科目をクリア。通常のホーム表示に戻る（アニメーション付き）
「科目を変更」をタップ科目選択シート（.sheet）を表示
「他の科目を表示」をタップ折りたたみを展開し全科目を表示
折りたたみ内の科目カードをタップ通常のタイマー画面遷移（既存フロー）
設定 → フォーカスモードOFFピン留め状態は保持したまま機能を無効化。再ONで即時復元

使用するデータモデル

// Subject（既存）— ピン留め対象
@Model final class Subject {
    var id: UUID = UUID()
    var name: String = ""
    var colorHex: String = "#007AFF"
    var iconName: String = "book"
    // ...
}

// UserSettings（既存）— フォーカス科目IDと機能ON/OFFを保持
@Model final class UserSettings {
    // v1.1.0 追加
    var focusSubjectID: UUID? = nil          // ピン留めされた科目のID
    var focusSubjectModeEnabled: Bool = true  // フォーカスモードのON/OFF
}
UserSettings への新規プロパティ追加に伴い、SwiftData マイグレーション対応が必要。 focusSubjectID・focusSubjectModeEnabled にはデフォルト値を設定済み。

実装メモ

  • ピン留め科目IDは UserSettings.focusSubjectID に UUID? として保持。nil = ピン留めなし
  • ヒーローカードから計測開始は HomeViewModel.startTimer(subject:) の既存メソッドを呼び出す（新規ロジック不要）
  • ピン留め科目が削除された場合は自動的に focusSubjectID = nil にリセット（Subject 削除フックで処理）
  • 折りたたみは @State private var isOtherSubjectsExpanded: Bool = false でローカル管理
  • ガイドバナー非表示フラグは UserDefaults（AppGroup不要）に保存
  • フォーカス状態のアニメーションは .animation(.spring(), value: focusSubjectID) で実装
  • 設定画面の「フォーカスモード」トグルは UserSettings.focusSubjectModeEnabled にバインド
  • 「切り替えて計測」タップ時は既存タイマー停止→ピン留め科目で再起動の2ステップを自動処理
