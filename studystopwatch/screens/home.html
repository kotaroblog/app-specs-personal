ホーム画面（HomeView）

タイマー実行の中心画面

v1.0画面ID: SS-HOME-01

目的・役割

科目を選んでタイマーを開始・停止する画面。タイマー実行中はLive Activityとも連動。

UI 構成（v1.2.2）

v1.2.2 でカード順序を再設計。「大型開始CTA → 今日目標リング（主表示）→ 今週・今月（補助）→ ポモドーロ → 文脈カード」に整理。 科目一覧の各行を主導線（計測ボタン）と副導線（…メニュー）に整理し視認負荷を低減。
順序要素説明
1大型計測開始CTA（startCTACard）重要カードスタイル（太い枠線＋底面シャドウ）。前回続き（直近セッションの科目）がある場合は「前回の続き: [科目名]」、フォーカス科目モード時は「今日の1科目: [科目名]」を表示。大きな「計測スタート」ボタン＋弱い「計測せず手動で記録」テキストリンク
2今日目標カード（todayGoalCard）重要カードスタイル。大リングで当日達成率を主表示。基準は effectiveDailyGoalMinutes（日次StudyGoal優先→設定の1日の目標にフォールバック）。計測中リング・達成演出・ホーム表示すべて同一基準を使用
3今週・今月カード（2枚）通常カードスタイル（影なし・軽い境界線のみ）。compact 表示で横並び
4ポモドーロ設定トグル集中25分/休憩5分の自動切替。通常カードスタイル
5文脈カード群（contextCards）連続日数・試験カウントダウン・モチベーション。今日進捗ブロックは撤去（今日目標カードへ集約済み）。試験日未設定時は「カレンダータブから試験日を登録」リンク表示
—科目一覧（subjectList）主導線は「計測」ボタン。「手動で記録」・「編集」・「削除」は各行末の「…」メニューへ集約。科目未登録時は ContentUnavailableView「設定タブから科目を追加してください」
—タイマー実行中表示（timerRunningView）タイマー実行中のみ全面切替。HH:MM:SS 大字 + 一時停止/停止ボタン（スタイルはタイマー機能仕様参照）
—目標達成オーバーレイ日次目標達成時に「目標達成！」フルスクリーンオーバーレイ（タップで閉じる）
—ツールバー（右上）共有ボタン（accessibilityLabel: "学習記録を共有"）

ビジュアル階層方針（v1.2.2）

「重要カードスタイル」（太い枠線＋底面シャドウ）は主役カードにのみ適用し、画面全体の情報階層を明確化。 通常カードは軽い境界線のみ（影なし）。Dynamic Type 上限の不必要な制限は設けない。
スタイル適用対象
重要カードスタイル開始CTA・今日目標カード
通常カードスタイル今週・今月・ポモドーロ・文脈カード群

状態

状態表示
停止中（科目あり）startCTACard + todayGoalCard + 今週/今月 compact + ポモドーロ + contextCards + subjectList
停止中（科目なし）ContentUnavailableView「科目が登録されていません」
実行中timerRunningView に全面切替（経過時間大字 + 完了ボタン（主）+ 一時停止/再開（弱））
一時停止中timerRunningView 内に再開ボタン
ポモドーロ集中中「集中タイム N分」表示
ポモドーロ休憩中「休憩タイム N分」表示
フォーカス科目モード ONstartCTACard に「今日の1科目: [科目名]」を表示。ピン留め科目カードを上部強調。他科目は折りたたみ（showOtherSubjects トグルで展開）
学習日変更検出scenePhase.active 復帰時に todayKey を更新し @Query を再構築
試験日未設定contextCards に「カレンダータブから試験日を登録」リンク表示

遷移

操作先

「計測せず手動で記録」リンク手動記録シート（sheet）

使用するデータモデル

StudySession（保存対象）+ Subject（科目選択）
TimerViewModel { isRunning, isPaused, elapsedSeconds, currentSubject, isPomodoroMode, pomodoroPhase }
effectiveDailyGoalMinutes: StudyGoal（日次）?.targetMinutes ?? settings.dailyGoalMinutes

実装メモ

  • UserDefaults 経由でタイマー永続化（24時間境界）
  • scenePhase 監視でバックグラウンド時の経過時間補正 / 学習日変更時に todayKey を更新して @Query を再構築
  • Live Activity と連動（startActivity / updateActivity / endActivity）
  • フォーカス科目モード: settings?.focusSubjectModeEnabled + settings?.focusSubjectID で制御。FocusSubjectPickerSheet で変更
  • dashboardCard は todaySessions / weekSessions / monthSessions を3つの独立 @Query で取得（パフォーマンス改善）
  • goalsRow は activeGoals（isClosedでフィルタ）から日/週/月の StudyGoal を取得
  • 共有ボタンに .accessibilityLabel("学習記録を共有") を付与（HIG準拠）
  • 完了セッション数 completedSessionCount（AppStorage）をインクリメントし、閾値でレビューリクエスト（requestReview）
  • effectiveDailyGoalMinutes: 日次StudyGoal優先→設定の dailyGoalMinutes にフォールバック（計測中リング・達成演出・カード表示で統一）
  • 科目一覧の「…」メニューに「手動で記録」・「編集」・「削除」を集約（主ボタンを「計測」のみに絞る）
