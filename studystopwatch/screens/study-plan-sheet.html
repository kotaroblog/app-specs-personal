学習計画シート（StudyPlanSheet）

タスクを日付に自動配分する学習計画生成シート

v1.0 画面ID: SS-PLAN-01

目的・役割

教材タイトル・総量・終了日を入力すると「1日あたり何ページ/問」を計算し、 StudyTask として自動配分・一括生成するモーダルシート。 カレンダー画面のツールバー「計画」ボタンから呼び出される。

UI 構成

UI/UX改善: 入力ガイド（プレースホルダー・サブテキスト）と単位プリセット候補を追加し、 初回利用時の迷いを軽減。
要素説明
科目セレクター登録済み Subject から選択（任意）。未選択時は「未分類」タスク生成
タイトル入力計画タイトル（例: 数学ワーク Chapter3）。キーボード自動フォーカス（@FocusState titleFocused）
総量入力 + 単位プリセット（改善）総量を数値入力。単位は「ページ」「問」「回」「章」「時間」のプリセットボタンで即選択。「カスタム」選択時はフリー入力欄を表示
入力ガイド（改善）各フィールドにプレースホルダーとサブテキスト（例: 「例: 数学ワーク」「例: 50」）を追加
開始日・終了日 DatePicker開始日（デフォルト今日）と終了日（デフォルト試験日 or 30日後）の DatePicker
分割方法セグメント毎日 / 曜日を選ぶ / 1タスク の3種類（SplitMode）
曜日選択（曜日モード時）日〜土の曜日チェックボックス（includedWeekdays: Set<Int>）
プレビューサマリ「N日間 / 1日あたり N ページ」を計算してリアルタイム表示
生成ボタン「学習計画を作る」。canGenerate（タイトル+総量+日付が有効）のときのみ有効

状態

状態表示内容
初期表示各フィールド空欄。終了日 = 試験日（未設定なら今日+30日）。プレビューは「- 日間」
タイトル・総量入力済みプレビューサマリが更新される
生成可能（canGenerate = true）「学習計画を作る」ボタンが有効化
生成完了dismiss() でシートを閉じる。カレンダー画面にタスクが反映される
対象日なし（曜日絞りすぎ等）「対象日が0日です。曜日を確認してください」警告テキストを表示

遷移

操作遷移先
「学習計画を作る」タップ（生成完了）シートを閉じる（CalendarScreen に戻る）
「キャンセル」タップシートを閉じる（変更破棄）

使用するデータモデル

// 生成されるタスク
@Model final class StudyTask {
    var id: UUID
    var title: String
    var isCompleted: Bool
    var dueDate: Date?     // targetDays の各日付が設定される
    var subject: Subject?
    var sortOrder: Int
    var createdAt: Date
}

// 入力参照
@Model final class UserSettings {
    var examDate: Date?   // 終了日のデフォルト値として参照
}

@Model final class Subject {
    var id: UUID
    var name: String
    var colorHex: String
}

実装メモ

  • 分割ロジック: targetDays 計算プロパティ。splitMode == .single のときは [endDate] のみ返す
  • 総量 ÷ 日数の切り上げ配分（端数は最終日タスクに吸収）
  • 単位プリセット: ["ページ", "問", "回", "章", "時間"]。カスタム選択時は useCustomUnit = true
  • 生成した StudyTask は modelContext.insert で一括登録後 try modelContext.save()
  • 試験日（settings?.examDate）が設定済みの場合、終了日の初期値として使用（init() では SwiftData 未ロードのため nil を渡し、onAppear で上書き）
  • titleFocused = true を onAppear で設定してキーボードを自動表示
