オンボーディング（OnboardingView）

初回起動時のセットアップウィザード

v1.0画面ID: SS-ONB-01

必須フロー（最大4画面）

v1.2.2 でフローを大幅圧縮。無料利用開始を既定とし、オンボーディング内にペイウォールを置かない。 ページ名は Firebase Analytics で計測（OnboardingPage.analyticsName）。 互換性維持のため OnboardingPage enum の定義・Analyticsイベント（onboarding_page_view / onboarding_complete）・UserSettings/Subject 保存ロジックは維持（moved-out ページの構造体も残置）。
  1. welcome — アプリ紹介（歓迎メッセージ）
  2. goalSelection — 目標タイプ選択: 定期テスト / 高校受験 / 大学受験
  3. subjectSelection — 科目選択（プリセット科目から複数選択）
  4. examDate — 試験日設定（必須フローの最終ページ）
    「この日で始める」→ examDate を保存して完了 / 「試験日は未定（あとで設定）」→ examDate 未保存のまま完了

必須フローから除外した項目と移設先

以下はオンボーディング必須フローから外し、ホーム遷移後の任意導線に移設。
旧ステップ移設先
motivation（動機入力）設定タブ → ユーザー設定
scoreGoals（科目別点数目標）科目編集画面（SubjectEditView）→ 点数目標セクション
schoolName（志望校名）設定タブ → ユーザー設定
studyHours（1日の目標時間）設定タブ → 1日の目標設定
planCreation / planResult（学習計画）カレンダータブ → 学習計画シート
commit（誓約確認）廃止（ホーム到達で自然に開始）
notification（通知許可）設定タブ → 通知設定 or ホーム初回表示タイミング
paywall（課金）設定タブ / 各種Premiumロック機能からの導線

完了後の遷移

onboardingCompleted = true → ContentView（試験日設定あり・なし問わず同一パス）

試験日未設定時の補完導線

ホーム画面の文脈カードに「試験日をカレンダータブから登録」リンクを表示。
