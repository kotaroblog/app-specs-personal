成績記録機能

テスト結果（点数・偏差値等）を記録・可視化し、学習時間と成績の両軸で受験勉強を管理する機能

v1.1.0 機能ID: SS-FEAT-GRADE-01 無料（v1.2.0で件数制限予定）

概要

StudyStopwatch は学習時間の計測を主軸とするが、受験生にとって最終的なKPIは 「テストで何点取れたか」である。成績記録機能を追加することで、 「勉強した時間」と「その結果としての点数」を一つのアプリで管理できるようにする。

ユーザー価値: 学習ログと成績ログの一元管理。「毎日2時間勉強したのに点数が上がらない」 という課題の可視化→勉強方法の改善につながる。
ビジネス価値: アプリの利用シーン拡大（テスト前後の記録）によるリテンション向上。 v1.2.0 の Premium 機能への導線（件数制限解除）。

関連画面

  • 成績セグメント メインビュー（SS-GRADES-01）
  • テスト記録入力フォーム・sheet（SS-GRADES-02）
  • テスト記録詳細・sheet（SS-GRADES-03）

テスト記録 新規作成フロー

テスト記録 削除フロー

sequenceDiagram participant U as User participant V as GradeDetailView participant VM as GradesViewModel participant DB as SwiftData U->>V: 「記録を削除」ボタンをタップ V->>U: 確認アラートを表示 U->>V: 「削除」を選択 V->>VM: deleteRecord(id) VM->>DB: 該当 TestRecord を delete DB-->>VM: 削除完了 VM-->>V: dismiss（詳細画面の sheet を閉じる） V-->>U: 成績セグメントに戻る。リストから削除される

グラフ描画フロー

sequenceDiagram participant V as GradeReportView participant VM as ReportViewModel participant DB as SwiftData V->>VM: onAppear / フィルタ変更 / セグメント切り替え VM->>DB: @Query でフィルタ条件に合う TestRecord を取得（日付昇順） DB-->>VM: [TestRecord] VM->>VM: 教科ごとにグルーピング。scoreRate(%) を算出 VM-->>V: chartData: [SubjectChartData] を更新 V->>V: Swift Charts で LineMark を描画

ビジネスルール

  • 同一日・同一教科・同一テスト種別のデータを複数件記録可能（上書きではなく追記）
  • 偏差値入力欄はテスト種別が「模試」の場合のみ表示する。他の種別では入力できない
  • 点数は整数でも小数でも入力可（Double で保存）。例: 85 / 85.5 両方OK
  • 教科との紐付けはリレーション（Subject?）で管理する。科目削除時は記録自体は残し、subjectをnilにする（nullify）
  • 画面構成変更: 成績機能は新規タブではなく、レポートタブ内に上部 SegmentedPicker（学習時間 / 成績）を追加して統合する（5タブ構成は維持）
  • v1.1.0時点の課金ルール: 件数制限なし。全件保存・表示可能
  • v1.2.0以降の課金ルール（Out of Scope）: 無料=直近10件のみ表示。11件目以降はボカし表示＋Premium誘導バナー。記録自体は保存される
  • グラフのY軸は得点率（%）に統一する。点数・満点が科目によって異なるため、得点率で比較可能にする
  • グラフに表示するデータはフィルタ条件（教科・テスト種別）に連動する
  • グラフのデフォルトフィルタ: 「定期テスト」に設定（旧: すべて混在）
  • 履歴一覧のデフォルトフィルタ: 「定期テスト」に設定
  • 得点率カラー: 80%以上=緑 / 60〜79%=オレンジ / 60%未満=赤
  • まとめて登録: 複数科目のテスト結果を一括で保存できる。有効な行（教科選択済み + 点数入力済み）のみ insert する
  • まとめて登録の「テスト名」は TestRecord の memo フィールドに格納する
  • まとめて登録で模試を選択した場合、各科目行に偏差値・順位の入力欄が追加表示される

データモデル

@Model final class TestRecord {
    var id: UUID = UUID()
    var date: Date = Date()
    var testTypeRaw: String = TestType.regular.rawValue
    var subject: Subject? = nil
    var score: Double = 0.0
    var maxScore: Double = 100.0
    var deviation: Double? = nil
    var rank: Int? = nil
    var memo: String? = nil
    var createdAt: Date = Date()

    var scoreRate: Double { maxScore > 0 ? score / maxScore * 100 : 0 }
    var testType: TestType { TestType(rawValue: testTypeRaw) ?? .other }
}

enum TestType: String, Codable, CaseIterable {
    case regular = "regular"  // 定期テスト
    case mock    = "mock"     // 模試
    case quiz    = "quiz"     // 小テスト
    case other   = "other"    // その他

    var displayName: String {
        switch self {
        case .regular: return "定期テスト"
        case .mock:    return "模試"
        case .quiz:    return "小テスト"
        case .other:   return "その他"
        }
    }
}
マイグレーション注意: TestRecord は v1.1.0 での新規追加モデル。 既存 Subject モデルに testRecords: [TestRecord] のリレーションを追加するため、 Subject モデルのマイグレーション対応が必要。デフォルト値 = [] を設定すること。

実装メモ

  • 成績機能は GradeReportView として実装し、既存 ReportView 内に SegmentedPicker で切り替えて埋め込む
  • SwiftData クエリ: @Query(sort: \TestRecord.date, order: .reverse) var records: [TestRecord]
  • グラフ用データ変換: Dictionary(grouping: filteredRecords, by: \.subject?.name ?? "不明") でグルーピング
  • フォームの点数・満点入力は @State private var scoreText: String = "" で文字列管理し、Double(scoreText) でパース
  • テスト種別変更時に偏差値フィールドを .transition(.opacity) で表示/非表示アニメーション
  • SS-GRADES-02 / SS-GRADES-03 はいずれも .sheet で表示する。.navigationDestination は使わない
  • TabView のタブ追加・AppTab enum の変更は不要（5タブ構成を維持）
  • v1.2.0 課金ロジックのための件数チェックは v1.1.0 では実装しない（コードに TODO コメントを残す）
  • Subject 削除時の deleteRule: .nullify により TestRecord.subject が nil になるため、グラフ描画時に nil subject のデータは「科目不明」として扱う
  • グラフのデフォルトフィルタ状態: @State var selectedTestType: TestType = .regular
  • 得点率カラー: scoreRate >= 80 ? .green : scoreRate >= 60 ? .orange : .red
  • BatchTestEntrySheet の行モデルは @Observable class BatchEntryRow で管理し、各行の得点率をリアルタイム算出
  • TestRecordDetailView: テスト種別ラベルの表示条件から偏差値フィールドの存在チェックを削除し、常時表示に変更
  • グラフに種別フィルタ Menu を追加: Menu("フィルタ") { ForEach(TestType.allCases) { ... } }

Out of Scope（v1.3.0 以降）

  • 学習時間×成績の相関分析（StudySession と TestRecord の日付照合）
  • 志望校判定（A〜E判定）
  • CSVエクスポート（TestRecord 全件をCSV出力）
  • 目標スコア逆算・ギャップ表示（目標点数に対して現状何点足りないか）
