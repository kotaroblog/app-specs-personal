広告制御

インタースティシャル・バナー広告の表示条件・クールダウン・上限管理

v1.0 機能ID: SS-FEAT-AD

概要

AdMob を使ったバナー広告とインタースティシャル広告を管理する機能。 プレミアム会員・動画ボーナス適用中のユーザーには広告を非表示にする。 インタースティシャルはセッション時間・クールダウン・1日上限・セッション回数の4条件すべてを満たした場合のみ表示。

関連画面

  • ホーム（タイマー） — セッション完了後にインタースティシャルを表示
  • プレミアム — 広告非表示の特典訴求

広告種別

種別配置表示条件
バナー広告（BannerAdView）ContentView 下部（全タブ共通）非プレミアム・動画ボーナス未適用時
インタースティシャル広告セッション完了直後下記4条件すべて満たす場合
リワード動画広告プレミアム画面 / 動画ボーナスボタン非プレミアム・1日1回まで（v1.0.5追加）

インタースティシャル表示条件（4条件AND）

条件閾値備考
プレミアム除外isPremium == false動画ボーナス適用中も非表示
最低セッション時間5分以上短時間セッションでの広告UX低下を防ぐ
クールダウン前回表示から5分以上経過lastShownDate を UserDefaults に保存
1日表示上限5回以下 / 日dailyCount を UserDefaults に保存（日付変わりでリセット）
セッション回数3回ごとsessionCountSinceLastAd カウンター管理

InterstitialAdManager API

// Views/Components/InterstitialAdManager.swift
final class InterstitialAdManager: NSObject, ObservableObject {
    @Published var isAdReady: Bool = false

    func loadAd()
    func showAdIfEligible(
        isPremium: Bool,
        videoBonusActive: Bool,
        sessionDurationSeconds: Int,
        onComplete: @escaping () -> Void
    )

    // 内部メソッド
    private func canShow(sessionDurationSeconds: Int) -> Bool
    private func recordShown()
    private func resetDailyCountIfNeeded()
}

処理フロー（インタースティシャル）

sequenceDiagram participant U as User participant HV as HomeView participant AM as InterstitialAdManager participant AD as AdMob SDK U->>HV: セッション完了（STOP） HV->>AM: showAdIfEligible(isPremium, sessionDuration) AM->>AM: canShow() で4条件チェック alt 表示可 AM->>AD: present(from:) AD-->>U: 広告表示 AM->>AM: recordShown()（カウンタ・日時更新） AD-->>HV: dismissCompletion else 表示不可 AM-->>HV: onComplete() を即呼び出し end

ビジネスルール

  • DEBUG ビルド時は #if DEBUG でGoogleテスト用Ad Unit IDに自動切替
  • 動画ボーナス適用中（videoBonusExpiry が未来）はバナー・インタースティシャルとも非表示
  • 1日のカウントリセットはアプリ起動時に日付比較で判定
  • リワード動画は1日1回のみ（rewardAdWatchedDate を UserDefaults に保存）

エラー処理

発生条件対応
広告ロード失敗isAdReady = false のまま。次回起動時に再ロード
表示中にアプリがバックグラウンド移行AdMob SDK が自動処理。onComplete は dismissCompletion 内で確実に呼ぶ
リワード動画のコールバック未到達タイムアウト（30秒）後に動画ボーナスを付与しない

実装メモ

  • InterstitialAdManager は NSObject + GADFullScreenContentDelegate 継承
  • dailyCount リセット: アプリ起動時に lastResetDate と今日の日付を比較
  • 本番Ad Unit IDのInfo.plist設定（リリース前に必ず実施）
  • リワード広告（GADRewardedAd）実装はv1.0.5で追加
