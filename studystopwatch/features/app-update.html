アプリ更新チェック機能

App Store JSON APIで新版検知

v1.0機能ID: SS-FEAT-UPDATE

概要

App Storeの公開JSON APIから最新バージョンを取得し、現在のアプリバージョンと比較。新版があればアラートでApp Storeへの誘導。

AppUpdateService API

final class AppUpdateService {
    func checkForUpdate() async throws -> UpdateInfo?
}

struct UpdateInfo {
    let latestVersion: String
    let releaseNotes: String
    let appStoreURL: URL
}

処理フロー

  1. App Store JSON API へリクエスト（itunes.apple.com/lookup?bundleId=...）
  2. レスポンスから latestVersion 取得
  3. 現在の CFBundleShortVersionString と比較
  4. 新しければ Alert 表示 + App Storeへのリンク

呼び出しタイミング

  • ContentView の .task で起動時に1回
  • 1日1回まで（UserDefaults に lastChecked を保存）

エラー処理

  • 通信失敗 → サイレント（次回再試行）
  • パース失敗 → サイレント（Logger.error）

App Store レビュー依頼仕様

@Environment(\.requestReview)（StoreKit 2）を使用してレビュー依頼ダイアログを表示する。
iOS の OS 側制限により、1 年間に最大 3 回しかダイアログは表示されない。

発火条件（AND 条件）— 目標設定ユーザー向け

#条件備考
1今日の学習目標を初めて達成した瞬間目標設定済みユーザーのみ対象
2セッション完了回数が 10 回以上累計カウント
3前回のレビュー依頼から 30 日以上経過UserDefaults に lastReviewRequestDate を保存して管理

フォールバック条件 — 目標未設定ユーザー向け

目標が設定されていないユーザーはマイルストーン達成時に発火する。条件 2（10回以上）と条件 3（30日経過）を同時に満たす場合のみ発火。

マイルストーン（セッション完了回数）備考
20 回—
50 回—
100 回—

実装メモ

  • SwiftUI @Environment(\.requestReview) private var requestReview を使用
  • OS 側の表示回数制限（年間最大 3 回）により、ダイアログが必ず表示されるとは限らない
  • UserDefaults["lastReviewRequestDate"] に最終依頼日時を保存し、30 日ガードを実装する
  • 発火判定は SessionCompleteViewModel または GoalTrackerService 内で行う
