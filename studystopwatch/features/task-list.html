やることリスト機能

期限日（dueDate）ベースで学習タスクを管理し、カレンダーと連携して計画・実績を一体管理する機能

v1.3.0 機能ID: SS-FEAT-TASK-01

概要

ユーザーが学習タスクに「期限日（dueDate）」を設定し、カレンダー画面の日ビューで期限日ごとに管理できる機能。 v1.2.0 で旧「期間バケツ（day / week / month）モデル」から C案: 期限日モデル に移行。 受験生の実態（「○日までに○ページ終わらせる」）に合わせた設計。 タスクは StudyTask モデルで管理され、カレンダー日ビューでは選択日を期限とするタスクを一覧表示する。 学習計画自動配分（StudyPlanSheet）により、長期課題を日単位タスクへ自動分割できる。

設計変更の背景（v1.1 → v1.2）

  • 旧設計: タスクを「今日用・今週用・今月用」の期間バケツで分類（periodRawValue = day/week/month）
  • 新設計: タスクに dueDate: Date? を設定。カレンダーの選択日とマッチするタスクを表示
  • 移行理由: 受験生は「火曜日までに数学ワーク50ページ」のように期限を意識して計画を立てる。期間バケツでは複数日にまたがる計画が表現できなかった

関連画面

  • カレンダー画面（CalendarScreen） — タスクの作成・編集・削除・完了操作の主要 UI（日ビュー・週ビュー・月ビュー）
  • 目標・やることリスト画面（GoalEditSheet） — 目標設定と完了率表示
  • ホーム画面 — CompactGoalCard でタスク件数（N/M）を表示

データモデル（StudyTask）

@Model final class StudyTask {
    var id: UUID = UUID()
    var title: String = ""
    var isCompleted: Bool = false
    var dueDate: Date? = nil              // 期限日（v1.2〜。nil の場合は旧タスクまたは期限なし）
    var subject: Subject? = nil           // 紐づく科目（v1.3〜。学習計画生成時は必須）
    var linkedSessionID: UUID? = nil      // 「記録して完了」で紐づいた StudySession.id
    var completedAt: Date? = nil          // 完了タイムスタンプ
    var sortOrder: Int = 0
    var createdAt: Date = Date()

    // 後方互換フィールド（v1.2 以降は非推奨・移行後は参照しない）
    var periodRawValue: String = "day"    // 旧: "day" / "week" / "month"
    var periodStart: Date = Date()        // 旧: 期間開始
    var periodEnd: Date = Date()          // 旧: 期間終了
}

linkedSessionIDUUID?記録して完了で紐づいたセッションID
completedAtDate?完了タイムスタンプ

periodEndDate後方互換フィールド（v1.2以降は非推奨）

Phase 3: 学習計画自動配分（StudyPlanSheet）

カレンダー画面右上の「📅」ボタンからアクセス。長期課題を日単位タスクへ自動分割する。

入力項目説明
タイトル例: 数学ワーク
科目（必須）v1.3〜: 科目選択を必須化。学習計画から生成されるタスクには必ず科目が紐づく。未選択時は保存ボタンを非活性にする
合計 + 単位例: 100 ページ（ページ / 問 / 回 / 章 / 時間 / 自由入力）
済み（任意）進捗差し引き（例: すでに20ページ済なら20を入力）
開始日デフォルト: 今日
試験日UserSettings から自動取得。変更可
土日を除くトグル（ON で土日をスキップして配分）
  • プレビュー: 「1日あたり Xページ × Y日間」をリアルタイム表示
  • 保存: 毎日1タスク（dueDate・subject 付き）を一括生成
  • タイトル形式: 「数学ワーク: 5ページ（3日目/20日）」

共通コンポーネント: SubjectProgressRow

GoalEditSheet および週ビューの「今週の進捗」セクションで再利用するコンポーネント。

表示要素説明

科目名Subject.name

処理フロー

クイック追加フロー

sequenceDiagram participant U as ユーザー participant V as DayCalendarView participant VM as CalendarViewModel participant DB as SwiftData U->>V: テキスト入力 → 「追加」ボタンをタップ V->>VM: addTask(title, dueDate: selectedDate) VM->>DB: modelContext.insert(StudyTask) DB-->>VM: 保存完了 VM-->>V: @Query 経由でリスト更新 V-->>U: 追加したタスクが当日の一覧に表示

期限付き追加フロー（TaskCreationSheet）

sequenceDiagram participant U as ユーザー participant V as DayCalendarView participant S as TaskCreationSheet participant VM as CalendarViewModel participant DB as SwiftData U->>V: 「📅 期限付き」ボタンをタップ V->>S: .sheet で TaskCreationSheet を表示 U->>S: タイトル入力 + DatePickerで期限日を選択 U->>S: 「追加」ボタンをタップ S->>VM: addTask(title, dueDate: pickedDate) VM->>DB: modelContext.insert(StudyTask) DB-->>VM: 保存完了 VM-->>V: @Query 経由でリスト更新 V-->>U: sheet が閉じ、タスクが追加される

「記録して完了」フロー

sequenceDiagram participant U as ユーザー participant V as DayCalendarView participant MES as ManualStudyEntrySheet participant VM as CalendarViewModel participant DB as SwiftData U->>V: 「📝 記録して完了」ボタンをタップ V->>MES: .sheet で表示（initialNote, onSaved クロージャを渡す） U->>MES: 科目・時間・メモを入力して保存 MES->>DB: modelContext.insert(StudySession) MES->>VM: onSaved(session.id) コールバック呼び出し VM->>DB: task.isCompleted = true, task.completedAt = now, task.linkedSessionID = session.id DB-->>VM: 保存完了 VM-->>V: @Query 経由でリスト更新（タスクが完了状態に） V-->>U: チェックマーク付きで完了表示

学習計画自動配分フロー（StudyPlanSheet）

sequenceDiagram participant U as ユーザー participant S as StudyPlanSheet participant VM as PlanViewModel participant DB as SwiftData U->>S: 「📅」ボタンからシートを開く U->>S: タイトル・合計・単位・開始日・試験日・土日除くを入力 S->>VM: previewPlan(inputs) VM-->>S: 「1日あたり X単位 × Y日間」をリアルタイム表示 U->>S: 「保存」ボタンをタップ S->>VM: savePlan() VM->>DB: StudyTask を日数分 insert（各タスクに dueDate・title 形式「数学ワーク: 5ページ（N日目/Y日）」） DB-->>VM: 保存完了 VM-->>S: dismiss（シートを閉じる） S-->>U: カレンダーに日単位タスクが反映される

データ移行（v1.1 → v1.2）

StudyTaskMigrationService: 初回起動時に旧タスクの periodStart → dueDate を自動変換（一度だけ実行）。
  • 移行条件: dueDate == nil かつ periodStart != Date() のタスク
  • 変換: task.dueDate = Calendar.current.startOfDay(for: task.periodStart)
  • 移行完了後は UserDefaults にフラグを保存して再実行を防ぐ
  • 旧フィールド（periodRawValue / periodStart / periodEnd）は削除せず後方互換フィールドとして残す

ビジネスルール

  • 期限日が同じタスクは dueDate == selectedDate のクエリで取得する。時刻は無視し日付のみで比較
  • dueDate = nil のタスクはどの日にも表示しない（「未整理タスク」として将来のセクションで対応予定）
  • 「記録して完了」でキャンセルした場合、タスクは未完了のまま維持する
  • 完了済みタスクの再オープン（チェック解除）は可能。isCompleted = false、completedAt = nil、linkedSessionID = nil にリセットする
  • タスクの sortOrder はユーザーが並び替えた順序を保持（デフォルトは createdAt 昇順）
  • 1日あたりのタスク上限は設けない
  • StudyPlanSheet: 試験日 - 開始日が0日以下の場合、保存ボタンを非活性にする
  • StudyPlanSheet: 土日除くトグルが ON の場合、配分日数を平日のみでカウントし、土日はタスクを生成しない
  • StudyPlanSheet: 科目未選択時は保存ボタンを非活性にする（v1.3〜: 科目は必須）
  • 期限切れ（dueDate < today）かつ未完了のタスクは、当日の日ビューにも「期限切れ」バッジ付きで表示する（期日を過ぎたタスクが消えないよう配慮）
  • 週ビューの取りこぼしセクション: 期限切れかつ未完了のタスクをオレンジ色で別グループ表示する（責めない演出。赤は使用しない）
  • SubjectProgressRow の「その他」グループ: subject == nil のタスクを集計して表示する

エラー処理

発生条件対応
タスク追加時のタイトルが空文字追加ボタンを disabled にして保存させない
SwiftData 保存失敗Logger でエラーログを出力。UI 上は変更なし（楽観的更新しない）
「記録して完了」でセッション保存後にタスク更新が失敗Logger でエラーログを出力。セッションは保存済みのため再試行は不要（タスクを手動で完了させることで回復可能）
StudyPlanSheet で試験日が開始日より前保存ボタンを非活性。「試験日は開始日以降を設定してください」とインライン表示
データ移行（v1.1 → v1.2）で変換失敗Logger でエラーログ。失敗した個別タスクをスキップして続行。次回起動時に再試行しない（フラグ保存済み）

実装メモ

  • StudyTask は @Model final class。全プロパティにデフォルト値あり（マイグレーション対応）
  • @Query のフィルタは dueDate の日付範囲条件（startOfDay 〜 endOfDay）を使用
  • 旧フィールド periodRawValue / periodStart / periodEnd は削除せず残す（後方互換）
  • ManualStudyEntrySheet への拡張: initialNote: String? パラメータ、onSaved: ((UUID) -> Void)? コールバックを追加
  • CompactGoalCard（ホーム画面）は今日の StudyTask を別 @Query で取得し「N/M件」を表示
  • StudyTaskMigrationService: AppDelegate/onAppear で一度だけ呼び出す。UserDefaults["taskMigrationV12Completed"] フラグ管理
  • 期限切れバッジ: dueDate < today の未完了タスクに「期限切れ」ラベルを赤色で付与
  • print() デバッグ禁止。os.log（Logger）を使用
  • StudyTask に subject フィールドを追加（v1.3〜）。SwiftData マイグレーション対応: デフォルト値 nil を設定する
  • SubjectProgressRow: GoalEditSheet・週ビューから再利用する共通コンポーネント。引数は subject: Subject、completed: Int、total: Int
  • 週ビュー取りこぼしセクション: dueDate < startOfDay(today) かつ isCompleted == false のタスクをオレンジ（Color.orange）グループで表示
  • GoalEditSheet のタスク一覧: @Query で目標期間内タスクを取得後、subject でグルーピングして SubjectProgressRow に渡す
  • StudyPlanSheet の科目選択: Picker で Subject 一覧から選択。selectedSubject が nil の場合は保存ボタンを disabled にする
