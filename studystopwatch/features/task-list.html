やることリスト機能

今日 / 今週 / 今月単位で学習タスクを管理し、セッション記録と連携して完了できる機能

v1.0 機能ID: SS-FEAT-TASK-01

概要

ユーザーが「今日やること」「今週やること」「今月やること」を事前に登録し、 学習セッション終了と同時にタスクを完了できる機能。 タスクは StudyTask モデルで管理され、期間種別（day / week / month）ごとに独立したリストとして表示される。 ホーム画面の CompactGoalCard にはタスク進捗（N/M 件）が表示され、学習計画の意識化を促す。

関連画面

  • 目標・やることリスト画面（GoalEditSheet） — タスクの作成・編集・削除・完了操作の主要 UI
  • カレンダー画面（DayCalendarView） — 選択日のタスクをインライン表示・操作
  • ホーム画面 — CompactGoalCard でタスク件数（N/M）を表示

処理フロー

タスク追加フロー

sequenceDiagram participant U as ユーザー participant V as GoalEditSheet participant VM as ViewModel participant DB as SwiftData U->>V: タイトルを TextField に入力 U->>V: 「追加」ボタンをタップ V->>VM: addTask(title, period) VM->>VM: periodStart / periodEnd を算出 VM->>DB: modelContext.insert(StudyTask) DB-->>VM: 保存完了 VM-->>V: @Query 経由でリスト更新 V-->>U: 追加したタスクが一覧に表示

「記録して完了」フロー

sequenceDiagram participant U as ユーザー participant V as GoalEditSheet participant MES as ManualStudyEntrySheet participant VM as ViewModel participant DB as SwiftData U->>V: 「📝 記録して完了」ボタンをタップ V->>MES: .sheet で表示（initialNote, onSaved クロージャを渡す） U->>MES: 科目・時間・メモを入力して保存 MES->>DB: modelContext.insert(StudySession) MES->>VM: onSaved(session.id) コールバック呼び出し VM->>DB: task.isCompleted = true, task.completedAt = now, task.linkedSessionID = session.id DB-->>VM: 保存完了 VM-->>V: @Query 経由でリスト更新（タスクが完了状態に） V-->>U: チェックマーク付きで完了表示

シンプル完了フロー

sequenceDiagram participant U as ユーザー participant V as GoalEditSheet participant VM as ViewModel participant DB as SwiftData U->>V: チェックボックスをタップ V->>VM: toggleComplete(task) VM->>DB: task.isCompleted = true, task.completedAt = now DB-->>VM: 保存完了 VM-->>V: リスト更新（チェックマーク付き） V-->>U: 完了表示

ビジネスルール

  • タスクは期間（day / week / month）ごとに独立。同じタイトルでも期間が異なれば別レコード
  • periodStart / periodEnd は登録時の選択期間から算出し、固定値で保存する（後から期間が変わっても影響なし）
  • 「記録して完了」でキャンセルした場合、タスクは未完了のまま維持する
  • 完了済みタスクの再オープン（チェック解除）は可能。isCompleted = false、completedAt = nil、linkedSessionID = nil にリセットする
  • タスクの sortOrder はユーザーが並び替えた順序を保持（デフォルトは createdAt 昇順）
  • 1期間あたりのタスク上限は設けない
  • 週 / 月ビューでは完了率（完了数 / 全体数 × 100）を整数で表示する（小数点以下切り捨て）

外部連携

連携先用途備考
SwiftData（StudyTask）タスクの永続化・@Query での取得AppGroup 共有なし（Widget 非対応）
SwiftData（StudySession）「記録して完了」時に生成されるセッションlinkedSessionID で紐づけ
ManualStudyEntrySheet手動セッション記録 UIonSaved / initialNote を追加拡張済み

エラー処理

発生条件対応
タスク追加時のタイトルが空文字追加ボタンを disabled にして保存させない
SwiftData 保存失敗Logger でエラーログを出力。UI 上は変更なし（楽観的更新しない）
「記録して完了」でセッション保存後にタスク更新が失敗Logger でエラーログを出力。セッションは保存済みのため再試行は不要（タスクを手動で完了させることで回復可能）

実装メモ

  • StudyTask は @Model final class。全プロパティにデフォルト値あり（マイグレーション対応）
  • @Query のフィルタは periodRawValue と periodStart/periodEnd の範囲条件を組み合わせる
  • ManualStudyEntrySheet への拡張: initialNote: String? パラメータ、onSaved: ((UUID) -> Void)? コールバックを追加
  • CompactGoalCard（ホーム画面）は today の StudyTask を別 @Query で取得し「N/M件」を表示
  • TaskPeriod enum: day / week / month の rawValue は String
  • print() デバッグ禁止。os.log（Logger）を使用
