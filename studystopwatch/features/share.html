シェア機能（4種カード）

学習結果を4種類のカードとして画像化し、SNS共有・写真保存ができる機能

v1.2.0 機能ID: SS-FEAT-SHARE

概要

学習結果を4種類のカード（Daily / Summary / Heatmap / Graph）として画像化し、SNS に共有する機能。 v1.2.0 で Instagram Story 専用レイアウトを廃止（Facebook API 制約による）し、 InstagramShareService を削除。代わりに「写真に保存」ボタンを追加し、 ユーザーが写真ライブラリに保存してから任意の SNS に貼り付けられるようにした。

v1.2.0 の変更概要

  • Instagram Story 専用レイアウト廃止（Facebook API の制約により不安定のため削除）
  • InstagramShareService（pasteboard 経由の URLスキーム起動）を削除
  • PhotoLibrarySaver（写真ライブラリへの保存）に置き換え
  • LSApplicationQueriesSchemes の instagram-stories エントリを削除

エラー処理

発生条件対応
写真ライブラリ権限が拒否されている「設定アプリから写真アクセスを許可してください」アラート。「設定を開く」ボタンで UIApplication.shared.open(settingsURL)
写真保存中にエラー発生Logger でエラーログ出力。「保存に失敗しました。もう一度お試しください。」アラートを表示
ImageRenderer で画像生成失敗Logger でエラーログ出力。保存・共有ボタンを非活性にしてユーザーに通知しない（再試行で回復見込み）

実装メモ

  • InstagramShareService を削除。UIPasteboard への書き込みコードも合わせて削除
  • Info.plist の LSApplicationQueriesSchemes から instagram-stories エントリを削除
  • PhotoLibrarySaver は PHPhotoLibrary.requestAuthorization(for: .addOnly) を使用（.readWrite は不要）
  • 「写真に保存」ボタン: isLoading 中は ProgressView をインライン表示し、ボタンは disabled
  • Premium テーマに「PREMIUM」オレンジバッジを明示追加（ロック状態の発見性向上）
  • テーマ選択 UI: 無料テーマは通常表示。プレミアムテーマは右上に PREMIUM バッジ付きで表示し、未課金ならタップで Paywall を表示
  • print() デバッグ禁止。os.log（Logger）を使用
