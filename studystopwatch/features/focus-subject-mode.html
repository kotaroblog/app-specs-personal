フォーカス科目モード

ホーム画面に「今日の1科目」をピン留めし、選ぶ疲れをなくしてタイマー起動までを1タップで完結させる機能

v1.1.0 機能ID: SS-FEAT-FOCUS-01 無料

概要

複数の科目を登録している受験生は「今日どれを勉強するか」を毎回選ぶことに認知コストを使ってしまう。 フォーカス科目モードは1科目をピン留めしておくことで、アプリを開いた瞬間に「今日やること」が 大画面で表示され、即タイマー起動できる体験を提供する。

ユーザー価値: 起動→計測開始の動線を「科目選択なし」で完結させ、習慣化の障壁を下げる。
ビジネス価値: デイリーアクティブ率の向上。「毎日開くアプリ」として定着させる。
無料提供の理由: 課金の動機付けよりも DAU 向上・習慣化のほうがLTV への貢献が大きいと判断。

処理フロー

ピン留め設定フロー

sequenceDiagram participant U as User participant V as HomeView participant VM as HomeViewModel participant DB as SwiftData U->>V: ピン留めボタンをタップ（科目Aを選択） V->>VM: setFocusSubject(subjectID: A.id) VM->>DB: UserSettings.focusSubjectID = A.id を保存 DB-->>VM: 保存完了 VM-->>V: focusSubjectID が更新され状態変化 V-->>U: ヒーローカード表示に切り替わる（アニメーション）

フォーカス科目からタイマー起動フロー

sequenceDiagram participant U as User participant V as HomeView（フォーカス状態） participant VM as HomeViewModel participant TA as TimerActor participant LA as LiveActivity U->>V: 「計測開始」ボタンをタップ V->>VM: startTimer(subject: focusSubject) VM->>TA: タイマー開始（既存 startTimer フロー） TA-->>VM: タイマー開始完了 VM->>LA: Live Activity 開始（既存フロー） LA-->>VM: Activity ID 取得 VM-->>V: isTimerRunning = true V-->>U: ボタンが「停止」表示に切り替わる。経過時間のリアルタイム更新開始

ピン留め科目削除時の自動リセットフロー

sequenceDiagram participant U as User participant V as SubjectEditView participant VM as SubjectEditViewModel participant DB as SwiftData U->>V: 科目Aを削除 V->>VM: deleteSubject(A) VM->>DB: Subject A を削除 VM->>DB: UserSettings.focusSubjectID == A.id ならば nil にリセット DB-->>VM: 保存完了 VM-->>V: 科目一覧更新 V-->>U: ホーム画面は自動的にピン留めなし状態に切り替わる

ビジネスルール

  • ピン留めできる科目は常に1科目のみ。別の科目をピン留めすると自動的に前のピン留めが解除される
  • フォーカスモードは設定画面から OFF にできる。OFF 時はピン留め状態を保持（ON に戻したとき復元）
  • ピン留め科目が削除された場合、フォーカスモードは自動的にピン留めなし状態になる（フォーカスモード自体は維持）
  • タイマーが別の科目で起動中にフォーカス科目の「計測開始」をタップした場合、既存タイマーを停止してから再起動する（ユーザーに確認アラートは不要。1タップで切り替え）
  • 折りたたまれた他科目からのタイマー起動も可能（フォーカスモードはあくまでホームの「デフォルト表示」であり、他科目の利用を制限しない）
  • 科目数が1のみの場合はピン留めボタンを非表示にする（1科目なら常にそれが「フォーカス科目」と同義であるため）

外部連携

連携先用途備考
SwiftData（UserSettings）focusSubjectID・focusSubjectModeEnabled の永続化新規プロパティ2件追加。デフォルト値設定済み
既存タイマー機能（TimerActor）フォーカス科目でのタイマー起動・停止新規ロジック不要。既存 startTimer / stopTimer を呼び出すのみ
既存 Live Activity（ActivityKit）フォーカス科目でタイマー起動時のロック画面表示既存フローをそのまま利用
UserDefaults（AppGroup不要）ガイドバナー非表示フラグキー: focusModeGuideBannerDismissed

エラー処理

発生条件対応
UserSettings の保存失敗Logger でエラーをログ出力。ピン留め状態はメモリ上で保持し、次回起動時に再試行
focusSubjectID に対応する Subject が存在しない（削除済み等）自動的に focusSubjectID = nil にリセット。ユーザーへの通知なし（自然な動作として扱う）
タイマー切り替え中のエラー既存タイマー機能のエラーハンドリングに委譲

実装メモ

  • HomeViewModel に @Published var focusSubjectID: UUID? を追加。UserSettings からロード
  • ピン留めボタンのアクション: vm.setFocusSubject(id: subject.id)（トグル動作：すでにピン留め中なら解除）
  • UserSettings への focusSubjectID・focusSubjectModeEnabled 追加は SwiftData マイグレーション対応が必要。デフォルト値を設定すること
  • HomeView は focusSubjectID != nil && focusSubjectModeEnabled の条件分岐でフォーカスView / 通常Viewを切り替える
  • アニメーション: .animation(.spring(response: 0.4, dampingFraction: 0.8), value: focusSubjectID)
  • ガイドバナーは @AppStorage("focusModeGuideBannerDismissed") var bannerDismissed = false で管理
  • 科目数チェック: subjects.count <= 1 の時はピン留めUI非表示
