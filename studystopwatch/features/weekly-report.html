週次ふりかえりレポート機能

前週（月〜日）の学習実績をスナップショットとして保存し、Premium ユーザーにコーチコメント付きで提示する

機能ID: SS-FEAT-WEEKLY-01 v1.2.0 Premium 限定

概要

毎週月曜のアプリ起動時に前週（月〜日）の学習セッションを集計し、 WeeklyCoachReport モデルとして SwiftData に保存する。 Premium ユーザーはレポートタブの「先週のふりかえり」カードでコーチコメント・比較グラフを閲覧できる。 無料ユーザーには「Premium ロック」バナーを表示してアップグレードへ誘導する。

コーチ文言はモデルに保存せず View 側で導出するため、文言改善が過去レポートにも遡及して効く。 集計ロジックは BGTask を使わずアプリ起動時トリガーとする（電池・権限のコストを最小化）。

関連画面

  • レポート画面（SS-REPORT-01） — 「先週のふりかえり」カードを表示
  • プレミアム画面（SS-PREMIUM-01） — ロックバナーのタップ遷移先
  • 設定画面（SS-SET-01） — 週次ふりかえり通知のON/OFF は通知設定セクションで管理

処理フロー

sequenceDiagram participant App as App起動 participant WRS as WeeklyReportService participant SW as SwiftData participant FG as FeatureGate participant V as ReportView participant UN as UNUserNotificationCenter App->>WRS: generateIfNeeded(context:) WRS->>SW: fetch WeeklyCoachReport (weekStart == 先週月曜) alt 生成済み WRS->>UN: scheduleWeeklyReminder（冪等） WRS-->>App: return（何もしない） else 未生成 WRS->>SW: fetch StudySession（先週・前々週） WRS->>WRS: 集計（totalSeconds / topSubject / studyDayCount 等） alt totalSeconds == 0 WRS->>UN: scheduleWeeklyReminder WRS-->>App: return（空週はレポートを作らない） else totalSeconds > 0 WRS->>SW: insert WeeklyCoachReport → save WRS->>UN: scheduleWeeklyReminder end end V->>FG: canAccessWeeklyCoachReport alt Premium FG-->>V: true V->>SW: latestReport(context:) SW-->>V: WeeklyCoachReport V->>V: コーチコメントを View で導出・表示 else 無料 FG-->>V: false V->>V: ロックバナーを表示（週次レポートは Premium 機能です） end

データモデル（WeeklyCoachReport）

@Model
final class WeeklyCoachReport {
    var id: UUID = UUID()

    /// 対象週の月曜学習日開始インスタント（重複チェックのキー）
    var weekStart: Date = Date()

    /// 週の合計学習時間（秒）
    var totalSeconds: Double = 0

    /// 前々週の合計学習時間（秒） — 前週比の算出用
    var previousWeekSeconds: Double = 0

    /// 最も学習時間が多かった科目名（記録なしなら空文字）
    var topSubjectName: String = ""

    /// 最多科目が全体に占める割合（0〜100）
    var topSubjectPercent: Double = 0

    /// 週のうち勉強した日数（0〜7）
    var studyDayCount: Int = 0

    /// 生成時点での試験日までの残日数（未設定なら nil）
    var examDaysRemaining: Int?

    /// 生成時点での1日の目標分数（未設定なら nil — 達成率算出用）
    var dailyGoalMinutes: Int?

    var generatedAt: Date = Date()

    // Computed properties（保存せず View で算出）
    var changePercent: Double? { … }       // 前週比増減率
    var goalAchievementPercent: Double? { … } // 1日目標達成率
}

ビジネスルール

  • 生成タイミング: アプリ起動時に WeeklyReportService.generateIfNeeded(context:) を呼ぶ。前週（月〜日）分が未生成なら生成する
  • 空週はスキップ: 前週の totalSeconds == 0 の場合はレポートを保存しない（見せても嬉しくないため）
  • 週の境界: StudyDay.calendar の weekOfYear を使用。resetHour を加算して学習日基準の境界に揃える
  • 文言は保存しない: コーチコメント文言は View で導出（totalSeconds・studyDayCount・changePercent 等から条件分岐）。文言変更が過去レポートに遡及して効く
  • Premium 限定: FeatureGate.canAccessWeeklyCoachReport が false のユーザーにはロックバナーを表示
  • 週次通知: 毎週月曜朝9時に「先週のふりかえりが届きました」プッシュ通知（通知許可済みの場合のみ）。識別子: "weeklyReportReminder"。設定の復帰リマインダーON/OFFとは独立して管理

エラー処理

発生条件対応
SwiftData save 失敗logger.error で記録。次回起動時に再生成を試みる（weekStart の重複チェックで冪等）
dateInterval(of: .weekOfYear) が nilguard で早期 return。週次通知のみスケジュールして終了
通知権限が未許可／拒否通知スケジュールをスキップ（エラーなし。ユーザーが設定で許可したあとの次回起動から有効化）
前週が完全に空（totalSeconds == 0）レポートを保存せず、週次通知のみスケジュール

実装メモ

  • WeeklyReportService は @MainActor enum（静的メソッド群。インスタンス不要）
  • 集計クエリは全件 fetch 後に Swift 側でフィルタ（インデックスが少ない現状でも十分な件数）。件数増加時は #Predicate で DB 側フィルタに切り替えを検討
  • 学習日の境界は StudyDay.resetHour（デフォルト4時）を加算して揃える。dateInterval(of: .weekOfYear) は startOfDay（0時）を返すため、StudyDay.applyResetHour(to:resetHour:) で補正する
  • コーチコメントの View 側導出ロジック例:
    studyDayCount == 0 → 「先週は記録がありませんでした。今週こそ！」
    changePercent > 10 → 「先週より +xx% 増加！好調です」
    goalAchievementPercent >= 100 → 「週間目標達成！すばらしい」
  • Premium ロックバナーのタップ時は AnalyticsManager.shared.log(.weeklyReportLockTapped) を呼び、続けて PremiumView へ sheet 遷移する
  • 週次通知識別子 "weeklyReportReminder" は WeeklyReportService.reminderID static let として定義済み
  • 週次通知は復帰リマインダー（"returnReminder"）と識別子が異なるため、互いに干渉しない
  • テスト: weekStart に過去月曜を渡してモデルを手動 insert し、latestReport(context:) で取得できることを確認する
