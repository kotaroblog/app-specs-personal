Live Activity機能

ActivityKit連携・ロック画面とDynamic Island

v1.1機能ID: SS-FEAT-LA

概要

ActivityKitを使用してロック画面・Dynamic Islandにタイマー経過時間を表示する機能。 経過時間はアンカー日時（計測開始基準時刻）を起点にシステム側で自動更新するため、 アプリがバックグラウンドに入っても表示が凍結しない。終了時のクリーンアップを含む安全な実装。

経過時間の表示方式（v1.1）

変更点: 「毎秒アプリ側で経過時間を更新」する方式から、 「アンカー日時（計測開始の基準時刻）＋ システムの経過時間自動更新（Text(timerInterval:)）」方式に変更した。 これによりアプリがバックグラウンドに入っても経過時間の表示が凍結しなくなった。
  • 計測中: アンカー日時を起点にシステムが経過時間を自動でカウントアップ表示する（アプリが前面にいなくても止まらない）
  • 一時停止中: 凍結した経過秒を静的表示する
  • アプリ側は「開始/一時停止/再開/フェーズ切替」などの状態変化時のみ Activity を更新する（秒ごとの更新は不要）

処理フロー（v1.1）

  1. タイマー開始 → Activity を起動（アンカー日時＝計測開始時刻をセット）
  2. 状態変化時（一時停止 / 再開 / ポモドーロのフェーズ切替）→ アンカー日時・一時停止経過秒を更新
  3. 計測中の経過時間 → システムがアンカー日時を起点に自動表示（アプリ側の毎秒更新は行わない）
  4. タイマー停止 → Activity を終了
  5. アプリ起動時 → 残存 Activity を検出・削除（クリーンアップ）

ビジネスルール

  • NSSupportsLiveActivities = true（Info.plist）
  • 同時に1つのActivityのみ（既存があれば終了してから新規作成）
  • 失敗時はサイレント（タイマー本体は正常動作）
