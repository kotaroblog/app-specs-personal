通知機能

ローカル通知（日次リマインダー / 復帰リマインダー / 週次ふりかえり）

v1.2.0 機能ID: SS-FEAT-NOTIF

概要

すべてローカル通知（UNUserNotificationCenter）で実装する。サーバー不要。v1.2.0 時点で3種類の通知識別子を管理している。

通知識別子一覧

識別子種別タイミング有料/無料ON/OFF
"dailyReminder" 日次リマインダー 毎日・ユーザー設定時刻（繰り返し） 無料 設定画面トグル
"returnReminder" 復帰リマインダー 最後の記録から3日後の朝9時（1回のみ。次の記録保存で上書き） 無料 設定画面トグル（デフォルト ON）
"weeklyReportReminder" 週次ふりかえり 毎週月曜朝9時（繰り返し） Premium 常時有効（通知権限が許可されている場合）

NotificationService API（v1.2.0）

@MainActor final class NotificationService: NSObject {
    static let shared: NotificationService
    static let returnReminderID = "returnReminder"
    static let returnReminderEnabledKey = "returnReminderEnabled"  // @AppStorage キー

    func activateDelegate()           // アプリ起動時に1度呼ぶ（通知タップ計測）
    func requestPermission() async -> Bool
    func scheduleReminder(hour: Int, minute: Int)   // dailyReminder を予約
    func cancelReminder()                            // dailyReminder を取り消す
    func rescheduleReturnReminder(lastStudyDate: Date)  // 復帰リマインダーを上書き予約
    func cancelReturnReminder()                      // 復帰リマインダーを取り消す
}

日次リマインダー（dailyReminder）

  1. 初回 ON 時に requestAuthorization([.alert, .sound, .badge])
  2. 許可されたら UNCalendarNotificationTrigger(dateMatching: components, repeats: true) を作成
  3. 毎日ユーザー設定時刻に通知配信
  4. OFF 時に removePendingNotificationRequests(withIdentifiers: ["dailyReminder"])

通知本文: 「今日の勉強は始めましたか？少しずつ頑張りましょう！」

復帰リマインダー（returnReminder）v1.2.0 新規

学習記録（手動/タイマー）を保存するたびに予約を上書きする。継続して勉強している限り通知は届かない。

処理フロー

sequenceDiagram participant VM as TimerViewModel / ManualEntrySheet participant NS as NotificationService participant UNCenter as UNUserNotificationCenter VM->>NS: rescheduleReturnReminder(lastStudyDate: Date()) NS->>UNCenter: removePendingNotificationRequests(["returnReminder"]) NS->>NS: isReturnReminderEnabled? (UserDefaults) alt OFF NS-->>VM: (skip) else ON かつ通知権限 authorized NS->>NS: 3日後の朝9時の fireDate を計算 NS->>UNCenter: add(request, identifier: "returnReminder", repeats: false) NS->>NS: AnalyticsManager.log(.returnReminderScheduled) end

ビジネスルール

  • 最後の記録から3日後の朝9時に1回だけ発火
  • 通知タップ計測（returnReminderOpened）は UNUserNotificationCenterDelegate.didReceive で実装
  • 設定画面の @AppStorage(returnReminderEnabledKey) トグルでON/OFF。デフォルト ON
  • fireDate が過去日になる場合（3日後が既に過去）は予約しない

通知タイトル: 「勉強、ちょっとだけ再開しない？」
通知本文: 「3日空いても大丈夫。5分の記録からまたスタートしよう」

週次ふりかえり（weeklyReportReminder）v1.2.0 新規

WeeklyReportService が管理する。詳細は 週次ふりかえりレポート機能仕様 を参照。

  • アプリ起動時に WeeklyReportService.generateIfNeeded(context:) 内で予約を張り直す（冪等）
  • 毎週月曜朝9時・繰り返しトリガー（UNCalendarNotificationTrigger(dateMatching: components, repeats: true)）
  • 通知権限が unauthorized の場合はスキップ（権限なし時はサイレントに失敗）

通知タイトル: 「先週のふりかえりが届きました」
通知本文: 「先週の勉強をまとめました。今週のペースづくりに役立てよう」
