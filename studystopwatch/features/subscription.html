SubscriptionManager API

@Observable @MainActor final class SubscriptionManager {
    static let shared: SubscriptionManager
    var products: [Product]
    var isPremium: Bool

    func loadProducts() async
    func purchase(_ product: Product) async throws
    func restorePurchases() async throws
    func startTransactionListener()
}

FeatureGate（v1.2.0 新規）

Premium 判定の単一ソース: View から SubscriptionManager.isPremium を直接参照せず、 必ず FeatureGate.shared を経由する。将来の Remote Config 連携・A/B テストもここで完結させる。 
@Observable @MainActor final class FeatureGate {
    static let shared: FeatureGate

    var testRecordFreeLimit = 10  // 無料ユーザーの成績記録閲覧上限件数

    var canAccessAllTestRecords: Bool    // Premium のみ全件表示
    var canAccessPremiumShareTheme: Bool // Premium のみプレミアムシェアテーマ
    var canAccessPremiumStats: Bool      // Premium のみ詳細統計（前週比/1日平均/ベスト曜日）
    var canAccessWeeklyCoachReport: Bool // Premium のみ週次ふりかえりレポート
    var shouldShowAds: Bool             // 無料プランのみ広告表示
}

Premium 機能境界（v1.2.0 時点）

機能無料PremiumFeatureGate フラグ
広告（Banner / Interstitial）表示非表示shouldShowAds
シェアカードのプレミアムテーマ非表示3種解放canAccessPremiumShareTheme
レポート詳細統計（前週比/1日平均/ベスト曜日）非表示・ロックUX表示canAccessPremiumStats
成績記録の閲覧件数直近10件（保存は全件）無制限canAccessAllTestRecords / testRecordFreeLimit
週次ふりかえりレポートblur表示＋ロック誘導全内容表示canAccessWeeklyCoachReport

処理フロー（購入）

  1. Product.products(for:) で商品取得
  2. 購入時 product.purchase()
  3. 結果を checkVerified() で検証
  4. 成功 → transaction.finish() → isPremium = true
  5. Transaction.updates 監視で自動更新・解約も処理

実装メモ

  • FeatureGate.shared.canAccessAllTestRecords が false の場合、records.prefix(testRecordFreeLimit) を使って表示を絞る（保存済みデータは削除しない）
  • 成績ロックバナー表示時に gradeLockShown(hiddenCount:) を計測。バナータップ時に gradeLockTapped を計測
  • 週次カードロックタップ時に weeklyReportLockTapped を計測
  • PremiumView の source パラメータを必ず渡す（paywall_view の source 計測に必要）
