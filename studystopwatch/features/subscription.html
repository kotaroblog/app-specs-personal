PremiumBenefit 単一ソース（v1.2.1 新規）

特典表示の単一ソース: プレミアム特典の文言・順序・アイコンは PremiumBenefit 構造体（enum/struct）として一元管理する。 設定画面の PremiumView とオンボーディング最終ページのペイウォールは PremiumBenefit.allCases をループして特典リストを生成する（各画面でのハードコード禁止）。
順序特典FeatureGate フラグ
1成績を無制限に振り返り（無料は直近10件）canAccessAllTestRecords
2週次レポートcanAccessWeeklyCoachReport
3詳細統計（前週比 / 1日平均 / ベスト曜日）canAccessPremiumStats
4限定テーマ（シェアカードのプレミアムテーマ3種）canAccessPremiumShareTheme
5広告なし（Banner / Interstitial）shouldShowAds（false）

表示順方針: 記録の深掘り（成績・週次）を先頭、広告除去を末尾に配置。

FeatureGate（v1.2.0 新規）

Premium 判定の単一ソース: View から SubscriptionManager.isPremium を直接参照せず、 必ず FeatureGate.shared を経由する。将来の Remote Config 連携・A/B テストもここで完結させる。 
@Observable @MainActor final class FeatureGate {
    static let shared: FeatureGate

    var testRecordFreeLimit = 10  // 無料ユーザーの成績記録閲覧上限件数

    var canAccessAllTestRecords: Bool    // Premium のみ全件表示
    var canAccessPremiumShareTheme: Bool // Premium のみプレミアムシェアテーマ
    var canAccessPremiumStats: Bool      // Premium のみ詳細統計（前週比/1日平均/ベスト曜日）
    var canAccessWeeklyCoachReport: Bool // Premium のみ週次ふりかえりレポート
    var shouldShowAds: Bool             // 無料プランのみ広告表示
}

ロック表示UIの共通化（PremiumLockOverlay）v1.3

無料ユーザー向けの「ぼかし＋lockアイコン＋誘導文言」ロック表示UIを、 レポート詳細統計・週次コーチレポート・成績履歴・シェアテーマの4箇所での個別実装から 共通コンポーネント PremiumLockOverlay に統一した。見た目・タップ挙動・計測を1箇所に集約する。

画面別コンバージョン計測（source付与）v1.3

購入イベント（購入開始 purchase_start / 購入成功 purchase_success）に、 どの画面からペイウォールを開いたかを示す発火元（source）を付与した。 これによりペイウォール発火元ごとのコンバージョン率を分析できる。 計測イベントの詳細はロギング方針を参照。

処理フロー（購入）

  1. Product.products(for:) で商品取得
  2. 購入時 product.purchase()
  3. 結果を checkVerified() で検証
  4. 成功 → transaction.finish() → isPremium = true → ペイウォールを閉じる
  5. キャンセル・失敗・保留 → ペイウォールに留まり原因に応じたエラー文言を表示（v1.2.1）
  6. Transaction.updates 監視で自動更新・解約も処理

実装メモ

  • FeatureGate.shared.canAccessAllTestRecords が false の場合、records.prefix(testRecordFreeLimit) を使って表示を絞る（保存済みデータは削除しない）
  • ロック表示は PremiumLockOverlay に寄せる（4画面で手書きしない）。表示・タップ計測はコンポーネント内で一元化
  • 成績ロックバナー表示時に gradeLockShown(hiddenCount:) を計測。バナータップ時に gradeLockTapped を計測（移行期間中は共通イベントと併送）
  • 週次カードロックタップ時に weeklyReportLockTapped を計測（移行期間中は共通イベントと併送）
  • PremiumView の source パラメータを必ず渡す（paywall_view・purchase_start・purchase_success の source 計測に必要）
  • SubscriptionPlanButton は年額・月額共通の購入ボタンコンポーネント（v1.2.1 で共通化）。各画面でボタンUIをコピーせずこのコンポーネントを使う
  • force unwrap（!）禁止。guard let または if let でアンラップする（v1.2.1 でコード全面対応済み）
