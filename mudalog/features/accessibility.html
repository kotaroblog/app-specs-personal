アクセシビリティ設計

タップ領域・VoiceOver・Dynamic Type・色コントラストの全体方針

ML-FEAT-A11Y

概要

ムダログのアクセシビリティ改善は、操作の明確さ・色の読みやすさ・スクリーンリーダー対応・文字サイズ変更対応の4軸で実施する。 WCAGガイドライン（AA基準）をターゲットとし、視覚・操作両面での障壁を下げることを目指す。

関連画面

  • ホーム画面（ML-HOME-01）
  • 発見画面（ML-DISCOVERY-01）
  • 集中画面（ML-FOCUS-01）
  • オンボーディング画面（ML-ONBOARDING-01）

1. タップ領域（44×44 pt 以上）

Apple HIG 準拠。全操作要素のタップ領域を 44×44 pt 以上に統一する。

月移動ボタン（ReportView 等）frame 44×44 以上

3. AccessibilityAnnouncer（UIAccessibility.post）

操作結果・エラーを VoiceOver で即時読み上げるため AccessibilityAnnouncer ユーティリティを新設。 UIAccessibility.post(notification: .announcement, argument:) をラップしたシンプルな構造体。

発生タイミング読み上げ内容
保存成功トースト表示時Microcopy.Success.* の文字列
保存・更新エラー発生時Microcopy.Error.* の文字列
プラン作成失敗時Microcopy.Error.planFailed
プラン進捗更新失敗時Microcopy.Error.progressFailed
記録数不足でプラン候補なし時「まずはムダをいくつか記録すると…」

6. アクセシビリティサイズ（isAccessibilitySize）への4画面対応

dynamicTypeSize.isAccessibilitySize を判定条件として使用し、 横並び／横スクロールレイアウトを縦積みに切り替える対応を4箇所に実施した。 この分岐により、極端な文字拡大時でも各 UI コンポーネントが重ならず読める状態を保証する。

対象コンポーネント画面通常時レイアウトアクセシビリティサイズ時レイアウト
FocusPlanRow（7日プランの各行）集中（ML-FOCUS-01）HStack（タイムラインマーカー＋Day 番号＋タイトル＋ステータスバッジ 横並び）VStack（Day 番号・タイトル縦積み、ステータスバッジを下段）
発見テーマカード（熱中の芽セクション）発見（ML-DISCOVERY-01）横スクロールカードリスト（1枚 184 pt 固定幅）縦積みリスト（幅フル活用、横スクロールなし）
SectionHeader（セクション見出し）発見（ML-DISCOVERY-01）HStack（タイトル＋操作リンク 横一列）VStack（タイトル行と操作リンクを縦に分離）
月ピッカー（ReportView）レポート（月次ふりかえり）HStack（◀ 2026年6月 ▶ の横一列）VStack（年月を上段、前月・次月ボタンを下段に縦積み）
実装パターン: 各コンポーネントで @Environment(\.dynamicTypeSize) private var typeSize を取得し、 if typeSize.isAccessibilitySize { ... } else { ... } で分岐する。 共通の ViewBuilder ヘルパーには切り出さず、各コンポーネントが独立して判定することで、 将来の個別調整をしやすい状態を保つ。

7. Launch Screen 欠落によるウィンドウサイズ異常（修正済み知見）

根本原因: project.yml の info.properties に UILaunchScreen キーが存在しない場合、 iOS はアプリを「レガシー互換モード」で起動する。この状態ではシミュレーター上のウィンドウサイズが 実際の画面サイズ（例: iPhone 15 Pro Max で 402×778 pt）ではなく 異常な固定値（320×384 pt）で報告され、レイアウト計算がすべてこの誤値を元に行われる。

この異常は通常の端末ではアプリが正常に見えているように見えるが、 シミュレーター上でのイラスト高さ計算・ScrollView コンテンツサイズ・コンポーネントの配置オフセット などすべての px ベース計算に影響する。オンボーディング画面での以下の現象はすべてこれが真因だった：

  • タイトル・本文テキストが画面に表示されない
  • イラストが途中で切れる
  • スクロール不能な領域に要素が配置される
項目内容
修正方法project.yml の info.properties に UILaunchScreen: {} を追加
注意点空辞書 {} で明示することで iOS が「モダン起動シーケンス」を使用する。値を省略したり別キー名を使うと効果がない
恒久化xcodegen generate を再実行してもこの設定が失われないよう project.yml に明示する。削除禁止
副次修正ウィンドウサイズ正常化後に可視化されたイラスト内コンテンツ量超過（3シーン）は個別に修正済み（詳細は オンボーディング仕様書 を参照）

エラー処理

発生条件対応
UIAccessibility.post が無効環境で呼ばれるOS が無視するため特別なガードは不要
AccessibilityAnnouncer を空文字で呼ぶannounce("") は呼ばない（各呼び出し元でガード）

実装メモ

  • 新規 UI コンポーネントを追加する際、操作要素には必ず .frame(minWidth:minHeight:) で 44 pt を確保する
  • テキスト色を決める際、装飾色（明るい緑・紫・黄）は使わず *Text サフィックスのトークンを選ぶ
  • エラーメッセージ・成功トーストは必ず AccessibilityAnnouncer.announce() をセットで呼ぶ
  • 画面レビュー時は Accessibility Inspector（Xcode）でラベル抜けを確認する
