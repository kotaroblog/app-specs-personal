7日プランのテーマ不一致対応

進行中プランと異なるテーマを選択した際の確認ダイアログ設計

ML-FEAT-PLAN-CONFLICT

概要

7日プランは同時に1件しか進行できない仕組み（isActive = true が1件のみ）。 発見画面でユーザーが進行中プランと異なるテーマのカードをタップした場合、 旧設計では無言で遷移していたため、ユーザーが「誤って進行中プランを消してしまったか」 と混乱するリスクがあった。 本機能ではアラートで現状を説明し、ユーザーが明示的に選択できるようにする。 既存データの削除・終了は一切行わない。

関連画面

  • 発見画面（ML-DISCOVERY-01）
  • 集中（7日プラン）画面（ML-FOCUS-01）

処理フロー

sequenceDiagram participant U as ユーザー participant D as DiscoveryView participant A as Alert participant F as FocusView U->>D: テーマカードをタップ D->>D: openFocusPlan(preferredTheme:) 呼び出し D->>D: activePlan を確認 alt 進行中プランなし D->>D: createPlan() でプラン作成 D->>F: selectedTab = .focus に切替 else 同一テーマ D->>F: openActivePlan() で集中タブへ else 別テーマ D->>A: PlanConflictPrompt をセット A-->>U: 確認ダイアログ表示 alt 「進行中のプランを見る」選択 A->>D: openActivePlan() D->>F: 集中タブへ切替 else 「キャンセル」選択 A-->>D: ダイアログを閉じるだけ（状態変更なし） end end

ビジネスルール

  • 7日プランは同時に isActive = true が1件のみ（FocusPlan の整合性制約）
  • 別テーマのカードをタップしても、進行中プランは自動で終了・削除されない
  • 「キャンセル」を選んでも記録・プランのデータに一切変更なし
  • 「進行中のプランを見る」を選ぶと集中タブへ遷移（新規プランは作られない）
  • 新規プランを作りたい場合は現在のプランを7日完了させる必要がある（強制終了機能なし）

アラート仕様

項目内容
タイトル「進行中のプランがあります」
メッセージ「現在『{activeThemeTitle}』の7日プランが進行中です。7日プランは1つずつ進める仕組みです。『{requestedThemeTitle}』は、今のプランを終えてから始められます。」
ボタン1「進行中のプランを見る」（default role）
ボタン2「キャンセル」（.cancel role）

PlanConflictPrompt 構造体

private struct PlanConflictPrompt: Identifiable {
    let id = UUID()
    let activeThemeTitle: String   // 進行中プランのテーマ名
    let requestedThemeTitle: String // ユーザーが選択したテーマ名
}

@State private var planConflict: PlanConflictPrompt? として保持し、 alert(isPresented:presenting:) バインディングで表示する。 キャンセル時は planConflict = nil にするだけで状態がリセットされる。

エラー処理

発生条件対応
新規プラン作成時の modelContext.save() 失敗planMessage（destructiveText）表示 + AccessibilityAnnouncer.announce()。プランは作成されていないため追加のロールバック不要
activePlan が nil なのに openActivePlan() を呼ぶguard let で早期 return。UI 上このケースは発生しない設計

実装メモ

  • openFocusPlan(preferredTheme:) の最初で planMessage = nil にリセット（前回エラー表示を消す）
  • 同テーマ判定は preferredTheme != activePlan.themeTitle の文字列比較（大文字小文字区別あり）
  • アラート内の themeTitle はユーザー入力文字列のため、XSS 等のリスクなし（クライアントローカル表示のみ）
