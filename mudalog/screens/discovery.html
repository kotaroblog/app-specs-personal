発見画面

蓄積されたムダ記録からテーマを分析し、7日プランの入口となる画面

ML-DISCOVERY-01

目的・役割

ユーザーが繰り返し行ってしまうムダ行動のパターンを「熱中の芽」として可視化する。 テーマカードをタップすると7日プランを作成でき、集中タブへ遷移する。 進行中のプランと異なるテーマを選んだ場合は確認ダイアログを表示し、 ユーザーの意図しないプラン上書き・削除が起きないよう設計されている。

テーマカードのアフォーダンス設計

テーマカードが「操作可能」であることを視覚的に明示するため、カード下部にアクションラベル＋矢印アイコンを常時表示する。
状態アクションラベル動作
当テーマのプランが進行中「進行中を見る」＋矢印集中タブへ直接遷移
それ以外のテーマ「このテーマで試す」＋矢印プラン作成フロー（conflict判定あり）

テーマ不一致ダイアログ仕様

進行中プランと異なるテーマを選択した場合、無言で遷移せず Alert を表示する。 既存データの削除・終了は一切行わない。
項目内容
タイトル「進行中のプランがあります」
メッセージ「現在『{activeThemeTitle}』の7日プランが進行中です。7日プランは1つずつ進める仕組みです。『{requestedThemeTitle}』は、今のプランを終えてから始められます。」
ボタン1「進行中のプランを見る」→ openActivePlan() して集中タブへ
ボタン2「キャンセル」（.cancel role）→ ダイアログを閉じるだけ

詳細設計は 機能仕様書（ML-FEAT-PLAN-CONFLICT） を参照。

使用するデータモデル

@Query(sort: \WasteLog.createdAt, order: .reverse) private var logs: [WasteLog]
@Query(sort: \FocusPlan.createdAt, order: .reverse) private var plans: [FocusPlan]

var activePlan: FocusPlan? { plans.first { $0.isActive } }
var allInsights: [ThemeInsight] { MudaInsightEngine.themeInsights(from: activeLogs) }
var hotThemes: [ThemeInsight] { MudaInsightEngine.hotThemes(from: activeLogs) }

実装メモ

  • PlanConflictPrompt は Identifiable な private struct。alert(presenting:) で安全にバインド
  • テーマカードの accessibilityHint でアクション内容を VoiceOver ユーザーに伝える（「進行中の7日プランを開きます」等）
  • プラン作成失敗時は planMessage + AccessibilityAnnouncer.announce() で両方に伝達
  • テーマカード幅は 184pt 固定（横スクロール）。カード内テキストは lineLimit(2) でオーバーフロー防止
  • 色ローテーションは「装飾色（accentGreen/accentInk/paleYellow）」と「テキスト用濃色」を明確に分離して使う
