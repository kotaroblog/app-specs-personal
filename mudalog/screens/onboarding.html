オンボーディング画面

初回起動時にアプリの世界観・コンセプトを4ページで伝えるウィザード

ML-ONBOARDING-01

目的・役割

「ムダを責めない」「飽きてもいい」「戻ってくることに意味がある」というアプリの核心哲学を マスコットのイラストと短いコピーで直感的に伝える。 操作説明よりも「世界観への共感」を優先したデザイン。

ページ構成（4ページ）

ページタイトル本文要旨イラストシーン
1無駄は、伏線になる。役立たなそうな寄り道もあとから熱中につながるforeshadowing（伏線）
2飽きても大丈夫。続かなかったことも探索の一部。失敗ではないkindness（優しさ）
3何度も戻るものを見つけよう。記録が増えると自然に戻っているテーマが見えるdiscovery（発見）
47日だけ、軽く極めてみる。完璧を目指さず7日だけ小さく進めるfocus（集中）

UI 構成

要素種類説明
トップバーHStack「スキップ」テキストボタン（右端）。最終ページでは非表示
ページコンテンツ（TabView）TabView (.page)各ページは OnboardingPageView。水平スワイプで切替
OnboardingPageViewScrollView + VStackイラスト → 見出し → 本文。ScrollView 化により Dynamic Type 大サイズでも本文が読める
イラストエリアOnboardingIllustrationView画面高さに応じて illustrationHeight を動的算出。内部の装飾テキストは Dynamic Type 固定サイズ
ページインジケーターHStack dotsカスタム実装（indexDisplayMode: .never）
フッターボタンPrimaryButton / 次へボタン最終ページ以外は「次へ」、最終ページは「はじめる」

スクリーンショットモード対応

撮影時にイラスト上部が切れる不具合を修正済み。screenshotContent と standardContent で別レイアウトを使い分ける。
項目通常モードスクリーンショットモード
ページ切替TabView スワイプ現在の selection ページを1枚固定表示
レイアウトScrollView 内 VStackVStack（ScrollView なし）でイラスト→タイトル→本文を縦並び固定
イラスト高さscreenshotIllustrationHeight(for:) で動的算出同上（ステータスバー合成方式変更後）
本文 lineLimitなし（Dynamic Type 追従）4行固定
見出し lineLimitの固定なし（ScrollView 化で対応）なし

OnboardingPageView の Dynamic Type 対応

旧実装では見出しに lineLimit を固定していたため、 Dynamic Type の大サイズ設定時に本文が画面外にはみ出す問題があった。 ScrollView 化により、本文テキストが常に読める状態を保証する。

状態

状態表示内容
初回起動（.initial モード）ページ0から開始・「スキップ」あり
設定からの再表示（.manual モード）ページ0から開始・スキップボタンなし（任意閉じ可）

遷移

操作遷移先
「はじめる」ボタン（最終ページ）onboardingStore.complete() → ホーム画面
「スキップ」ボタンonboardingStore.skip() → ホーム画面
設定画面からの再表示sheet で本画面を表示 → 任意に閉じる

実装メモ

  • OnboardingIllustrationView 内部の装飾テキストは .font(.system(size: 固定値)) で Dynamic Type 固定（イラストの一部として扱うため）
  • screenshotIllustrationHeight(for:) はステータスバー領域を除いた画面高さから算出。合成時のクリッピングを防ぐ
  • OnboardingPageView は ScrollView(.vertical) で囲み、lineLimitや高さ固定を行わない
  • TabView の .page スタイルで indexDisplayMode を .never にし、カスタムドットインジケーターを下部に配置
