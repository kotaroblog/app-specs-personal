オンボーディング画面

初回起動時にアプリの世界観・コンセプトを4ページで伝えるウィザード

ML-ONBOARDING-01

目的・役割

「ムダを責めない」「飽きてもいい」「戻ってくることに意味がある」というアプリの核心哲学を マスコットのイラストと短いコピーで直感的に伝える。 操作説明よりも「世界観への共感」を優先したデザイン。

ページタイトル本文要旨イラストシーン
1無駄は、伏線になる。役立たなそうな寄り道もあとから熱中につながるforeshadowing（伏線）
2飽きても大丈夫。続かなかったことも探索の一部。失敗ではないkindness（優しさ）

OnboardingPageView の Dynamic Type 対応

旧実装では見出しに lineLimit を固定していたため、 Dynamic Type の大サイズ設定時に本文が画面外にはみ出す問題があった。 ScrollView 化により、本文テキストが常に読める状態を保証する。

アクセシビリティサイズ（dynamicTypeSize.isAccessibilitySize が true）の場合、 各ページのイラストエリアはマスコットのみのシンプル表示に簡略化する。 イラスト内の装飾テキストやコンポーネント（SproutCard・MiniTimelineCard 等）は Dynamic Type と連動しないため、アクセシビリティサイズ設定時にコンテンツと重なる恐れがある。 マスコットのみの表示に切り替えることで重なりを根本的に回避する。

Launch Screen 欠落バグとイラスト表示崩れの経緯

重大バグ（修正済み）: project.yml に UILaunchScreen 設定が欠けていたため、 iOS がアプリを「レガシー互換モード」で起動し、シミュレーター上のウィンドウサイズが 実際の画面サイズ（402×778 pt）ではなく異常な固定値（320×384 pt）となっていた。 これがオンボーディング画面でタイトル・本文が見えない、イラストが途中で切れる などの重大な表示崩れの真因だった。 project.yml の info.properties に UILaunchScreen: {} を追加して修正・恒久化済み。

ウィンドウサイズが正常化（402×778 pt）したことで、以下3シーンのイラスト内コンテンツ量の問題が可視化され、追加修正を行った。

ページ問題修正内容
2ページ目（優しさシーン）PillChip タップ領域拡大（38→44 pt）の副作用でマスコットの装飾バッジとチップが重なっていた配置オフセット調整で解消
3ページ目（発見シーン）SproutCard×3枚＋ThemeChip×2個の合計高さがイラスト枠（318 pt）を超え、上下が中央基準で切り取られていたカードを padding・フォントサイズ縮小でコンパクト化
4ページ目（集中シーン）MiniTimelineCard（Day1〜4 のタイムライン）がタイトル文字と重なっていたカードをコンパクト化し、マスコットサイズを大→中に変更

実装メモ

  • OnboardingIllustrationView 内部の装飾テキストは .font(.system(size: 固定値)) で Dynamic Type 固定（イラストの一部として扱うため）
  • screenshotIllustrationHeight(for:) はステータスバー領域を除いた画面高さから算出。合成時のクリッピングを防ぐ
  • OnboardingPageView は ScrollView(.vertical) で囲み、lineLimitや高さ固定を行わない
  • TabView の .page スタイルで indexDisplayMode を .never にし、カスタムドットインジケーターを下部に配置
  • アクセシビリティサイズ時のイラスト簡略化は dynamicTypeSize.isAccessibilitySize で分岐し、装飾コンポーネントをすべて非表示にしてマスコットのみを表示する
  • project.yml の UILaunchScreen: {} は xcodegen generate のたびに失われないよう info.properties に明示する（削除禁止）
