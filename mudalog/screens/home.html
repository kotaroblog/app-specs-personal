ホーム画面

記録CTAを最上位に置き、日常の利用習慣をホームから形成するメイン画面

ML-HOME-01

目的・役割

起動直後に「ムダを記録する」導線を最上位に提示し、記録習慣の形成を促す。 最近の記録・頻出テーマを可視化し、発見タブへの自然な誘導も行う。 ヒーローカードでブランドコピーとマスコットを維持しつつ、コンパクトに表示して記録CTAと共存させる。

UI 構成（セクション順）

v1.0 でセクション順を「情報優先度」に基づき再構成。旧順序は「ヒーロー → 今日の探索 → 月次レポート → 最近のムダ → 最近のテーマ」だったが、 日常利用で最重要な「記録CTA」と「最近の記録」を上位に移動した。
#セクション構成要素備考
1ヘッダーアプリ名「ムダログ」・キャッチコピー・設定ボタン設定ボタンは44×44pt Circle
2ヒーローカード（記録CTA）ブランドコピー・マスコット（writing variant）・「ムダを記録する」PrimaryButtonコピーは Dynamic Type 追従で自然折返し（スクショ用は固定3行）
3最近のムダ最新3件のログ行・「すべて見る」ナビリンク空状態は EmptyStateView（idle variant）。CTAは直上ヒーローに委譲し重複させない
4最近よく戻っているテーマ横スクロール ThemeChip 群・「発見へ」ショートカットログが少ない間は空文を表示
5今日の探索当日件数に応じた見出し文・進捗バッジ（Lavender Circle）スクリーンショットモード時は非表示
6月次レポート「今月のふりかえり」テキスト・レポートへのナビリンクスクリーンショットモード時は非表示

状態

状態表示内容
ログなし（初回）ヒーローカード + EmptyStateView（idle：励ましコピー・マスコット）。「ムダを記録する」ボタンは EmptyStateView には出さない（ヒーローカードの PrimaryButton と重複回避）
ログあり全セクション表示。最近のムダに最新3件
スクリーンショットモード探索カード・レポートカードを非表示。マスコット・コピーは従来の3行固定レイアウト

遷移

操作遷移先
「ムダを記録する」ボタンタップ記録タブ（LogEntryView）
ログ行タップLogDetailView（NavigationStack push）
「すべて見る」タップAllLogsView（NavigationLink）
「発見へ」タップ発見タブ（DiscoveryView）
「今月のふりかえりを見る」タップReportView（NavigationLink）
月次レポートへのプッシュ通知タップappState.pendingMonthlyReportPush = true → ReportView
設定ボタンタップSettingsView（NavigationLink）

実装メモ

  • LogRowView の tint/background は index % 3 の 3色ローテーション（緑・紫・琥珀の 濃色テキスト トークンを使用）
  • 「すべて見る」NavigationLink の minHeight を 44 pt 確保（アクセシビリティ対応）
  • ヒーローカードは ViewThatFits で横幅に応じて水平／垂直レイアウトを自動切替
  • isScreenshotMode は #if DEBUG / ScreenshotMode.isActive で判定。本番ビルドでは常に false
  • ブランドコピー「無駄を人生の伏線に。」「今日も、無駄をひとつ増やそう。」は変更禁止（ブランドの核）
