集中（7日プラン）画面

発見したテーマを7日間の小さなアクションに分けて実行する画面

ML-FOCUS-01

目的・役割

発見タブで選んだ「熱中の芽テーマ」を7日間の具体的アクションに分解し、 毎日1アクションずつチェックしながら進められる実行画面。 「完璧にやらなくていい、1つ触れればOK」という哲学のもと、 低摩擦でアクションを完了できるUXを提供する。

UI 構成

セクション構成要素表示条件
タイトルブロック「集中」見出し・「これは、ただの無駄ではないかもしれません。」常時
テーマカードテーマアイコン円・テーマ名・日付・マスコット（focus variant）プランあり
次のアクションカード「今できること」緑ソフトカード・今日のアクション文進行中プランあり・非スクショ時

FocusPlanRow（チェックボックス型UI）

7日プランの各行は「チェックボックス型UI」として設計し、タップで完了・未完了を切り替えられることを視覚的に明示する。 単なるリスト行ではなくボタンとして実装することで、操作可能性を伝える。
状態見た目
completed（完了）チェックマーク付きアイコン・テキストに取消線
today（今日）ハイライト背景・未チェックアイコン
notStarted（未着手）グレー調・未チェックアイコン

行のタップは toggleCompletion(for:in:) を呼び出す。完了済みをタップすると notStarted に戻る（再試行可能設計）。

状態

状態表示
プランなし空状態 EmptyStateView（focus）＋「発見を見る」ボタン
進行中プランありテーマカード・次のアクション・プランカード・今日のひとこと・伏線・「今日の1つを完了にする」ボタン

toggleCompletion ロジック

// 完了 ↔ 未完了のトグル（ロールバックつき）
func toggleCompletion(for item: FocusPlanItem, in plan: FocusPlan) {
    // 1. スナップショット保存
    // 2. ステータス更新
    // 3. refreshStatuses() で today を再割り当て
    // 4. modelContext.save()
    // 5. 失敗時 → スナップショットで全アイテムをロールバック
    //           → statusMessage + AccessibilityAnnouncer.announce()
}

遷移

操作遷移先
「今日の1つを完了にする」ボタンcurrentItem を toggle（同一画面更新）
FocusPlanRow タップ当該行の toggle（同一画面更新）
「探索に戻る」リンク発見タブ（selectedTab = .discovery）
「月のふりかえりを見る」ボタンReportView（navigationDestination push）
「発見を見る」ボタン（空状態）発見タブ

使用するデータモデル

@Query(sort: \FocusPlan.createdAt, order: .reverse) private var plans: [FocusPlan]

var activePlan: FocusPlan? { plans.first { $0.isActive } }
var completedPlan: FocusPlan? { plans.first { !$0.isActive && $0.completedAt != nil } }

// アクセシビリティ: 各行に accessibilityLabel / Value / Hint を付与
.accessibilityLabel("Day\(item.day) \(item.title)")
.accessibilityValue(item.status.displayTitle)
.accessibilityHint(plan.isActive ? "タップで完了と未完了を切り替えます" : "")

実装メモ

  • プランカードの行説明文（「行をタップすると…」）は非スクショ時のみ表示
  • スクショ時は items を prefix(4) に制限してコンパクト表示
  • FocusPlanRow の disabled 状態は !plan.isActive（完了プランは行タップ不可）
  • 今日のひとことは completedCount に基づき4段階で切り替わる（0件・1〜2件・3〜5件・6件以上）
  • refreshStatuses() は保存成功後のみ呼ぶ（失敗時はロールバック対象）
