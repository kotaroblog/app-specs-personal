振り返りセッション

週/月/年の5ステップ振り返りフロー。完了画面で星がひとつ灯る。

S2実装済 画面ID: LR-SESSION-01

目的・役割

「立ち止まり、見返し、言葉にする」体験を5ステップの儀式として提供する中核画面。 週次・月次・年次で共通フローを使い、期間・問い文言・ラベルを ReflectionType で切り替える。 セッション完了時に ReflectionSession を SwiftData へ保存し、選択したログの isHighlight を true に更新する。 C案規律: セッション中（STEP1〜5）は星・グロー・絵文字を一切出さない。完了画面でのみ星が1つ灯る（唯一許可する発光演出）。

ステップ構成

STEP画面内容
STEP 1ReflectionStepReview期間内のログ一覧を見返す（記録がない場合も進行可能）
STEP 2ReflectionStepChoose心に残ったログをハイライトとして選ぶ（複数可・任意）
STEP 3ReflectionStepPrompts静かな問い（種別ごとに3問ローテーション）に自由記述で答える
STEP 4ReflectionStepMood期間の総括感情を5種から1つ選ぶ（任意）
STEP 5ReflectionStepNote気づき・自由メモを入力（任意）→「振り返りを完了する」ボタン
完了ReflectionCompletionView星が1つふわりと灯る演出（fade 600ms ease-out）。累計振り返り数・期間・総括感情を表示。「閉じる」で dismiss。

UI 構成

要素種類説明
プログレスバーReflectionProgressBarSTEP1〜5の進捗を線形表示。右端に閉じる（×）ボタン。完了画面では非表示。
コンテンツエリアstep 値で @ViewBuilder 切替各 Step View をスライドアニメで切り替え。進む: trailing→leading。戻る: leading→trailing。
中断確認アラートAlert「振り返りを中断しますか？」入力内容は保存されない旨を明示。hasAnyInput が false の場合はアラートなしで dismiss。
期間ラベルperiodLabel週次: 「MM/DD – MM/DD」、月次: 「YYYY年M月」、年次: 「YYYY年」

静かな問い（ReflectionPrompts）

種別問い1問い2問い3
週次この一週間で、心が動いた瞬間は？会えてよかった人は？来週、大切にしたいことは？
月次今月、自分の中で変わったことは？手放してよかったものは？印象に残っている一日は？
年次今年の自分を一言で表すと？来年の自分へ、一言。今年いちばんの"心が動いた"出来事は？

状態

状態表示・挙動
通常（step 0〜4）プログレスバー表示。各 Step View をスライド遷移。
保存中（isSaving: true）STEP5 の完了ボタンを無効化（重複送信防止）。
保存成功completedCount を SwiftData から取得。step を stepCount(5) に進め完了画面へ。
保存失敗Logger.error でログ記録。isSaving を false に戻し STEP5 に留まる（エラーアラートは S3 以降で追加予定）。
完了画面プログレスバー非表示。星の fade 演出。「閉じる」ボタンで dismiss。
中断確認hasAnyInput が true の場合に閉じる（×）タップでアラート表示。

期間計算ルール

種別periodStartperiodEnd
weekly今日から6日前の日の開始（startOfDay）今日の 23:59:59
monthly今月の月初（startOfMonth）今日の 23:59:59
yearly今年の元日 0:00:00今日の 23:59:59

期間は init(type:) で確定し、@Query の filter に渡す。開いてから時をまたいでも期間は変わらない。

保存ロジック

  • answers: 各 Step3 の回答（空白のみは除外）を [ReflectionAnswer] に変換。
  • highlightLogIds: STEP2 で選んだ LogEntry の id 集合。SwiftData に保存後、対応 LogEntry の isHighlight = true を更新。
  • note: STEP5 の気づきメモ（空白のみは nil 保存）。
  • modelContext.save() 成功後に completedCount を FetchDescriptor でカウント取得。
  • 保存エラーは Logger(subsystem: "com.happyboy1002.LifeReview", category: "Reflection") に .error レベルで記録。

遷移

操作遷移
「次へ」ボタン（STEP1〜4）次の STEP へスライド遷移（forward: true）
「戻る」ボタン（STEP2〜5）前の STEP へスライド遷移（forward: false）
「振り返りを完了する」（STEP5）finish() → 保存 → 完了画面へ
×ボタン（セッション中・未入力）そのまま dismiss
×ボタン（セッション中・入力あり）中断確認アラート → 「中断する」で dismiss
「閉じる」（完了画面）dismiss。ふりかえりホームに戻る。

使用するデータモデル

// 期間述語は init で確定
@Query(filter: #Predicate<LogEntry> { $0.date >= start && $0.date <= end },
       sort: [SortDescriptor(\.date, order: .reverse)])
private var entries: [LogEntry]

// 完了時に挿入
let session = ReflectionSession(
    type: type,
    periodStart: periodStart,
    periodEnd: periodEnd,
    moodSummary: mood,
    answers: savedAnswers,
    highlightLogIds: Array(highlightIds),
    note: trimmedNote.isEmpty ? nil : trimmedNote
)
modelContext.insert(session)
// ハイライト更新
for entry in entries where highlightIds.contains(entry.id) {
    entry.isHighlight = true
}

実装メモ

  • sessionType: ReflectionType? が Identifiable なため、.fullScreenCover(item:) で種別ごとに自動的に新しいインスタンスが生成される。
  • アニメーション: .asymmetric(insertion: .move(edge: .trailing), removal: .move(edge: .leading))。戻る時は逆。duration 0.28秒。
  • interactiveDismissDisabled(true) でスワイプダウンを禁止（中断確認を必ず経由させる）。
  • 完了演出の星 fade: 600ms ease-out 1回のみ（C案「1画面1灯」。紙吹雪の代替）。
  • Logger は private static let logger として宣言しシングルトン運用。
