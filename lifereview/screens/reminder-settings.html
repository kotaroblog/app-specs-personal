リマインダー設定

週次振り返りの曜日・時刻をユーザーが指定する端末内ローカル通知設定画面。

S3実装済 画面ID: LR-REMINDER-01

目的・役割

ユーザーが週次振り返りを習慣化できるよう、毎週指定した曜日・時刻にローカル通知を送る設定画面。 端末内の UNUserNotificationCenter を使用するため、サーバーへのデータ送信は一切行わない。 設定→通知セクションから遷移する。

プライバシーポリシー上の重要事項: 通知スケジュール（曜日・時刻）はデバイス内にのみ保持し、サーバー送信・外部共有は行わない。 通知許可ダイアログはシステム提供のもの（UNUserNotificationCenter.requestAuthorization）を使用する。

UI 構成

要素種類説明
有効/無効トグルToggle（SwiftUI）「週次リマインダー」ラベル付き。ON にすると通知許可リクエストを発行し、以降の設定を表示する。OFF にすると既存の通知を全キャンセル。
許可拒否時の案内callout warn通知許可が拒否されている場合「設定アプリ → LifeReview → 通知を有効にしてください」とリンク付きガイドを表示（UIApplication.open で設定アプリへ遷移）。
曜日ピッカーPicker（.segmented スタイル）日・月・火・水・木・金・土の7択。既定: 日（Weekday index 1）。トグルが ON の場合のみ表示。
時刻ピッカーDatePicker（.hourAndMinute・displayedComponents: .hourAndMinute）既定: 21:00。トグルが ON の場合のみ表示。
次回通知日時プレビューText（textMuted）「次回: 〇月〇日（日）21:00」を動的計算して表示。設定変更のたびにリアルタイム更新。
通知プレビューカードAppCard（静的表示）実際に届く通知のタイトル/本文をプレビュー表示（下記「通知内容」参照）。

通知内容

項目内容
タイトル「そろそろ、ふりかえりませんか」
本文「この一週間で、心が動いた瞬間は？」
繰り返し毎週同じ曜日・同じ時刻（UNCalendarNotificationTrigger・repeats: true）
タイムゾーンAsia/Tokyo 固定
サウンドUNNotificationSound.default
通知 ID"lifereview.weekly.reminder"（固定・再設定時に上書き）

状態

状態表示内容
初期表示（未設定）トグルOFF。曜日・時刻ピッカーは非表示。既定値（日・21:00）はトグルON時に適用。
トグルON・通知許可済み曜日ピッカー + 時刻ピッカー + 次回通知プレビュー + 通知プレビューカードを表示。
トグルON・通知未許可システム許可ダイアログを表示。許可された場合は「許可済み」状態へ遷移。
トグルON・通知拒否済みcallout warn（設定アプリへの案内）を表示。曜日・時刻ピッカーは非表示。
トグルOFF既存の通知を UNUserNotificationCenter でキャンセル（identifier: "lifereview.weekly.reminder"）。

遷移

操作遷移先
画面を閉じる（または戻るボタン）設定画面に戻る
通知拒否時の「設定を開く」リンクをタップUIApplication.open(URL(string: UIApplication.openSettingsURLString)!) → システム設定アプリ（LifeReview通知設定）

使用するデータモデル

// 設定値は @AppStorage で端末内に永続化（UserDefaults 経由・SwiftData 非使用）
@AppStorage("reminder.enabled") private var isEnabled: Bool = false
@AppStorage("reminder.weekday") private var weekday: Int = 1    // 1=日曜
@AppStorage("reminder.hour")    private var hour: Int = 21
@AppStorage("reminder.minute")  private var minute: Int = 0

// 通知スケジュール（UNUserNotificationCenter）
// ※ サーバー送信なし・端末内完結
func scheduleWeeklyReminder(weekday: Int, hour: Int, minute: Int) {
    let center = UNUserNotificationCenter.current()
    let content = UNMutableNotificationContent()
    content.title = "そろそろ、ふりかえりませんか"
    content.body  = "この一週間で、心が動いた瞬間は？"
    content.sound = .default

    var components = DateComponents()
    components.weekday = weekday    // 1=日 〜 7=土
    components.hour    = hour
    components.minute  = minute
    components.timeZone = TimeZone(identifier: "Asia/Tokyo")

    let trigger = UNCalendarNotificationTrigger(dateMatching: components, repeats: true)
    let request = UNNotificationRequest(
        identifier: "lifereview.weekly.reminder",
        content: content,
        trigger: trigger
    )
    center.removePendingNotificationRequests(withIdentifiers: ["lifereview.weekly.reminder"])
    center.add(request)
}

実装メモ

  • 通知許可状態は画面表示のたびに UNUserNotificationCenter.current().getNotificationSettings で非同期確認する（アプリ外で変更される場合があるため）。
  • トグルを ON にする瞬間に requestAuthorization(options: [.alert, .sound]) を呼ぶ。許可済みの場合はダイアログは出ない（システム挙動）。
  • Flutter 版の flutter_local_notifications 相当の機能を UNUserNotificationCenter に置換。@AppStorage は SharedPreferences からの移行先。
  • 通知の identifier を固定（"lifereview.weekly.reminder"）することで、再設定時に removePendingNotificationRequests で確実に上書きできる。
  • 次回通知日時は Calendar.current.nextDate(after: .now, matching: components, matchingPolicy: .nextTime) で計算して表示する。
  • C案デザイン規律: 設定画面はシンプル構成。金色の装飾は使わない。
