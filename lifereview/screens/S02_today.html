今日タブ（S02）

毎日開いて軽く記録する入口。残り時間ミニ表示・記録 CTA・直近の記録・○年前の今日を集約する。

実装済 画面ID: LR-S02

目的・役割

ユーザーが毎日起動したときの最初の受け皿。「記録する」という行動を最小摩擦で促すことが主目的。 残り時間のミニ表示（のこり時間カード）でアプリの世界観を常に想起させつつ、 主役は「今日を記録する」CTA ボタン（画面で最も明るい要素：金ソリッド面）に置く。 直近 3 件の記録プレビューと「○年前の今日」機能で、過去の記録との縦断的なつながりも提供する。

C案規律（1画面1灯）: 「今日を記録する」ボタン（amber 背景）が画面内で最も明るい要素。他の要素は全て沈める。グロー効果・星・絵文字は出さない。

UI 構成

要素種類説明
NavigationStackコンテナタイトル「今日」（明朝インライン）。右上に「+」ボタン（記録追加の代替入口）。
残り時間ミニカード（RemainingTimeMiniView）NavigationLink → LifeTimelineScreen「のこり時間」ラベル（caption・tracking 2）＋残り年数・ヶ月数（30pt semibold monospacedDigit）＋ThinProgressBar（amber 細線）。タップで人生の週グリッド全画面へ遷移。UserProfile が存在しない場合は非表示。
「今日を記録する」CTAButton「square.and.pencil」アイコン＋テキスト（16pt semibold・#14110A 色）。背景 AppColors.amber・角丸 14px・高さフル幅。タップで LogFormView（新規）をシートで表示。
直近の記録セクション（recentSection）VStack + ForEach見出し「直近の記録」（明朝 18pt）＋件数バッジ。直近 3 件を LogCardView で表示。各カードは NavigationLink → LogDetailView。記録がない場合は EmptyStateView（moon.stars・「今日のことを、ひとつ残してみませんか」）。
○年前の今日セクション（onThisDaySection）VStack + ForEach同月日の過年度記録を「○年前」バッジ付きの LogCardView で表示。記録が存在する場合のみセクション表示。

サブ画面（Today タブから遷移）

記録フォーム（LogFormView）

要素説明
カテゴリ選択（必須）7種（体験・達成・人間関係・作品・創作・学び・旅・日常の幸せ）をグリッドチップで選択。未選択時は「保存」ボタン disabled。
気持ち選択（任意）5種（喜び・充実・穏やか・疲れ・不安）をアイコン＋ラベル縦並びチップで選択。「クリア」ボタンで解除可能。
メモ入力（任意）TextEditor（minHeight 120）。最大 500 文字想定。プレースホルダーなし。
日付選択（任意）DatePicker（in: ...Date()・displayedComponents: .date）。既定は本日。過去日付への記録も可能。
ツールバー「閉じる」（cancellationAction）・「保存」（confirmationAction）。保存時に modelContext.save() → 失敗時はアラート + rollback。
編集モードLogEntry を引数に渡すと編集モード（既存値でフォームを初期化）。タイトル「記録を編集」。

記録詳細（LogDetailView）

要素説明
ヘッダーカテゴリアイコン（48px 円バッジ・カテゴリ色 14%透過）＋カテゴリ名＋感情タグ（任意）。
本文テキストentry.content が空でない場合のみ表示。
ツールバーメニュー「ellipsis.circle」メニューボタン → 「編集（pencil）」「削除（trash・destructive）」。削除は確認アラートを経由。
タイトル日付（ymd フォーマット）。navigationBarTitleDisplayMode: .inline。

人生の週グリッド全画面（LifeTimelineScreen）

要素説明
WeekGridBodyサマリー（現在年齢・経過週・残り週・合計週）＋凡例＋WeekGridCanvas（52列 × targetAge 行・Canvas 描画）。
タイトル「人生の週」インラインナビゲーションタイトル。
UserProfile なし時「プロフィールを設定してください」テキストを textMuted で表示。

状態

状態表示内容
初期表示（記録なし）残り時間ミニカード（UserProfile あり）→ 金 CTA → recentSection に EmptyStateView（moon.stars）
記録あり直近 3 件の LogCardView を表示。同月日の過去記録があれば「○年前の今日」セクションも表示。
UserProfile 未設定残り時間ミニカード非表示。CTA と記録セクションのみ表示。
LogFormView 表示中.sheet でオーバーレイ表示。背後のタブはそのまま残る。

遷移

操作遷移先
残り時間ミニカードをタップLifeTimelineScreen（人生の週グリッド全画面）へ NavigationLink push
「今日を記録する」ボタンタップLogFormView（新規作成）を .sheet で表示
ナビバー「+」ボタンタップLogFormView（新規作成）を .sheet で表示
LogCardView タップLogDetailView を NavigationLink push
LogDetailView 内「編集」LogFormView（編集モード）を .sheet で表示
LogDetailView 内「削除」→ 確認記録を削除して LogDetailView を dismiss（前画面へ）

アクセシビリティ対応

  • 残り時間ミニカード: .accessibilityElement(children: .combine) ＋ .accessibilityLabel("のこり時間") ＋ .accessibilityValue("\(years)年\(months)ヶ月")。
  • ナビバー「+」ボタン: .accessibilityLabel("記録を追加")。
  • 直近の記録セクション見出し: .accessibilityAddTraits(.isHeader)。
  • LogFormView のカテゴリチップ: .accessibilityAddTraits(isSelected ? .isSelected : [])。
  • LogDetailView のメニューボタン: .accessibilityLabel("記録の操作メニュー")。
  • LogCardView にはアクセシビリティ属性を LogCardView 内部で設定（今後の拡張を見据え）。

実装メモ

  • TodayView は NavigationStack を持つ。今日タブ内のすべての遷移はこの NavigationStack 内で完結する。
  • LogFormView は新規・編集兼用。entry: LogEntry? = nil が nil なら新規、non-nil なら編集。
  • LogDetailView の削除は modelContext.delete(entry) → modelContext.save() → 失敗時 rollback + アラート。
  • onThisDay の計算は @Query の結果（逆順全件）をフィルターしているため、記録件数が増えると計算コストが上がる。将来的にはクエリを分離すること。
  • 「○年前の今日」は today のみ計算。記録が0件の場合はセクションごと非表示（if !onThisDay.isEmpty）。
  • AppColors.background（.ignoresSafeArea なし）を ScrollView の背景に設定。ナビゲーションバーは UINavigationBarAppearance でダーク統一済み。
