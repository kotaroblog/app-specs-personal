振り返りセッション機能

5ステップフロー・保存・ハイライト更新・期間計算ロジックの機能仕様。

S2実装済 機能ID: LR-FEAT-SESSION

概要

LifeReview の中核機能。「立ち止まり、見返し、言葉にする」儀式を5ステップで提供し、 完了時に ReflectionSession を SwiftData へ保存、選択したログの isHighlight を更新する。 週次・月次・年次を ReflectionType で統一フローとして扱い、 期間計算・問い文言・ラベル表示のみを type に応じて切り替える設計。

関連画面

  • ふりかえりホーム（LR-REFLECT-HOME） — セッション起動ポイント
  • 振り返りセッション（LR-SESSION-01） — メイン実装画面
  • これまでの軌跡（LR-TIMELINE-01） — 完了後の確認先

処理フロー

sequenceDiagram participant U as ユーザー participant Home as ReflectHomeView participant Session as ReflectionSessionView participant Context as ModelContext U->>Home: リズムカードをタップ（週/月/年） Home->>Session: fullScreenCover(item: type) Session->>Session: period(for: type) で対象期間を確定 Session->>Context: @Query で期間内 LogEntry を取得 U->>Session: STEP1 ログを見返す → 「次へ」 U->>Session: STEP2 ハイライト選択 → 「次へ」 U->>Session: STEP3 静かな問いに回答 → 「次へ」 U->>Session: STEP4 総括感情を選択 → 「次へ」 U->>Session: STEP5 気づきメモ入力 → 「振り返りを完了する」 Session->>Context: ReflectionSession を insert Session->>Context: 選択 LogEntry の isHighlight = true Session->>Context: modelContext.save() Context-->>Session: 保存成功 Session->>Session: step = stepCount → 完了画面へ Session-->>U: 星がひとつ灯る演出（fade 600ms） U->>Session: 「閉じる」 Session-->>Home: dismiss

ビジネスルール

  • 周期は強制しない。「やりたい時に」できる設計。ただし前回からの経過日数をホームで表示し、静かに促す。
  • 未記録期間でも振り返り可能（「今週は記録が少なかった」も気づき）。ログ0件でも STEP1 を通過できる。
  • 各入力（ハイライト選択・問い回答・感情・メモ）はすべて任意。1つも入力せず完了してもよい。
  • answers は空白のみの回答を除外して保存する。
  • note は空白のみの場合は nil として保存する。
  • 保存は isSaving フラグで二重送信を防ぐ。
  • 中断時（×ボタン）: hasAnyInput が true の場合はアラートで確認。false の場合は確認なし dismiss。
  • セッション画面は interactiveDismissDisabled(true) でスワイプダウンを禁止。×ボタン経由のみ閉じられる。

期間計算ロジック

flowchart TD A[ReflectionType] -->|weekly| B["periodStart = 今日-6日 の startOfDay
periodEnd = 今日の 23:59:59"] A -->|monthly| C["periodStart = 今月の月初（startOfMonth）
periodEnd = 今日の 23:59:59"] A -->|yearly| D["periodStart = 今年の元日 0:00:00
periodEnd = 今日の 23:59:59"]

期間は init(type:) 時に確定し、@Query の filter 述語として渡す。 セッションを開いてから日付が変わっても期間は変わらない（init 時確定）。

実装メモ

  • ReflectionType: Identifiable（id = rawValue）。.fullScreenCover(item: $sessionType) で型安全に種別ごとのセッションを起動できる。
  • 問い（prompts）は ReflectionPrompts.forType(type) で取得。コード定数テーブルで管理（多言語化対応の余地あり）。
  • アニメーション: withAnimation(.easeInOut(duration: 0.28)) + .transition(.asymmetric(...))。
  • セッションの answers は [Int: String]（prompts インデックスキー）で STEP3 中に保持 → 完了時に [ReflectionAnswer] に変換して保存。
  • Flutter 版との構造対応: drift の answers: TextColumn（JSON文字列） → Swift の answers: [ReflectionAnswer]（Codable配列）。SwiftData が内部的にシリアライズする。
