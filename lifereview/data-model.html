データモデル

SwiftData @Model 構造とリレーション（Phase S2 時点）

技術移行メモ: LifeReview は Flutter（drift / Riverpod）から Swift（SwiftData / @Observable）へ全面移行中。 本ドキュメントは Swift 実装の正を記述する。Flutter 版の drift スキーマ（ReflectionSessions テーブル等）は参考として併記する。
Phase S3 での主な技術置換: flutter_local_notifications → UNUserNotificationCenter（通知）、SharedPreferences → @AppStorage（リマインダー設定の永続化）。

UserProfile

ユーザープロフィール（常に1件のみ存在）。生年月日と目標年齢から残り時間・週グリッドを計算する。

@Model
final class UserProfile {
    var id: UUID = UUID()
    /// 生年月日
    var birthDate: Date = Date()
    /// 目標年齢（人生の終端として扱う想定年齢）
    var targetAge: Int = 80
    var createdAt: Date = Date()
}

// 計算 extension（UserProfile+Life.swift）
extension UserProfile {
    var targetDate: Date      // 誕生日 + targetAge 年後の同月日
    func remaining(from:) -> TimeInterval       // 残り秒数（過去ならゼロ）
    func progress(from:) -> Double             // 進捗率 0.0〜1.0
    func remainingYearsMonths(from:) -> (years: Int, months: Int)
    var totalWeeks: Int                        // targetAge × 52
    func elapsedWeeks(from:) -> Int            // 経過週数
    func currentAge(from:) -> Int              // 現在年齢
}
注意: UserProfile は常に1件のみ存在する設計。オンボーディング完了時に1件 insert し、以降は更新のみ行う。 設定画面の「全データを削除」で削除後、onboardingCompleted=false を設定してオンボーディングに戻る。

LogEntry

日々の記録（ログ）。振り返りの素材になる。

@Model
final class LogEntry {
    var id: UUID = UUID()
    var date: Date = Date()
    /// 本文（任意・最大500文字想定）
    var content: String?
    /// 添付画像のファイルパス配列
    var imagePaths: [String] = []
    /// カテゴリ（rawValue で保存 → LogCategory enum アクセサで取得）
    var categoryRaw: String = LogCategory.experience.rawValue
    /// 感情タグ（任意・rawValue で保存）
    var moodRaw: String?
    /// 振り返りで「心に残った」と選んだ記録か
    var isHighlight: Bool = false
    var createdAt: Date = Date()
}

ReflectionSession

完了した振り返りセッション（週次・月次・年次共通）。蓄積されたこれ自体がアプリ固有の価値資産になる。

@Model
final class ReflectionSession {
    var id: UUID = UUID()
    /// 種別（rawValue で保存）— weekly / monthly / yearly
    var typeRaw: String = ReflectionType.weekly.rawValue
    var periodStart: Date = Date()
    var periodEnd: Date = Date()
    /// 期間の総括感情（任意・rawValue で保存）
    var moodSummaryRaw: String?
    /// 静かな問いへの回答（[{prompt, answer}]）
    var answers: [ReflectionAnswer] = []
    /// ハイライトに選んだ LogEntry の id 集合
    var highlightLogIds: [UUID] = []
    /// 気づき・自由記述（任意）
    var note: String?
    var completedAt: Date = Date()
    var createdAt: Date = Date()
}

ReflectionAnswer（埋め込み型）

struct ReflectionAnswer: Codable, Hashable {
    var prompt: String   // 問い文言
    var answer: String   // ユーザーの回答
}

ReflectionType（3種）

rawValuelabelshortLabel対象期間

monthly月の振り返り月当月月初〜今日末
yearly年の振り返り年今年元日〜今日末

ReflectionPrompts（コード定数テーブル）

問い文は assets ではなくコード内の ReflectionPrompts 定数で集約（多言語化を見据えた設計）。

種別問い
週次「この一週間で、心が動いた瞬間は？」「会えてよかった人は？」「来週、大切にしたいことは？」
月次「今月、自分の中で変わったことは？」「手放してよかったものは？」「印象に残っている一日は？」
年次「今年の自分を一言で表すと？」「来年の自分へ、一言。」「今年いちばんの"心が動いた"出来事は？」

BucketItem

バケットリスト項目（死ぬまでにやりたいこと）。

@Model
final class BucketItem {
    var id: UUID = UUID()
    /// やりたいこと（必須）
    var title: String = ""
    /// メモ（任意）
    var memo: String?
    /// 達成済みか
    var isCompleted: Bool = false
    /// 目標期日（任意）
    var targetDate: Date?
    /// 達成日時（任意・isCompleted=true の時に設定）
    var completedAt: Date?
    var createdAt: Date = Date()
}
  • createdAt 降順で一覧表示。
  • isCompleted=true にする際、completedAt = Date() を同時に設定する（BucketListView.toggle()）。
  • isCompleted を false に戻す際、completedAt = nil にリセットする。

Relation

大切な人（家族・友人など）。残り会える回数を見積もる対象。

@Model
final class Relation {
    var id: UUID = UUID()
    /// 名前（必須）
    var name: String = ""
    /// 続柄（父・母・親友など・任意）
    var relation: String?
    /// 現在の年齢（任意・nil の場合は回数計算を行わない）
    var currentAge: Int?
    /// 想定の終端年齢（既定: 80）
    var targetAge: Int = 80
    /// 年あたりに会える回数（既定: 1）
    var meetingsPerYear: Int = 1
    var createdAt: Date = Date()
}

// 残り会える回数（RelationRow 内のロジック）
var remainingMeetings: Int? {
    guard let age = currentAge else { return nil }
    return max(0, targetAge - age) * meetingsPerYear
}
  • currentAge が nil の場合、残り回数は表示せず「年○回」のみ表示する（RelationRow）。
  • meetingsPerYear の範囲: 1〜365（RelationEditSheet の Stepper 設定）。

マイグレーション方針

SwiftData の @Model は全プロパティにデフォルト値必須。新規プロパティ追加時は必ずデフォルト値を設定する。 Swift 版では drift の schemaVersion 管理は不要だが、SwiftData の ModelConfiguration でスキーマバージョンが変わる場合は MigrationPlan を実装する。
  • isHighlight: Bool = false — LogEntry に Phase S2 で追加。デフォルト false により既存データは自動互換。
  • ReflectionSession は Phase S2 で新規追加テーブル。Flutter 版 drift の ReflectionSessions テーブル定義（schemaVersion 5）とデータ構造は対応しているが、移行ツールは未実装（新規ユーザー優先）。
  • Phase S4 補足: オンボーディングで収集する生年月日（profile.birthDate: TimeInterval）と目標年齢（profile.targetAge: Int）は @AppStorage で管理する。SwiftData の @Model クラスには追加しない（ユーザープロファイル情報は少量でモデル変更コストを避ける設計方針）。
