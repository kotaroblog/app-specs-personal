1. コンセプト（演出方針の核）

このアプリは「タスク管理」ではなく、「重荷が少しずつ崩れ落ちていく感覚」を体験として設計する。機能の豊富さではなく、1回のチェックで得られる感情密度が競争力の源泉。

ユーザーが感じるべき感情

  • 解放感 — 重荷が消えた
  • 軽さ — タスクが減ったことが体でわかる
  • 静かな達成感 — 派手ではないが「やった」という余韻
  • 整っている感覚 — 画面がきれいに落ち着く
  • 少しだけ気持ちいい余韻 — またやりたいと思わせる

方向性の定義

ゲーム化・ポイント化・バッジ化とは逆方向。目指すのは 禅 / ASMR / 触感UI / 静かな物理感 に近い体験。

2. UI 哲学

UIは主役ではない。主役は「砂」と「崩壊演出」。

  • 薄い / 控えめ / 無機質すぎない
  • 情報量を抑える / 余白重視
  • 要素が主張しすぎない — 視線は砂へ

キーワード

Sand  ·  Quiet  ·  Soft  ·  Weight  ·  Collapse  ·  Dust  ·  Calm  ·  Zen  ·  Air  ·  Minimal

7. SandPileView 設計（実装フェーズ戦略）

最初からリアル物理を実装しない。重要なのは「気持ちよさ」であり、アニメーション品質よりも感情密度を優先する。

フェーズ実装内容目標
Phase 1 MVP PNG / WebP 砂画像 + scale 変化 + offset 変化 + opacity 変化 + 粒子少量 リリース判断 PoC（2〜3時間想定）
Phase 2 Canvas 粒子 + 崩壊方向制御 + 重力感 v1.1 以降に検討
Phase 3 Metal Shader + distortion + fluid 感 必要と判断した場合のみ

8. タスク完了演出

最重要ポイント: 「速すぎない」。
「ぱっ・シュッ・即消え」ではなく「サラッと崩れる」感覚を最優先する。

演出要素時間
崩壊0.35〜0.55 秒
粒子余韻0.7 秒以内
Haptic即時（チェック瞬間）

9. アニメーション設計

タスクチェック手順

  1. タップ
  2. 微振動（Haptic 即時）
  3. カードを少し沈ませる（scale down 小）
  4. 崩壊開始
  5. 砂粒が流れる
  6. 残りカード再配置（spring アニメ）

推奨 Animation

withAnimation(
    .spring(
        response: 0.45,
        dampingFraction: 0.82
    )
)

10. Haptic

視覚演出と同等、またはそれ以上に重要な可能性がある。完了チェック時の触感がユーザーの「気持ちよさ」を決定づける。

.sensoryFeedback(
    .impact(weight: .light),
    trigger: completed
)

調整候補: .soft / .rigid / .impact の組み合わせ。実機テストで決定する。

11. 粒子演出ルール

粒子は少ない・軽い・空気感が原則。多量の粒子は「砂崩れ体験」ではなく「ゲームSE」になる。

判定演出
NG 爆発 / 火花 / 派手 / ソシャゲ感 / 大量粒子
OK 風 / 砂塵 / サラサラ / 微細 / 少量で空気感

12. サウンド

入れるなら極小。ユーザーが気づかない程度のアンビエント演出を目指す。

判定内容
推奨 サラ... / コト... / 粒落ち系（環境音に溶け込む音）
NG ポコン / かわいいSE / ゲームSE / 通知音的な音

13. タスクカード設計

  • 薄い / 少し浮く / 影は弱い / 紙と石の中間の質感
  • Corner Radius: 16〜24pt
  • Shadow: 弱め（.shadow(color: .black.opacity(0.05), radius: 12, y: 4)）
  • グラスモーフィズムは薄く適用（厚くすると主張が強くなりすぎる）
  • テキスト階層: タスクタイトル（大） / カテゴリ・時間（小・薄色）

14. パフォーマンス

粒子演出は最適化を怠るとフレームレートが低下し、体験が損なわれる。

判定内容
推奨 fake particle（opacity / scale アニメ） / image sequence / Canvas ベース描画
避ける 何百もの View 生成 / 毎フレーム geometry 更新 / 過剰 blur / SwiftUI diffing 多発

15. Claude Code 実装優先順位

  1. 静的 UI（Home / TaskCard / SandPile）
  2. タスク完了演出（scale / particle / haptic）
  3. ダークモード
  4. Canvas 粒子（Phase 2）
  5. Shader 最適化（Phase 3、必要な場合のみ）

16. MVP スコープ

判定項目
MUST 気持ちいいチェック / 軽い振動 / 崩壊感 / 空気感 / 高 fps（60fps 維持）
NOT NEEDED 複雑な管理機能 / AI / カレンダー同期 / 高機能タグ / ソーシャル機能
