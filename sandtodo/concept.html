1. コンセプト

一文要約

砂山が積み上がり、崩れ落ちることでタスクの重さと解放感を体で理解できるToDoアプリ。

キャッチコピー

「こなした仕事が、砂になって崩れる」

中核アイデア

  • 「完了 = 消える」ではなく「完了 = 崩れる」
  • 砂は「積まれていた仕事が減った」「重荷が崩れ落ちた」「処理した感覚」を視覚的に伝える素材
  • 木・ガラス・紙にはない「積み上がる／崩れる／流れる」の3つを同時表現
  • タスクが積もるほど砂山が大きくなる
  • 完了でサラサラ崩れる
  • 勢いよくスワイプで一気崩壊
  • 1日終わりに砂が流れ落ちる
  • 未完了は翌日に少し残る

単なるアニメではなく「状態変化を感情で理解させるUI」。物理演算はリアルにしすぎない（UIミニマル／崩れ0.3〜0.6秒／粒子少なめ／指追従／少し誇張）。

2. ペルソナ 3案

A: 焦りを感じるサラリーマン

  • 年齢・属性 28〜35歳 IT/商社/メーカー
  • Todoist管理疲れ、MS To Doは無機質と感じている
  • 求めていること: 重荷が消える体験、感情的な解放感

B: 自己管理に悩む大学生・若手社会人

  • 年齢・属性 20〜25歳
  • Forest/Habiticaを3日で辞める。Things 3は¥1,200で高い
  • 求めていること: 可愛さ・気持ちよさ重視のタスク管理

C: 副業フリーランス30代前半

  • 年齢・属性 30〜35歳 エンジニア/ライター/デザイナー
  • Notion重い、たすくま学習コスト高い
  • 求めていること: 1日終わりの切り替えルーティン、シンプルさ

5. MVP（v1.0 必須機能）

MVP-1: 砂山ビジュアルのタスクリスト

タスク件数に応じてAI生成砂山画像を切替（0件→空砂漠、5件以上→大砂山）。グラスモーフィズム風カード重ね表示。

MVP-2: スワイプ完了 + 砂崩れアニメ

Phase 戦略で実装する。

  • Phase 1（MVP）: PNG 砂画像 + scale / offset / opacity 変化（リアル物理不要）
  • Phase 2: Canvas 粒子 + 崩壊方向制御
  • Phase 3: Metal Shader（必要な場合のみ）

崩壊 0.35〜0.55 秒、粒子余韻 0.7 秒以内、Haptic 即時。「サラッと崩れる」感覚を最優先し、爆発・火花・派手・ソシャゲ感は禁止。詳細は デザイン・技術仕様書 参照。

MVP-3: 1日の終わり画面

23:59〜0:00のタイミングで残タスク数に応じた砂が流れ落ちる演出。ダークモード・月・進捗%・一言メッセージを表示。

v1.1以降（次フェーズ候補）

  • リピートタスク・リマインダー
  • カテゴリ/タグ（慎重に検討）
  • Apple Watch対応

10. 次フェーズ優先アクション

  1. Adobe Fireflyで砂山ステート別画像（0〜5段階）を生成し、生成ログを保存する
  2. SwiftUI Phase 1 PoC（PNG 砂画像 + scale / offset / opacity + Haptic、2〜3時間想定）で実現性確認。Phase 2 以降への進行可否を判断する
  3. アプリ名最終決定: J-PlatPat + App Store検索を実施する
  4. デザインカンプをFigma化（タスク追加・設定・初回起動画面も整理）
  5. 仕様書ポータルに画面仕様・データモデルを追加する
