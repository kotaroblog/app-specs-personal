SceneLabel レイアウト（WYSIWYG 統一）

撮影プレビュー（SwiftUI）と完成動画（CALayer）のテロップ配置を単一ソース化

1. 設計意図

撮影中にユーザーが見ているテロップ位置と、書き出し後の MP4 のテロップ位置を完全一致（WYSIWYG）させることが目的。 プレビューは SwiftUI（左上原点・point 単位）、書き出しは AVFoundation / CALayer（左下原点・pixel 単位）と座標系が異なるため、 両系統に共通の比率定数と、各系統用のヘルパで差分を吸収する。

  • 単一ソースApp/SceneLabelLayout.swift（enum）
  • 使用元（SwiftUI）ShootingView / SingleStepShootingView 内の SceneLabel
  • 使用元（CALayer）VideoCompositionService の makeStepNameLayer / makeTimeLayer
  • 本編テロップ縦位置下端から 22%（プレビュー・本編で統一）

3. SwiftUI 用ヘルパ

カードサイズ（プレビューの矩形）を基準に、SwiftUI モディファイアにそのまま渡せる値を返す。

  • leadingPadding(cardWidth:).padding(.leading) に渡す値
  • bottomPadding(cardHeight:).padding(.bottom) に渡す値
  • maxWidth(cardWidth:)SceneLabel(maxWidth:) に渡す折り返し幅
  • lineSpacing(cardHeight:)VStack(spacing:) に渡す行間
// ShootingView / SingleStepShootingView 共通
SceneLabel(
    stepName: step.name,
    time: timeText,
    stepNameFontSize: cardHeight * SceneLabelLayout.stepNameFontSizeRatio,
    timeFontSize: cardHeight * SceneLabelLayout.timeFontSizeRatio,
    maxWidth: SceneLabelLayout.maxWidth(cardWidth: cardWidth),
    spacing: SceneLabelLayout.lineSpacing(cardHeight: cardHeight)
)
.padding(.leading, SceneLabelLayout.leadingPadding(cardWidth: cardWidth))
.padding(.bottom, SceneLabelLayout.bottomPadding(cardHeight: cardHeight))

4. CALayer 用ヘルパ

レンダリングサイズ（ピクセル）を基準に、CALayer.frame の origin/size に直接代入できる値を返す。

  • layerOriginX(renderWidth:)テロップ frame.origin.x
  • layerOriginY(renderHeight:, fontSize:)テロップ frame.origin.y（descent slack 補正込み）
  • layerTextWidth(renderWidth:)テロップ frame.size.width
  • layerLineSpacing(renderHeight:)時刻 y を積み上げる際の spacing

5. descent slack 補正の仕組み

CATextLayer は top-anchored で描画されるため、グリフの下端が frame の下端より約 fontSize * 0.23 浮く（descent 領域の余白）。 SwiftUI Text の intrinsic 高さは概ね fontSize * 1.17（lineHeightFactor）なので、 CALayer の frame.height を fontSize * 1.4（textHeightFactor）に取りつつ、origin.y を下方向に補正することで両系統の glyph 下端を揃える。

  • descentSlackFactor = textHeightFactor - lineHeightFactor ≒ 0.23
  • layerOriginY(renderHeight:, fontSize:) = baseY - fontSize * descentSlackFactor
  • SwiftUI Text intrinsic（≒ fontSize * 1.17）と CATextLayer の glyph 下端が概ね揃う

6. 長文折り返し統一

SwiftUI と CATextLayer で同一の折り返し挙動（最大 2 行・末尾省略）を再現する。

  • SwiftUIText.lineLimit(SceneLabelLayout.maxLines).truncationMode(.tail)
  • CALayerisWrapped = true / truncationMode = .end / frame.height = singleLineHeight × actualLineCount
  • 行数測定actualStepNameLineCount(text:renderSize:fontName:) で NSAttributedString.boundingRect により実行数を測定

7. makeTimeLayer の追随

時刻レイヤはステップ名レイヤの実行数（1〜2 行）に追随して y 位置を決める。

  • stepNameLineCount（1...maxLines にクランプ）を受け取り、stepNameHeight × clampedLineCount 分だけ上に積む
  • 時刻の y = SceneLabelLayout.layerOriginY(renderHeight:, fontSize: stepNameFontSize) + stepNameHeight + SceneLabelLayout.layerLineSpacing(renderHeight:)
  • これにより 1 行時・2 行時のどちらでも、時刻の上端がステップ名の下端に密着する

8. 残課題（次タスク）

  • P0-1 / P0-2: ShootingView と SingleStepShootingView の 2 系統を SceneLabel 経由で完全統合し、テロップ描画コードの重複を解消する
  • 時刻レイヤの textWidth = 0.3 ハードコードを SceneLabelLayout 側の比率定数に集約する

セキュリティ・プライバシー

  • レイアウト計算は端末内で完結。外部送信なし
  • テロップ文字列はユーザー入力（ステップ名）と端末時刻のみで、機密性なし
