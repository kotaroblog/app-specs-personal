    • 背景上部28%のみグラデ（黒0.45 → 透明）。映像を遮らない
    • ルーティン名左上配置・動画高さ4.0%フォント・ユーザー選択フォント・opacity 0.95
    • 時刻ルーティン名の上に小さく（動画高さ2.4%・opacity 0.75・タイムスタンプON時のみ）

    v1.0変更前: 黒全画面 opacity 0.55 + 中央配置 + ルーティン名6.0% → 評価で「広告っぽい」「テンプレ感」と指摘され、ログ風に刷新。

    v1.0.6 で配置・サイズ・補正値を SceneLabelLayout enum に集約し、撮影プレビュー（SwiftUI）と完成動画（CALayer）の WYSIWYG を達成。

    • ステップ名（メイン）左下配置・動画高さ3.4%フォント・x=5%、y=22%
    • 時刻（サブ）ステップ名の上に小さく配置・動画高さ2.4%フォント（ステップ名の70%）

    • フォントユーザー選択（HelveticaNeue-Light等の6種）

    v1.0変更前: 右下に時刻を並列配置 → 右下のTikTok UI（いいね/コメント/シェア）と衝突 → 左下ステップ名の上に統合。

    SceneLabelLayout 単一ソース（v1.0.6 新設）

    配置比率・サイズ比率・メトリクス補正値を 1 ヶ所に集約し、撮影プレビュー（ShootingView / SingleStepShootingView）と本編テロップ（VideoCompositionService）の両方が同じ値を参照する。

    カテゴリキー値用途
    配置比率leadingRatio0.05左マージン（動画幅×5%）
    bottomRatio0.22下マージン（動画高さ×22%）
    maxWidthRatio0.55テロップ最大幅（動画幅×55%）
    lineSpacingRatio0.008行間（動画高さ×0.8%）
    サイズ比率stepNameFontSizeRatio0.034ステップ名フォント（動画高さ×3.4%）
    timeFontSizeRatio0.024時刻フォント（動画高さ×2.4%）
    メトリクス補正lineHeightFactor1.17CATextLayer の 1 行高さ係数
    textHeightFactor1.4テキスト全体高さの安全マージン
    descentSlackFactor（実装値）top-anchored 描画の下端浮き補正
    折り返しmaxLines2ステップ名の最大行数

    SwiftUI 用ヘルパ（leadingPadding / bottomPadding / maxWidth / lineSpacing）と CALayer 用ヘルパ（layerOriginX / layerOriginY(fontSize:) / layerTextWidth / layerLineSpacing）に分離。
    makeStepNameLayer / makeTimeLayer は SceneLabelLayout から座標とサイズを取得する。

    descent slack 補正（v1.0.6）

    • 課題CATextLayer は top-anchored 描画でグリフ下端が浮き、SwiftUI Text と縦位置がずれる
    • 補正式layerOriginY(renderHeight:fontSize:) は base - fontSize × descentSlackFactor で下方向に補正
    • 効果SwiftUI Text intrinsic と CATextLayer の glyph 下端が概ね揃い、WYSIWYG が成立

    長文ステップ名の折り返し（v1.0.6・maxLines = 2）

    • SwiftUI 側Text.lineLimit(maxLines) + .truncationMode(.tail)
    • CALayer 側isWrapped = true + truncationMode = .end + frame.height を実行数に合わせて動的化
    • 行数測定actualStepNameLineCount が NSAttributedString.boundingRect で実描画行数を算出（最大 maxLines）
    • 時刻の y 計算makeTimeLayer は stepNameLineCount を受け取り stepNameHeight に追随（時刻がステップ名にめり込まない）
    • frame.heightsingleLineHeight × lineCount

    ④ 透かし（v1.0で1行化・左上配置 / v1.2でy座標修正）

    • 対象無料ユーザーのみ・動画全体に常時表示
    • 位置左上（右下→左上に変更）
    • 内容「RoutineReel」のみ（CTA「App Storeで検索」は削除）
    • フォントHelveticaNeue-Medium・動画高さ2.0%・opacity 0.75
    • 影黒シャドウ radius 3
    • y 座標（v1.2修正後）y = renderSize.height - marginY - brandHeight（AVFoundation CALayer は左下原点のため、左上配置には height から引く必要がある）
    • y 座標（修正前）y = marginY（左下原点を考慮せず左下に配置されていた）

    v1.0変更前: 右下・2行（ブランド+CTA）・opacity 0.85 → 「動画体験を削る」「右下UI衝突」と指摘され、左上1行に簡素化。

    v1.2修正: y 座標のバグ修正により、撮影時の SwiftUI オーバーレイ（左上）と完成動画内のブランド名位置（左上）が一致するようになった。

    合成パイプライン

    1. クリップURL検証（FileManager 存在チェック）
    2. AVMutableComposition への各クリップ video/audio トラック挿入
    3. 各クリップの時間範囲を記録（テロップ表示制御用）
    4. renderSize 決定（最初のクリップから）
    5. AVMutableVideoComposition + AVVideoCompositionInstruction で transform 適用
    6. parentLayer に各レイヤー追加: テロップ → 透かし → 冒頭カード → アウトロカード
    7. 元音声ミュート設定（必要時）
    8. AVAssetExportSession で書き出し（プリセット自動選択）
    9. 進捗ポーリング（0.1秒間隔、progressHandler 経由）
    10. メモリ警告監視（書き出し中・OOM防止）
