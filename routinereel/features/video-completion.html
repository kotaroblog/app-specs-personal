VideoCompletion 完成実績モデル

「動画を1本つくり終えた」という完成実績の単一情報源（build69 追加）

build69 SwiftData @Model

概要

build69 より、完成本数・「今日完成したか」の判定を Video（アプリ内保存レコード）から VideoCompletion に分離した。カメラロール保存・アプリ内保存のどちらで完成させても 1件記録され、無料枠3本（Video）とは独立している。

これにより「カメラロールのみ保存するユーザー」が Hero「今日はもう1本つくれています」の達成条件を 満たせなかった問題（Video ベース判定の限界）を解消した。

SwiftData モデル定義

@Model
final class VideoCompletion {
    var id: UUID = UUID()
    var createdAt: Date = Date()
    var creationSessionId: UUID = UUID()   // 二重計上防止キー

    var routine: Routine?   // リレーション（@Relationship デフォルト cascade）
}

ビジネスルール

  • カメラロール保存が成功しても、アプリ内保存が成功しても、どちらも1件の VideoCompletion が作成される
  • 同一 creationSessionId（PreviewViewModel の1インスタンス = 1回の作成セッション）につき1件のみ挿入。カメラロール＋アプリ内の両方を保存しても完成本数は 1 本
  • カメラロールのみ保存は Video（アプリ内保存カウント）の無料3本枠を消費しない。消費するのは VideoStorageService.saveToApp 経由のみ
  • VideoCompletion はルーティン削除時に cascade delete される（Routine.completions: [VideoCompletion] リレーション経由）

VideoCompletionRecorder

@MainActor enum VideoCompletionRecorder。AVFoundation/UIKit 非依存の純粋ロジックとして実装し、SwiftData の in-memory コンテナのみでユニットテスト可能。

  • insertIfNeeded同一 creationSessionId が context 内に存在しない場合のみ insert し、作成した VideoCompletion を返す。既存なら nil（呼び出し側は戻り値を保持し、save 失敗時に delete でロールバック可能）
  • recordIfNeededinsertIfNeeded を呼び出し、新規記録なら true・既存スキップなら false を返す（@discardableResult）

VideoCompletionMigration（既存データ補完）

build69 導入前にアプリ内保存（Video）が既に存在するユーザーへの対応。

  • 戦略既存の Video レコードを video.id キー（creationSessionId として代用）で VideoCompletion として補完挿入。冪等（何度実行しても同じ結果になる）
  • マイグレーションフラグUserDefaults フラグ非依存（データ単位で挿入済みか判断するため）。同一 UUID の VideoCompletion が存在すれば挿入をスキップ
  • 実行タイミングRootView.onAppear / アプリ起動時に一度だけ実行

処理フロー

sequenceDiagram participant U as User participant PV as PreviewView participant VM as PreviewViewModel participant VCR as VideoCompletionRecorder participant CTX as ModelContext U->>PV: カメラロールに保存 / アプリに保存 PV->>VM: saveToPhotoLibrary() / saveToApp() VM->>VCR: recordIfNeeded(sessionId, routine, ctx) VCR->>CTX: fetch VideoCompletion where sessionId==? alt 既存なし（初回保存） CTX-->>VCR: []（0件） VCR->>CTX: insert VideoCompletion VCR-->>VM: true（新規記録） else 既存あり（2回目の保存） CTX-->>VCR: [existing]（1件以上） VCR-->>VM: false（スキップ） end VM-->>PV: 保存完了 PV-->>U: アラート表示

テスト

build69 で追加したユニットテスト（VideoCompletionTests）でカバーする主な観点:

  • 同一 sessionId で2回 recordIfNeeded を呼んでも VideoCompletion が1件のみ挿入されること
  • 異なる sessionId で呼ぶと別々に挿入されること
  • insertIfNeeded の戻り値（新規: 非nil, 既存: nil）が正しいこと
  • in-memory ModelContainer + ModelContext のみで依存なくテストできること
