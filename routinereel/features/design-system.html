アニメーション原則

許可されたアニメーション

  • DesignsPressableButtonStyle: 押下時 scale 0.94 spring(0.28, 0.6)
  • DesignsFloatModifier: 浮遊（offset Y、永続）— 使用箇所限定
  • DesignsPulseModifier: 脈動（scale、永続）— REC indicator のみ
  • DesignsRingPulseModifier: 波紋（scale + opacity 片道）— Step progress 現在ステップのみ
  • easeOut フェード（appear時のみ）
  • spring 切替（タブ切替・ステップ切替）
  • blur transition（カウントダウン出現時のみ）

禁止アニメーション

  • loop animation（scale bounce 連続）
  • shimmer
  • floating UI（中央＋ボタンも v1.0 で削除）
  • bounce（spring overshoot 過多）
  • 常時 gradient shift
  • neon glow

共通コンポーネント

  • DesignsCardContainerpadding / cornerRadius / background / shadow を一括適用
  • designsSheetStyle()presentationCornerRadius 32（DesignsRadius.sheet）+ presentationDragIndicator(.visible) + presentationBackground(warm white)。v1.1.0 で設定 / プリセット選択 / ルーティン編集 / 作品一覧 / Paywall の各シートに適用（シートスタイルを統一）
  • designsCardShadow()標準カード影を1行で適用
  • designsCTAGradientLinearGradient（designsAccentPink → designsAccentOrange）。v1.1.0 で全 CTA ボタンのグラデを共通定数に統一

Reduce Motion 対応（操作性改善バッチ）

iOS「視差効果を減らす」設定が有効な場合、永続的に動作するアニメーション（浮遊・脈動・波紋）を停止する。@Environment(\.accessibilityReduceMotion) を参照し、アニメーションの有無を切り替える。

アニメーションReduce Motion OFF（通常）Reduce Motion ON
DesignsFloatModifier（浮遊）永続的に Y オフセット往復静止（オフセット 0 固定）
DesignsPulseModifier（脈動）永続的に scale 往復（REC インジケーター等）静止（scale 1.0 固定）
DesignsRingPulseModifier（波紋）永続的に scale + opacity 片道アニメ静止（非表示）
  • 実装各 modifier の body 内で @Environment(\.accessibilityReduceMotion) var reduceMotion を参照し、true の場合は .animation(nil) + 固定値を返す
  • 押下アニメーションDesignsPressableButtonStyle の scale 押下フィードバックは Reduce Motion ON でも維持（ユーザー操作への直接フィードバックのため）
  • appear 系アニメーションフェードイン / spring 切替はユーザー操作起点のため Reduce Motion の影響外

Home UI 統一原則

  1. 純黒禁止（背景は warm white、VideoPlayer 再生中のみ例外）
  2. 常時アニメーションを最小化（「Dribbble感」回避）
  3. glassmorphism を増やさない（既存 ultraThinMaterial / thinMaterial の範囲内）
  4. 余白を広く（DesignsSpacing.xl 以上を多用）
  5. カード cornerRadius は 16〜32 の範囲
  6. shadow opacity は 0.04〜0.12 の範囲
  7. 太字使いすぎ禁止（Hero以外は medium/regular）
  8. 「足すより止める」精神

デザイン品質改善（e216de7, e012cc3, e3e6ca1）

Pro バッジのコントラスト是正（e216de7）

ゴールド地（designsProGold）に白文字を乗せていた Pro バッジのコントラスト比が約 1.5:1 と WCAG AA（4.5:1）を大きく下回っていた。濃色テキストトークン designsOnGold を追加し、コントラスト比を約 7:1 に改善した。

  • 変更前ゴールド地 × 白文字（コントラスト比 約1.5:1）
  • 変更後ゴールド地 × designsOnGold（深い茶金色・コントラスト比 約7:1）
  • 新規トークンdesignsOnGold: ゴールド地に乗せる濃色テキスト用カラートークン（DesignsColor.swift に追加）

AccentColor のブランド統一（e216de7）

iOS のシステム UI（アラートボタン / 戻るボタン / ShareSheet リンク等）が既定の青を使用しており、アプリのブランドオレンジとの不統一が生じていた。AccentColor アセットをブランドオレンジ（designsAccentOrange 相当）に更新し、.tint() を RootView に付与することで全システム UI をブランドカラーに統一した。

  • 変更箇所Assets.xcassets / AccentColor（Light / Dark 両モード対応）
  • 付与箇所RootView に .tint(Color.designsAccentOrange)（全画面に継承）
  • 影響範囲アラートボタン / NavigationStack 戻るボタン / ShareSheet アクション / TextEditor カーソル / TextField カーソル

主役 CTA 意匠統一（e216de7）

Paywall 購入ボタンに影が二重掛けされていた（.designsShadow(.cta) + 独自 shadow）問題と、角丸が DesignsRadius.xxl 逸脱の値だった問題をトークン準拠に是正した。あわせて SettingsView の閉じるボタンアイコンを統一し、未適用シートにシートスタイルを適用した。

  • Paywall 購入ボタン影の二重掛けを削除（.designsShadow(.cta) 1本のみに統一）。角丸を DesignsRadius.xl (22pt) に統一
  • SettingsView 閉じるボタンアイコンを chevron.down → xmark に変更（明示的な「閉じる」意図に統一）
  • 未適用シートdesignsSheetStyle()（角丸 32pt・ドラッグインジケータ・warm white 背景）を未適用のシートに追加適用し、シート外観を統一

ルーティンサムネのテーマ別配色（e216de7）

マイルーティンカードのサムネイル背景が全ルーティンで朝焼け（peach）固定だったため、ルーティンの識別性が低かった。アイコンからテーマ（朝 / 夜 / 運動 / 学習）を導出したグラデーションを適用し、視覚的な識別性を向上させた。

  • 変更前全ルーティンが morning プリセットの朝焼け peach グラデーション固定
  • 変更後ルーティンの iconName（または presetID）からテーマを導出し、対応するプリセット色温度グラデーションを使用（morning / night / workout / study の4テーマ、デフォルトは morning）
  • テーマ判定アイコン名から推定（例: sun 系 → morning、moon 系 → night、bolt 系 → workout、book 系 → study）
  • 効果ルーティン一覧でサムネ色による区別が可能になり、朝/夜/筋トレ/学習を一目で識別できる
