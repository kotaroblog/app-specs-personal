FeatureFlag 運用（廃止記録）

v1.0〜v1.3 のリスク管理手段として運用。v1.4 でコードごと削除し廃止。

2026-06-19 廃止。FeatureFlags.swift を削除し、DesignsHomeView を唯一のホーム画面として固定。RootView / SettingsView / SnapshotSeeder の @AppStorage(FeatureFlags.useDesignHomeKey) 参照もすべて削除。SettingsView の「β機能」セクションも削除済み。

目的（当時）

v1.0 リリース直前に大幅な UI 変更（DesignsHomeView）を投入するため、リリース後にクラッシュ・UX 不具合が発覚した場合に、 コード変更・再申請を経ずに旧UIへ即時切替できる経路を確保する。

廃止の経緯

  • 廃止判断v1.0 リリース後 1〜2 ヶ月の安定稼働を確認。重大バグ報告ゼロ。計画通り廃止実施
  • 削除ファイルApp/FeatureFlags.swift（useDesignHomeKey / useDesignHomeDefault / useDesignHome を含む）
  • 削除した参照元RootView.swift / SettingsView.swift / SnapshotSeeder.swift の @AppStorage(FeatureFlags.useDesignHomeKey)
  • 旧 HomeView削除済み。RootView は DesignsHomeView のみを呼び出す
  • SettingsView「β機能」セクション（「新しいホーム画面を使う」トグル）を削除

旧実装方式（参考）

  • 実装ファイルApp/FeatureFlags.swift（削除済み）
  • 永続化方式@AppStorage(UserDefaults)
  • フラグキーuseDesignHome
  • デフォルト値true（新UI有効）
  • スコープDesignsHomeView の表示切替のみ

旧コード構造（参考）

// App/FeatureFlags.swift（削除済み）
enum FeatureFlags {
    static let useDesignHomeKey = "useDesignHome"
    static let useDesignHomeDefault = true

    static var useDesignHome: Bool {
        UserDefaults.standard.object(forKey: useDesignHomeKey) as? Bool
            ?? useDesignHomeDefault
    }
}

// App/RootView.swift（廃止後は DesignsHomeView のみ）
var body: some View {
    if hasCompletedOnboarding {
        DesignsHomeView()  // 固定
    } else {
        OnboardingView { ... }
    }
}

セキュリティ・プライバシー

  • UserDefaults に保存される値は Bool 1個のみで機密性なし
  • 外部送信されない（リモート切替ではなくローカル切替のみ）
  • App Privacy 申告への追加項目は不要
