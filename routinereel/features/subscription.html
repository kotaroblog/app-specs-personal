フリーミアム上限の一元管理（v1.1.0）

v1.1.0 より VideoStorageService.freeRoutineLimit = 3 を唯一の上限定数として制定。以前は DesignsHomeView / HomeView に maxFreeRoutines = 2 として散在していた定数を廃止し、ルーティン作成上限・動画保存上限ともに 3 本 で統一した。

  • 定数VideoStorageService.freeRoutineLimit = 3（唯一の参照先）
  • 変更前maxFreeRoutines = 2 が DesignsHomeView / HomeView に散在
  • 変更後ルーティン上限 3 件・動画保存上限 3 件で統一
  • 影響箇所保存メーター（DesignsHomeView の storageMeterBadge）・PaywallTrigger routineLimit・SettingsView の proUpgradeCard 副題

課金壁発火タイミングと PaywallTrigger 分類（596896d）

commit 596896d で PaywallView に trigger 引数を追加（デフォルト .manual）。全 6 種の trigger を AnalyticsService.PaywallTrigger として定義し、発火元を計測可能にした。

trigger 値発火タイミング呼び出し元
routineLimitルーティン 4 件目作成時（v1.1.0以降・旧: 3 件目）中央＋ボタン / Quick Start チップ / 新規ルーティン作成ボタン / 複製ボタン
stepLimitルーティン内ステップ 6 つ目追加時RoutineEditView 内 addStepButton

manualユーザーの手動タップ（情報目的）Pro バッジ / ホームヘッダー / 設定画面など

各呼び出し元で paywallTrigger State を設定してから showPaywall = true にする実装パターンを採用（SwiftUI の sheet が同一フレーム内で trigger を参照するため）。

App Store Review Guideline 3.1.2(a) 推奨に対応し、年額プランの課金条件を明示する trialNoticeSection。v1.1.0 でトライアル廃止に伴い表示テキストを更新。

  • 表示テキスト（v1.1.0・現在）「年額プランで自動継続（¥3,800/年・月換算 ¥316）。」（トライアル文言を削除）
  • フォールバック（価格未取得時）「年額プランで自動継続。」

App Store Connect 入力資料との整合

Drive の 05_リリース・運用/App Store Connect 入力資料.md の以下を v1.0 リリース時に更新必須:

  • 説明文の BGM 関連記述を全削除
  • プレミアム特典の「全BGM・全フィルタ解放」→「全フィルタ・全フォント解放」
  • キーワード「BGM」を削除し別キーワードに置換

StoreKit 2 実装

  • SubscriptionManagerstatic let shared / @Observable @MainActor / SwiftData非依存
  • Products.storekitResources/StoreKit/Products.storekit でテスト時のローカル商品定義
  • isPremium判定subscription.transactions に有効な購読が含まれるか + DEBUG時は debugProEnabled
  • 復元PaywallView の「購入を復元」ボタンから restorePurchases() 起動
  • yearlySavingPercentage月額×12 vs 年額の割引率を整数で返す computed property。例: 月¥480×12=¥5,760 vs 年¥3,800 → (5760-3800)/5760×100 = 34。PaywallView の年額バッジ「X% お得」に動的反映、計算不能時は「おすすめ」フォールバック

Pro 訴求改善（2026-06-10 反映）

ホーム + 設定の保存メーター可視化

  • DesignsHomeView「最近の動画」見出し横に storageMeterBadge を配置。Free:「X/3」/ Pro:「∞ 保存」。Free の上限到達時はピンク強調＋タップで PaywallView 起動
  • SettingsViewproUpgradeCard の副題を「アプリ内保存 X/3 ・記録をもっと自由に」へ統合し、Pro 価値を常時可視化

Paywall 年額バッジ「X% お得」動的化

  • PaywallView.planButton年額の「おすすめ」固定文言を yearlySavingPercentage を用いた「X% お得」へ差し替え。価格未取得などで計算不能な場合は「おすすめ」にフォールバック

シェアトリガーのレビュー依頼（#2 連動）

  • ReviewRequestService.onSharedToSNSシェアの「実行成功」（外部アプリ open() success / ShareSheet completed=true）時のみ発火。タップだけでは呼ばない
  • 30日ガード他のレビュー依頼トリガーと共有。iOS 側の年3回制限が二重安全弁として機能

翌日復帰通知（#2 連動）

  • ReminderService.scheduleSharePromptNotificationシェア成功の翌日 10:00 にローカル通知。share-prompt prefix で識別し pending + delivered 双方の重複をクリーンアップ
  • 許可ステータス.authorized または .provisional を許可。未許可時はサイレントスキップ
