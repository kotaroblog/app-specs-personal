オンボーディングフロー

初回起動 → 通知許可 → ATT 遅延発火のリテンション設計

概要

通知許可ダイアログを大半のユーザーに表示してリテンション基盤を構築する。 iOS の ATT（App Tracking Transparency）ダイアログとの被りを避けるため、ATT は オンボーディング完了後に遅延発火させる。通知許可ダイアログを先に表示し、続けて ATT を表示することで、両ダイアログの許可率を高める設計とした。

v1.1.0 よりページ数を 4→3 ページに圧縮した（旧 Page 1「2つの撮影スタイル解説」を削除）。 撮影スタイル解説はクリップライブラリ画面の TipKit（ShootingStyleTip）で文脈表示に移行。

TabView の swipe 無効化

オンボーディング画面の TabView に DragGesture() を被せて swipe を無効化している。 これにより、ユーザーが Page 2（preset）を「あとで」操作以外で飛ばして Page 3（reminder）に到達することを防ぎ、 pendingRoutine が nil のまま reminder ページに到達する経路を塞いでいる。

ATT 遅延発火（build69 更新: 初回価値体験後まで遅延）

ATT ダイアログは OnboardingView ではなく RootView で発火する。build69 より発火タイミングをさらに遅延させ、 「初回撮影または初回の動画完成（初回価値体験）」後まで ATT 表示を後回しにした。 オンボ内では通知許可 → カメラ/マイク と権限ダイアログが連続するため、ATT をその直後に重ねないよう分散する設計。

  • 判定基準ATTrackingManager.trackingAuthorizationStatus ベースで毎回判定（永続化フラグは使わない）
  • 重複防止@State hasLoadedAdsInSession でセッション内重複ロードを防止。アプリ kill 後の再起動では再判定される
  • 発火トリガ（build69 以前）hasCompletedOnboarding の onChange と onAppear の両方で発火（オンボ完了直後・既存ユーザーの起動時の両ケース対応）
  • 発火トリガ（build69 以降）RootView が AppSettings.hasSeenFirstValueMoment（初回撮影または初回動画完成で true になるフラグ）を onChange で監視し、true になった時点で ATT を発火。既存ユーザー（hasSeenFirstValueMoment がすでに true）は onAppear で即発火（既存挙動と同じ）
  • 広告ロードATT 結果を待たずに即時ロード開始（non-personalized モードで初期化し、ATT 許可後に personalised に切り替える設計は変更なし）

リカバリ経路

初回ダイアログで通知を拒否したユーザーが、後から iOS 設定で許可した場合のリカバリ経路を用意している。

  • RootView.onAppear で refreshAllReminders(routines:) を呼出
  • 通知拒否後にユーザーが iOS 設定で許可 → 次回起動時に、塩漬けになっていた reminderEnabled = true の Routine が自動的にスケジュールされる
  • これにより「拒否したユーザーが後から許可した」という典型的なケースをコード変更なしで救える

オンボ完了後の撮影直行（build69 追加・F5）

プリセットを選択したユーザーはオンボ完了後そのまま連続撮影に進み、「選ぶ→撮る→完成」のフローを中断させない。

  • フロープリセット選択 → リマインダー設定（またはスキップ）→ オンボ完了 → consumePendingPresetIfAny が即時発火 → ShootingView 起動（entry_source: onboarding）
  • 「あとで」スキップ時プリセット未選択のままオンボを完了 → 空のホーム画面。Quick Start（「プリセットから選ぶ」）から任意のタイミングで開始可能
  • 配管既存の AppSettings.pendingPresetRawValue + DesignsHomeView.consumePendingPresetIfAny を再利用。オンボからの遷移もURL スキームと同一の配管を通る
  • Analyticscapture_started に entry_source: "onboarding" を付与

kill/再起動時の同名ルーティン二重作成防止（build69 追加）

オンボ中にアプリを強制終了→再起動した場合、オンボ完了前に作成しかけた Routine が残りつつ、再起動後に同名の Routine が再作成されて重複する問題があった。

  • 永続化フラグAppSettings.pendingOnboardingRoutineId（UUID）: オンボ中に作成した Routine の ID を保存する
  • RoutineOrphanFinderkill 後の再起動時に pendingOnboardingRoutineId が残っている場合、SwiftData からその UUID に一致する Routine を検索し再利用を判定する純粋関数
  • 再利用条件UUID 一致 かつ clips / videos / completions がすべて空 の場合のみ再利用（名前一致だけでは再利用しない。クリップが1件でもあれば「使用中のルーティン」として新規作成）
  • 再利用できない場合孤立した Routine を削除（または放置）し、新たに Routine を作成
  • オンボ完了後pendingOnboardingRoutineId をクリア

オンボ画面説明の具体化（build69 追加・F9）

プリセット選択・リマインダー画面の説明文を具体化し、ユーザーが操作の意味をすぐに理解できるよう改善した。

  • プリセット選択（Page 1）各プリセットカードに「選ぶと撮影順とテロップが準備されすぐ撮影に進む」旨の副説明を追加。ユーザーが「このボタンを押すと何が起こるか」を事前に理解できる設計
  • リマインダー（Page 2）「通知を設定すると、指定した時間に毎日リマインドします」等の具体的な効果説明を追加（抽象的な "filming reminder." の補足）

通知拒否時のアラート（build69 追加）

オンボでユーザーが iOS 通知ダイアログを拒否した場合、従来は無音で reminderEnabled = true が塩漬けになり、後からリマインダー設定を有効化しようとしても「なぜ届かないか」が分からない「無音故障」状態だった。

  • 変更前通知拒否後も reminderEnabled = true が保持され、次回起動時の refreshAllReminders で自動回復を期待するのみ。ユーザーには何も表示されなかった（無音故障）
  • 変更後通知拒否時に「iOS設定で通知を許可すると有効になります」アラートを表示。reminderEnabled = true は維持し、後日 iOS 設定で許可された際の自動回復経路（RootView.refreshAllReminders）も引き続き機能
  • RoutineEditView との対称性ScheduleOutcome.notAuthorized（F7）と同じアラート文言・振る舞いを採用し、全画面で通知拒否時の体験を統一

セキュリティ・プライバシー

  • 通知許可・ATT 許可状態は iOS 側で管理。アプリ側で保持するのは Routine の reminderEnabled / reminderTime のみ
  • 外部送信は行わない（ローカル通知のみ）
  • ATT 拒否時は AdMob を non-personalized モードで初期化（広告は表示される）

リマインダーページの戻る導線（e216de7）

オンボーディング Page 2（reminder）でプリセットを誤って選択してしまったユーザーが、リマインダー設定前に戻って選択し直せるよう「← 戻る」ボタンを追加した。

  • 変更前リマインダーページに戻る手段がなく、誤ったプリセットを選択してしまうとオンボーディングを完了させるしかなかった（後でルーティン編集が必要）
  • 変更後「← 戻る」ボタンを追加。タップすると Page 1（preset）に戻り、プリセットを選択し直せる
  • TabView との整合TabView の swipe 無効化は継続。戻るボタンタップのみ Page 1 へ遷移（currentPage = 1）し、swipe による意図しない遷移は引き続き防止

UIコピー「週2本制作ツール」表現への統一（d59a710）

プロダクトポジショニングをオンボーディング全体で「毎日続ける習慣アプリ」から「週2本つくる制作ツール」に統一した。

  • ウェルカム説明文（Page 0）「週2本の動画制作を習慣にする」方向性を打ち出す文言に変更。初見ユーザーがアプリの用途（ルーティン動画を制作するツール）を即座に理解できるよう刷新
  • 「選ぶ→撮る→完成」3ステップ表示（Page 0）ウェルカムページにアプリ使用の基本フローを3ステップで視覚的に示すセクションを新規追加。制作ツールとしての具体的な使い方を初回接触時に伝える
  • リマインダー見出し（Page 2）英語見出しを "daily reminder"/"habit." から "set a"/"filming reminder." に変更。「毎日の習慣」ではなく「撮影のリマインダー」として位置づけ直す。英語見出し+日本語説明という既存デザインパターンは維持
