書き出し・保存フロー（保存先状態管理）

完成動画の「カメラロール／アプリ内」2 保存先を明示的な状態として管理し、進捗文言・再試行・戻る操作・工程完了後の導線を保存先ごとに正しく出し分ける（f7bb7b0）。

概要

PreviewView の保存処理は、保存先（カメラロール／アプリ内）× 工程（音声除去 → フィルタ → 保存）の組み合わせで進む。従来は保存先を状態として持たなかったため、(1) 進捗文言が常に「カメラロールに保存中」になる、(2) 保存失敗時の再試行が常にカメラロール保存を実行する、といった保存先の取り違えバグがあった。これらを ExportDestination を軸とした状態モデルで解消し、あわせて「保存中は戻れない」「アプリ内保存後に次アクションを選べる」「カメラロール保存後もアプリ内保存できる」といった工程の順序依存も是正した。

状態モデル

ExportDestination（保存先）

  • .photoLibraryカメラロール（SNS 投稿用）。無料枠を消費しない
  • .appStorageアプリ内ストレージ（あとから再生用）。無料は 3 本まで消費

ExportOperation（保存工程）と進捗文言

音声除去・フィルタ・保存はいずれも正確な進捗率が取れないため indeterminate 表示。工程は保存先を内包し、文言と保存先を必ず一致させる。

Recovery（リカバリ導線）

Recoveryボタン挙動
.recompose再合成する合成自体が失敗 → 合成をやり直す
.retrySave(destination:)保存をやり直す合成は成功・保存のみ失敗 → 失敗した保存先だけ再試行（再合成しない）

バグ修正（f7bb7b0）: 旧実装は保存失敗時の再試行が常にカメラロール保存を実行していた。.retrySave(destination:) が失敗した保存先を保持し、retrySave は .error(_, .retrySave(destination)) 状態からのみ、その destination のみを再試行する。

保存の低レベル抽象化（VideoSaving プロトコル）

カメラロール（UISaveVideo）とアプリ内（VideoStorageService）への書き込みを VideoSaving プロトコルに集約し、PreviewViewModel からは保存先に応じて呼び分ける。テストでは SpyVideoSaving を注入し「appStorage 再試行が writeToPhotoLibrary を呼ばない」等の経路を検証する（PreviewSaveDestinationTests）。

メソッド役割
writeToPhotoLibrary(path:) -> PhotoLibraryWriteResultカメラロールへ書き込み。互換性チェック内包（.success / .incompatibleFormat / .failed）
writeToAppStorage(from:) -> VideoSaveOutputアプリ内へコピー＋サムネイル生成
deleteAppStorage(fileName:thumbnailFileName:)永続 save 失敗時のロールバック削除

PhotoLibraryWriteResult で「動画形式が非互換（→ 再合成が必要）」と「保存失敗（→ 同じ保存先で再試行）」を区別し、リカバリ導線を正しく出し分ける。

保存前処理の順序

prepareVideoForExport は a. 撮影音声の除去 → b. フィルタ適用 の順で新しい一時ファイルを生成する。マスター動画（composed_）は音声を保持したまま保持し、置き換えない。中間生成物（muted_）はフィルタ成功・失敗いずれでも削除して蓄積を防ぐ。詳細（音声トラックの実除去）は 動画合成仕様 を参照。

処理フロー（カメラロール保存・音声なし＋フィルタあり）

戻る操作の分岐（BackAction）

状態BackAction挙動
.exporting（保存前処理・保存中）blocked戻れない（戻るボタン無効化）
.composing（合成中）confirmCancelCompose確認の上で合成キャンセル
未保存confirmDiscard破棄確認アラート
保存済み or 初期状態dismissImmediatelyそのまま閉じる

保存済み判定は savedDestinations（実際に保存成功した保存先の集合）を単一情報源とする。Analytics 送信済み集合（sentVideoSavedDestinations）とは目的を分け、UI 状態の代用にしない。

工程完了後の導線

  • アプリ内保存成功後: confirmationDialog「アプリに保存しました」で「続けてカメラロールにも保存／ホームへ戻る／この画面に残る」を提示（旧: OK でプレビューに留まる）。
  • ExportCompleteView から仕上がり画面へ戻る: 「仕上がり画面に戻る」で PreviewView へ pop できるようにし、カメラロール保存が先でもアプリ内保存できる順序依存を解消（旧: カメラロール保存を先にするとアプリ内保存できなかった）。
  • SNS 導線の文言明確化: ExportCompleteView の SNS ボタンを「TikTokを開く」「Instagramを開く」＋補助「カメラロールから動画を選択」に変更し、自動投稿との誤解を防止（SNS シェアフロー参照）。

ビジネスルール

  • 進捗オーバーレイの文言は「工程 × 保存先」と必ず一致させる（保存先の取り違え表示を禁止）。
  • 保存失敗時の再試行は失敗した保存先のみ実行する（他方の保存先を巻き込まない）。
  • 保存前処理・保存の実行中（.exporting）は戻る操作を禁止する。
  • VideoCompletion は保存先に関わらず 1 セッション 1 件のみ記録する（creationSessionId で二重計上防止・VideoCompletion 完成実績モデル）。

エラー処理

発生条件対応
合成失敗 / 動画形式が非互換.recompose（再合成する）
音声除去・フィルタ・保存の失敗.retrySave(destination:)（失敗した保存先だけ再試行）
写真ライブラリ権限拒否.openSettings（設定アプリを開く）
