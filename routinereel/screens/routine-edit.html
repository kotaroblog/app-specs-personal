ルーティン編集（RoutineEditView）

新規ルーティン作成・既存ルーティン編集の両用画面。v1.0で Home UI に完全統一。

基本情報

  • 画面IDroutine-edit
  • 実装ファイルViews/Home/RoutineEditView.swift
  • 表示方式sheet（モーダル）
  • 起動経路①Quick Start カード ②マイルーティン → 編集する ③プリセット選択 → カスタム/プリセット選択 ④通知タップ → 編集メニュー
  • 背景色#FAF9F7 (warm white)

画面構成

  1. 名前: ルーティン名 TextField（カード）
  2. テロップのフォント: 横スクロールフォントチップ（無料2種 + Pro4種）
  3. 撮影リマインダー: Toggle + DatePicker（有効時のみ展開）
  4. ステップ: 縦並びカード（最大5個まで・ドラッグ並べ替え対応）
  5. ステップを追加: ダッシュ枠ボタン（accent orange）
  6. ルーティンを削除: 編集時のみ表示・赤opacity 0.75

  • 確認 alert「ルーティンを削除しますか？」+「この操作は取り消せません。」
  • 削除処理（v1.1.0更新・完全クリーンアップ）① ClipStorageService で全クリップの物理ファイルを削除 → ② ReminderService.cancelReminder() で定期通知解除 → ③ 保留中の通知 ID を掃除（pendingNotification cleanup）→ ④ modelContext.delete(routine)
  • 2経路の対称化ルーティン編集シートの削除ボタンと Home 画面のスワイプ削除の両経路で同じクリーンアップ処理を実行するよう統一した（e216de7）

ステップ並び替えの VoiceOver 代替アクション（v1.1.0）

ドラッグ＆ドロップによる並び替えは VoiceOver / スイッチコントロール環境では操作できない。アクセシビリティアクションを追加し、補助技術でも並び替えを可能にした。

  • 追加アクション各ステップ行に「上へ移動」/ 「下へ移動」の accessibilityAction を付与
  • 先頭ステップ「上へ移動」は非表示（先頭ステップでは無効）
  • 末尾ステップ「下へ移動」は非表示（末尾ステップでは無効）
  • VoiceOver ラベル各ステップ行に .accessibilityLabel("\(stepName)、\(order)番目") を付与し、現在の順番をフォーカス時に読み上げ

  • グリッドLazyVGrid 4列 + spacing DesignsSpacing.sm
  • 選択肢3 / 5 / 8 / 10 / 15 / 20 / 30 秒
  • 選択時background accent orange + 白テキスト

保存ロジック

保存ボタンタップで handleSaveTapped() → 削除ステップのクリップ確認 → save()（差分リコンサイル方式・e216de7）

  1. ルーティン名 trimming + 空チェック（保存ボタン無効化）
  2. 各ステップ名の空チェック（1件でも空ならば保存ボタン無効化・空テロップ動画の生成防止）
  3. reminderTime から hour/minute 抽出
  4. 新規作成 or 既存編集の分岐
  5. 既存編集時: 差分リコンサイル（詳細は下記セクション参照）
  6. 削除ステップのクリップが 1 件以上ある場合のみ警告 alert（他ステップのクリップ保持を明示）
  7. ReminderService.scheduleReminder() を Task で実行
  8. dismiss()

差分リコンサイル（e216de7 で全面改訂）

v1.1 以前は stepsChanged computed property でステップ構成の変化を一括検出し、変化があれば全クリップを削除していた。e216de7 で差分リコンサイル方式に変更し、ユーザーが削除したステップのクリップのみを削除するようにした。

  • 既存ステップ（originalStepID 一致）in-place 更新。名前修正・秒数変更・並び替えが行われてもクリップは一切削除しない
  • 削除ステップ（originalStepID が editableSteps に存在しない）そのステップのクリップのみ物理削除
  • 新規追加ステップ（originalStepID == nil）新規 Step エンティティを insert（クリップはまだ存在しないため削除処理なし）
  • クリップ保持操作ステップ名変更・秒数変更・並び替え・ステップ追加はすべてクリップを保持
  • クリップ削除操作ステップ削除 ×（バツボタン）のみ。削除後に保存したときに初めてそのステップのクリップが物理削除される

クリップ削除警告（e216de7 で精緻化）

削除するステップに紐づくクリップが 1 件以上ある場合のみ alert を表示する。全クリップ削除の旧文言を、削除されるクリップ数と「他は保持」の旨を明示した文言に更新した。

  • 文言「削除するステップに紐づく〇件のクリップが削除されます。他のステップのクリップは保持されます。続けますか？」
  • ボタン「削除して更新」(destructive) / 「キャンセル」
  • 表示条件削除対象ステップのクリップ数 ≥ 1。削除ステップがなければ alert なしで即時 save()

EditableStep（編集用一時データ）

struct EditableStep: Identifiable {
    let id = UUID()            // SwiftUI List 用識別子（UI 専用・永続化しない）
    var originalStepID: UUID?  // 元 Step の PersistentIdentifier（既存ステップ編集時のみ設定・差分リコンサイルで使用）
    var name: String
    var durationSeconds: Int
    var order: Int
}

保存時に Step エンティティへ変換。並べ替え/削除中に SwiftData オブジェクトを触らないため安全。originalStepID を参照して既存ステップの in-place 更新と削除ステップのクリップ削除を制御する（e216de7）。

空ステップ名バリデーション（e216de7 追加）

ステップ名が空（trimmingCharacters(.whitespaces) 後に空文字列）のステップが 1 件でも存在する場合、保存ボタンを無効化する。これにより空テロップが焼き込まれた動画の生成を防止する。

  • バリデーション対象各 EditableStep.name.trimmingCharacters(.whitespaces).isEmpty
  • 保存ボタン条件ルーティン名が非空 AND 全ステップ名が非空 の場合のみ有効
  • ユーザー影響空のステップ名 TextField にフォーカス中はキーボード表示のため保存ボタンが自然に遠ざかる（ガードとして機能）
