ルーティン編集（RoutineEditView）

新規ルーティン作成・既存ルーティン編集の両用画面。v1.0で Home UI に完全統一。

基本情報

  • 画面IDroutine-edit
  • 実装ファイルViews/Home/RoutineEditView.swift
  • 表示方式sheet（モーダル）
  • 起動経路①Quick Start カード ②マイルーティン → 編集する ③プリセット選択 → カスタム/プリセット選択 ④通知タップ → 編集メニュー
  • 背景色#FAF9F7 (warm white)

画面構成

  1. 名前: ルーティン名 TextField（カード）
  2. テロップのフォント: 横スクロールフォントチップ（無料2種 + Pro4種）
  3. 撮影リマインダー: Toggle + DatePicker（有効時のみ展開）
  4. ステップ: 縦並びカード（最大5個まで・ドラッグ並べ替え対応）
  5. ステップを追加: ダッシュ枠ボタン（accent orange）
  6. ルーティンを削除: 編集時のみ表示・赤opacity 0.75

各セクションの実装

共通: sectionLabel

uppercase + tracking 0.4 + tertiary text のヘッダー（SettingsView と統一）

名前セクション

  • 入力欄TextField「ルーティン名」(16pt regular) + designsCard + DesignsRadius.md (16pt) + designsCardShadow
  • 空チェックtrimmingCharacters(.whitespaces) で空判定、保存ボタン無効化

フォントセクション

  • 無料HelveticaNeue-Light / Inter
  • ProCaveat / Shadows Into Light / Playfair Display / Noto Serif JP（lock アイコン）
  • チップサイズ76×68pt cornerRadius DesignsRadius.sm
  • 選択時背景 accent orange + 白テキスト
  • 未選択背景 designsCard + チャコールテキスト
  • Pro 限定タップPaywallView 起動

リマインダーセクション

  • Toggle「毎日通知する」+ bell.fill アイコン（有効時 accent orange）
  • DatePickerToggle 有効時のみ展開 + Divider（アイコンインデント分パディング）
  • 保存時ReminderService.shared.scheduleReminder() を Task で実行

ステップセクション

  • ヘッダー「ステップ」+ 件数バッジ（右寄せ）
  • 行構成ドラッグハンドル + 番号(monospaced) + ステップ名 TextField + 秒数 Capsule + 削除 x
  • ドラッグ並べ替え.draggable + .dropDestination で並び替え対応
  • 秒数タップDurationPickerSheet 起動（presentationDetents [.height(280)]）
  • 削除アニメwithAnimation(.spring(duration: 0.25))

ステップ追加ボタン

  • スタイルaccent orange + 8% bg + ダッシュ枠 lineWidth 1 dash [4]
  • 課金壁無料5ステップまで（maxFreeSteps = 5）、6個目で Paywall 起動
  • アニメwithAnimation(.spring(duration: 0.3)) で追加

削除ボタン（編集時のみ）

  • 表示条件isEditing == true 時のみ
  • 色.red opacity 0.75（純赤を避ける）
  • 確認 alert「ルーティンを削除しますか？」+「この操作は取り消せません。」
  • 削除処理ReminderService.cancelReminder() → modelContext.delete(routine)

DurationPickerSheet（秒数ピッカー）

  • 表示方式.sheet + presentationDetents([.height(280)]) + dragIndicator
  • 背景Color.designsBackground (v1.0で黒→warm white)
  • グリッドLazyVGrid 4列 + spacing DesignsSpacing.sm
  • 選択肢3 / 5 / 8 / 10 / 15 / 20 / 30 秒
  • 選択時background accent orange + 白テキスト

保存ロジック

保存ボタンタップで handleSaveTapped() → クリップ存在チェック → save()

  1. name trimming + 空チェック
  2. reminderTime から hour/minute 抽出
  3. 新規作成 or 既存編集の分岐
  4. 既存編集時: 紐づくクリップ・ステップをスナップショット化してから削除（並行更新防止）
  5. 各 EditableStep から Step エンティティ生成 + modelContext.insert
  6. ReminderService.scheduleReminder() を Task で実行
  7. dismiss()

クリップ削除警告

既存ルーティンが紐づくクリップを持っている場合のみ alert を挟む:

  • 文言「このルーティンには〇件のクリップが保存されています。ステップを更新するとクリップライブラリがすべて削除されます。続けますか？」
  • ボタン「更新する」(destructive) / 「キャンセル」

EditableStep（編集用一時データ）

struct EditableStep: Identifiable {
    let id = UUID()
    var name: String
    var durationSeconds: Int
    var order: Int
}

保存時に Step エンティティへ変換。並べ替え/削除中に SwiftData オブジェクトを触らないため安全。
