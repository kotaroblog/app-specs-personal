クリップライブラリ（ClipLibraryView）

ルーティンに紐づく全クリップをステップごとに表示。動画生成への導線を Home UI 統一の CTA で提供。

基本情報

  • 画面IDclip-library
  • 実装ファイルViews/Library/ClipLibraryView.swift
  • 表示方式sheet（NavigationStack内 / item-based）
  • 起動経路ルーティンメニュー → クリップライブラリを開く / クリップタブ
  • 背景色#FAF9F7

画面構成

  1. ShootingStyleTip（TipKit）（v1.1.0追加・最大3回表示）
  2. ステップセクション（ルーティンの sortedSteps を順に展開）
  3. 動画を作るCTA（クリップが1件以上ある時のみ）

ShootingStyleTip（TipKit・v1.1.0）

オンボーディングから削除された「2つの撮影スタイル解説」をクリップライブラリ画面で文脈表示する TipKit ヒント。初めてクリップライブラリを開いた際に表示され、連続撮影と単発撮影の違いを案内する。

  • 表示位置画面最上部（ステップ一覧より上）
  • 表示回数上限最大3回（maxDisplayCount = 3）
  • 内容「連続撮影」（全ステップを一気に撮るモード）と「クリップライブラリ」（ステップ別に1本ずつ撮るモード）の使い分けを案内
  • 却下× ボタンで閉じる（以後カウントは継続し上限到達で自動非表示）
  • 関連オンボーディングフロー（旧 Page 1 削除の経緯）

ステップセクション

  • ヘッダーステップ名（15pt semibold） + 「· 5秒」（11pt light tertiary）+ Spacer + 撮影 Capsuleボタン
  • 撮影ボタンaccent orange + capsule背景 opacity 0.10 + plus アイコン
  • クリップ一覧横スクロール LazyHStack・spacing DesignsSpacing.sm
  • 空状態film アイコン + 「まだクリップがありません」カード（Color.designsCard + designsCardShadow）

クリップサムネ（v1.0 で拡大）

  • サイズ92×138pt（旧 80×120pt）
  • cornerRadiusDesignsRadius.xl (22pt)（旧 8pt）
  • shadowdesignsCardShadow
  • タップClipPlayerView 起動
  • 日時表記時刻のみ（HH:MM）。月日は親セクションで自明なため省略
  • contextMenu削除のみ

「動画を作る」CTA（Home CTA 統一）

  • 背景LinearGradient pink → orange + RadialGradient 白28%
  • 高さ54pt
  • cornerRadiusDesignsRadius.xl (22pt)
  • shadowaccent pink 28% / radius 12 / y 5
  • アイコンsparkles 13pt
  • 文言動画を作る（15pt semibold）
  • 押下DesignsPressableButtonStyle scale 0.97
  • 遷移先ClipGenerateView（fullScreenCover）

削除処理

  1. pendingDeleteClip に Clip セット → alert 表示
  2. 「削除」確定で ClipStorageService.deleteClip()（ファイル + サムネ削除）
  3. modelContext.delete(clip) で SwiftData レコード削除

ClipRoutineSelectSheet（596896d）

クリップタブでルーティンが 2 件以上存在する場合に表示するルーティン選択シート。暗黙の先頭選択（旧挙動）を廃止し、ユーザーに明示的に選ばせる。

  • 実装DesignsHomeView.swift 内の private struct（ClipRoutineSelectSheet）
  • 表示方式sheet（showClipRoutineSelect フラグで制御）
  • ルーティン行アイコン（symbolName または自動推薦）+ 名前 + 「Xステップ・Yクリップ」表示 + chevron.right
  • 選択onSelect(routine) コールバック → pendingClipLibraryRoutine にセット → シート dismiss 後 handleClipRoutineSelectorDismissed() で ClipLibraryView 起動
  • 新規作成「新しいルーティンを作成」ボタン → onCreateRoutine() → shouldCreateRoutineAfterClipSelector = true → dismiss 後 handleCenterPlusTap() 経由
  • 閉じる左上 × ボタン（xmark icon）

ClipGenerateView 選択状況カード（596896d）

動画生成時にどのステップのクリップが選択済みかを一目で把握できるカードを画面上部に追加。

  • 表示「選択済み X/Y」テキスト + 状態ラベル（「すべて選択済み」または「未選択あり」）
  • プログレスバーProgressView（value: selectedStepCount / total: totalStepCount）/ tint: accentOrange
  • 詳細テキスト選択0件 → 「少なくとも1つのクリップを選んでください」/ 全選択 → 「全ステップのクリップを使って動画を作成」/ 一部未選択 → 未選択ステップ名（最大3件）を列挙
  • 生成ボタン文言分岐全選択済み → 「動画を作成する」 / 一部のみ → 「選択したクリップで作成」
  • アクセシビリティ.accessibilityElement(children: .combine) + .accessibilityLabel（選択数 + 詳細テキスト）

ClipGenerateView: 空状態「撮影する」ボタン・二重タップ防止（v1.1.0・UI/UX改善バッチ）

  • 未撮影ステップの空状態従来はテキストのみ表示だったが「撮影する」ボタンを追加し、単発撮影（SingleStepShootingView）を直接起動できるようになった
  • 「動画を作成する」二重タップ防止ボタン押下後に即時 disabled 化し、合成処理の多重起動を防止

CreateSessionView: プレビュー準備失敗時のフォールバック（v1.1.0・UI/UX改善バッチ）

プレビュー準備に失敗した場合、従来は行き止まりテキストのみ表示されていたが、「クリップ選択に戻る」ボタンを追加して回復経路を明示した。

  • 表示条件CreateSessionView でプレビュー準備が失敗した状態
  • ボタン文言「クリップ選択に戻る」
  • 動作NavigationStack を pop して ClipLibraryView（クリップ選択）に戻る

作成セッション集約（CreateSessionView / 2026-06-06）

  • 実装ファイルViews/Library/CreateSessionView.swift
  • 役割ClipLibrary から起動する「撮影 → プレビュー → 動画生成 → エクスポート完了」の一連フローを 1 つのセッションコンテナに集約
  • 構成NavigationStack(path:) + ShootingView 用 fullScreenCover を内包。プレビュー / 生成 / エクスポート完了 は path に push する遷移へ統一
  • ShootingView 完了時PreviewView 起動責務を CreateSessionView へ移管（旧: ShootingView 内で直接 push）。撮影完了 → PreviewView 自動 push を CreateSessionView 側で制御
  • ClipLibraryView の責務縮小ClipGenerate / Preview / ExportComplete を直接保持せず、CreateSessionView へコールバック駆動で委譲。ClipLibrary 自身は一覧表示と撮影起動・動画作成起動の発火のみ担当
  • 「全ステップを連続撮影」ボタンCreateSessionView 経由のセッション起点として ClipLibrary 冒頭に追加。ステップ未撮影分を順に巡回する撮影フローを開始

ClipPlayerView トリムハンドルタップ領域 44pt 化（v1.1.0）

トリムハンドルの視覚サイズは 28pt だが、HIG 準拠の最小タップターゲット 44pt を確保するため .contentShape(Rectangle()) によりヒット領域を拡張した。

  • 視覚サイズ28pt（変更なし・デザイン維持）
  • ヒット領域44pt（.contentShape(Rectangle().size(CGSize(width: 44, height: 44)))）
  • 対象開始ハンドル（左端）/ 終了ハンドル（右端）の両方

トリム未保存変更の破棄確認（v1.1.0）

トリム変更を行った状態で閉じる操作（閉じるボタン / 下スワイプ）を行った場合、未保存変更の破棄確認ダイアログを表示する。

  • 発火条件トリム開始点・終了点を変更した状態（hasUnsavedTrimChanges == true）で閉じる / 下スワイプを実行
  • ダイアログタイトル「変更を破棄しますか？」
  • ダイアログメッセージ「保存されていないトリム変更は失われます。」
  • ボタン「破棄する」（role: .destructive）/ 「キャンセル」（撮影に戻る）
  • 変更前未保存変更があっても確認なしで閉じられ、変更が失われていた
