クリップライブラリ（ClipLibraryView）

ルーティンに紐づく全クリップをステップごとに表示。動画生成への導線を Home UI 統一の CTA で提供。

基本情報

  • 画面IDclip-library
  • 実装ファイルViews/Library/ClipLibraryView.swift
  • 表示方式sheet（NavigationStack内 / item-based）
  • 起動経路ルーティンメニュー → クリップライブラリを開く / クリップタブ
  • 背景色#FAF9F7

画面構成

  1. ステップセクション（ルーティンの sortedSteps を順に展開）
  2. 動画を作るCTA（クリップが1件以上ある時のみ）

ステップセクション

  • ヘッダーステップ名（15pt semibold） + 「· 5秒」（11pt light tertiary）+ Spacer + 撮影 Capsuleボタン
  • 撮影ボタンaccent orange + capsule背景 opacity 0.10 + plus アイコン
  • クリップ一覧横スクロール LazyHStack・spacing DesignsSpacing.sm
  • 空状態film アイコン + 「まだクリップがありません」カード（Color.designsCard + designsCardShadow）

クリップサムネ（v1.0 で拡大）

  • サイズ92×138pt（旧 80×120pt）
  • cornerRadiusDesignsRadius.xl (22pt)（旧 8pt）
  • shadowdesignsCardShadow
  • タップClipPlayerView 起動
  • 日時表記時刻のみ（HH:MM）。月日は親セクションで自明なため省略
  • contextMenu削除のみ

「動画を作る」CTA（Home CTA 統一）

  • 背景LinearGradient pink → orange + RadialGradient 白28%
  • 高さ54pt
  • cornerRadiusDesignsRadius.xl (22pt)
  • shadowaccent pink 28% / radius 12 / y 5
  • アイコンsparkles 13pt
  • 文言動画を作る（15pt semibold）
  • 押下DesignsPressableButtonStyle scale 0.97
  • 遷移先ClipGenerateView（fullScreenCover）

削除処理

  1. pendingDeleteClip に Clip セット → alert 表示
  2. 「削除」確定で ClipStorageService.deleteClip()（ファイル + サムネ削除）
  3. modelContext.delete(clip) で SwiftData レコード削除
