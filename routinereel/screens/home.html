ホーム画面（DesignsHomeView）

v1.0 から導入される新ホーム画面。FeatureFlag で旧 HomeView と切替可能。

基本情報

  • 画面IDhome
  • 実装ファイルViews/Designs/DesignsHomeView.swift
  • 導入バージョンv1.0
  • FeatureFlaguseDesignHome（デフォルト true）
  • 旧画面Views/Home/HomeView.swift（FeatureFlag false 時に表示）
  • 背景色#FAF9F7（朝の光感）

画面構成

  • ①上部バーアプリ名「RoutineReel」+ Pro バッジ（無料時はPaywall起動・Pro時は表示のみ）+ 設定アイコン
  • ②Heroセクション時間帯別挨拶 + 🔥連続日数 + 日付 + メインコピー + 感情コピー（時間帯別） + 「次に撮影」CTA
  • ③Quick Start横スクロール4プリセット（morning / night / workout / study）175×240pt カード
  • ④マイルーティン縦並びカード（タップで confirmationDialog 起動）+ 新規ルーティン作成ボタン
  • ⑤最近の動画VideoListView への導線（横スクロールサムネ + 「すべて見る」）
  • ⑥下部バナー広告（無料時のみ）+ 5タブバー + 中央＋ボタン

遷移一覧（7遷移）

トリガー遷移先表示方法
設定アイコンSettingsViewsheet
中央＋ / Quick Start / 新規ルーティン作成ボタンPresetSelectViewsheet
ルーティンカード → 編集するRoutineEditViewsheet（item-based）
ルーティンカード → 全ステップ続けて撮影 / 「次に撮影」CTAShootingViewfullScreenCover（item-based）
ルーティンカード → クリップライブラリを開くClipLibraryViewsheet（item-based / NavigationStack 内）
最近の動画 → すべて見る / サムネタップVideoListViewsheet（NavigationStack 内）
Pro バッジ（無料時）PaywallViewsheet

課金壁とサブスク連携

  • 無料ルーティン上限2件まで（maxFreeRoutines = 2）
  • 3件目作成時PaywallView 自動起動
  • 複製時の課金壁無料上限到達中は Paywall 起動して複製拒否
  • バナー広告無料時のみタブバー上に常時表示（高さ50pt）
  • Pro バッジPro時は非インタラクティブ（情報のみ）、無料時は Paywall 起動ボタン

通知ルーティング

リマインダー通知タップで起動された場合、NotificationRouter.shared.pendingRoutineId を監視し、 該当ルーティンの撮影画面（ShootingView）を fullScreenCover で起動する。

  • トリガー.onAppear および .onChange(of: pendingRoutineId)
  • 二重発火防止oldValue != nil ガードで nil リセット時の不要再発火を防止
  • 処理後pendingRoutineId = nil でクリア

マイクロアニメーション

  • Hero フェードインopacity 0→1 + offset 6→0 を easeOut 0.45秒、感情コピーは 0.15秒遅延、CTA は 0.25秒遅延の spring
  • Quick Start カードappear時 spring(0.55, 0.85) でフェード+スライドアップ
  • 中央＋ボタン静的ハロー（pink 0.15）+ 押下時 scale 0.90 spring
  • 「次に撮影」CTA押下時 scale 0.97 spring、ハロー・脈動なし

ロールバック方式

リリース後に新Home UIで不具合が発生した場合、FeatureFlag運用 経由でユーザー自身が 設定画面の「β機能 → 新しいホーム画面を使う」トグルを OFF にすれば、旧 HomeView に即時切替可能。 コード変更・再申請は不要。

v1.1 で予定される改善

  • 連続日数の実データ算出 — 現在は mockStreakDays = 7 固定
  • DesignsHomeView の200行ルール準拠 — 現在525行。サブView分割で対応予定
  • マイルーティン「すべて見る」展開画面 — 現在は3件まで表示で省略
  • DesignsRecordPreviewView の本番接続 — 撮影画面の完全リデザイン
