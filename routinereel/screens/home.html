ホーム画面（DesignsHomeView）

v1.0 から導入される新ホーム画面。FeatureFlag で旧 HomeView と切替可能。

基本情報

  • 画面IDhome
  • 実装ファイルViews/Designs/DesignsHomeView.swift
  • 導入バージョンv1.0
  • FeatureFlaguseDesignHome（デフォルト true）
  • 旧画面Views/Home/HomeView.swift（FeatureFlag false 時に表示）
  • 背景色#FAF9F7（warm white、朝の光感）

画面構成

  • ①上部バーアプリ名「RoutineReel」+ Pro バッジ（Pro時は表示のみ・無料時はPaywall起動）+ 設定アイコン
  • ②Heroセクションメインコピー（streak 状況で 3 パターン分岐）+ サブコピー + streak / 累計バッジ +「次に撮影」CTA。詳細は Hero streak メーター 参照
  • ③Quick Start横スクロール4プリセット（朝5分ルーティン / 夜の整いルーティン / 筋トレ10分ルーティン / 集中45分ルーティン）150×220pt カード（v1.0.6で画面幅2枚並びの見切れを解消）。タップで 直接 RoutineEditView 起動。v1.0 で プリセット名を検索意図ワードへ刷新（raw value は不変）
  • ④マイルーティン縦並びカード（タップで confirmationDialog 起動）+ 新規ルーティン作成ボタン。shotToday が true のときのみ chevron 手前に「今日 ✓」緑バッジを表示
  • ⑤最近の動画recentVideos が存在する場合のみ表示。実サムネ画像（VideoStorageService.thumbnailURL）+ 中央 play.circle.fill オーバーレイ。タップで RecentVideoPlayerSheet を fullScreenCover 起動。見出し横に 保存メーター
  • ⑥下部バナー広告（無料時のみ）+ 5タブバー + 中央＋ボタン

ルーティンメニュー（confirmationDialog）

マイルーティンカードタップ時のアクションシート 5項目:

  1. 全ステップ続けて撮影 → ShootingView
  2. クリップライブラリを開く → ClipLibraryView
  3. 編集する → RoutineEditView
  4. 複製する → duplicate(routine:) で実行
  5. 削除する（role: .destructive）→ 削除確認 alert → deleteRoutine() で物理ファイル削除 + cascade delete

課金壁とサブスク連携

  • 無料ルーティン上限2件まで（maxFreeRoutines = 2）
  • 3件目作成時PaywallView 自動起動
  • 複製時の課金壁無料上限到達中は Paywall 起動して複製拒否
  • Quick Start タップ時の課金壁同上、Paywall起動
  • バナー広告無料時のみタブバー上に常時表示（高さ50pt）
  • Pro バッジPro時は非インタラクティブ（情報のみ）、無料時は Paywall 起動ボタン

削除処理（deleteRoutine）

ルーティン削除時の物理ファイル + DBレコードの安全な削除順序:

  1. Array(routine.clips) スナップショット化（並行更新防止）
  2. 各 Clip の物理ファイル削除（ClipStorageService.deleteClip）
  3. Array(routine.videos) スナップショット化
  4. 各 Video の物理ファイル削除（VideoStorageService.deleteVideo）
  5. modelContext.delete(routine) で SwiftData cascade 削除

削除確認 alert で紐づくクリップ数・動画数をユーザーに提示。

通知ルーティング

リマインダー通知タップで起動された場合、NotificationRouter.shared.pendingRoutineId を監視し、該当ルーティンの撮影画面（ShootingView）を fullScreenCover で起動する。

  • トリガー.onAppear および .onChange(of: pendingRoutineId)
  • 二重発火防止oldValue != nil ガードで nil リセット時の不要再発火を防止
  • 処理後pendingRoutineId = nil でクリア

Hero streak メーター（v1.0 / 2026-06-10）

routines.flatMap(\$0.clips) を横断し、「今日 or 昨日に撮影記録あれば streak 継続、なければ 0 リセット」のルールで連続日数を算出してヒーロー上部に表示する。

Routine モデル拡張

  • lastShotAtcomputed property。routine.clips.map(\.recordedAt).max()
  • shotTodaycomputed property。Calendar.current.isDateInToday(lastShotAt)
  • currentStreakcomputed property。単一 Routine の連続日数
  • computeStreak(from:) static複数 Routine 横断 helper。DesignsHomeView の overallStreak / totalVideoCount / overallHasShotToday から呼び出す

表示パターン（3分岐）

条件バッジメインコピーサブコピー
記録ゼロ（overallStreak == 0 かつ totalVideoCount == 0）なし今日もルーティンを記録しよう5分だけでも。
streak ≥ 1 かつ overallHasShotToday == true🔥 N日連続 / 📹 累計 N 本連続 N 日、続いてます今日もお疲れさま。
streak ≥ 1 かつ overallHasShotToday == false🔥 N日連続 / 📹 累計 N 本連続 N 日、続いてます今日も1本、撮りましょう。

Quick Start プリセット名刷新 + URL スキーム（v1.0 / 2026-06-10）

プリセット表示名（displayName）

raw value（永続化キー・URL パス）旧表記新表記（検索意図ワード）
morningモーニングルーティン朝5分ルーティン
nightナイトルーティン夜の整いルーティン
workoutワークアウト筋トレ10分ルーティン
study勉強ルーティン集中45分ルーティン

ステップ名も日本語化（wake up→起きる, skincare→スキンケア, go out→出発, warm up→ウォームアップ ほか）。raw value は永続化キー・URL scheme パスとして不変。

URL スキーム消費フロー

  • スキームroutinereel://preset/<rawValue>（project.yml + Info.plist の CFBundleURLTypes に登録）
  • 受け口RoutineReelApp.body.onOpenURL でパースして AppSettings.pendingPresetRawValue に格納
  • 消費DesignsHomeView の .onAppear および .onChange(of: settings.pendingPresetRawValue) で consumePendingPresetIfAny 実行。同名 Routine が既に存在すれば再利用、無ければ新規作成し activeSession を起動（fresh start / 既存セッション 両対応）

マイクロアニメーション

  • Hero フェードインopacity 0→1 + offset 6→0 を easeOut 0.45秒、感情コピーは 0.15秒遅延、CTA は 0.25秒遅延の spring
  • Quick Start カードappear時 spring(0.55, 0.85) でフェード+スライドアップ
  • 中央＋ボタン静的ハロー（pink 0.15）+ 押下時 scale 0.90 spring
  • 「次に撮影」CTA押下時 scale 0.97 spring、ハロー・脈動なし

常時アニメーションは意図的に最小化（GPT評価「アニメーション頑張った感」回避のため）。

ロールバック方式

リリース後に新Home UIで不具合が発生した場合、FeatureFlag運用 経由でユーザー自身が設定画面の「β機能 → 新しいホーム画面を使う」トグルを OFF にすれば、旧 HomeView に即時切替可能。コード変更・再申請は不要。
