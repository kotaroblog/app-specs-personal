ホーム画面（DesignsHomeView）

v1.0 から導入される新ホーム画面。FeatureFlag で旧 HomeView と切替可能。

基本情報

  • 画面IDhome
  • 実装ファイルViews/Designs/DesignsHomeView.swift
  • 導入バージョンv1.0
  • FeatureFlaguseDesignHome（デフォルト true）
  • 旧画面Views/Home/HomeView.swift（FeatureFlag false 時に表示）
  • 背景色#FAF9F7（warm white、朝の光感）

画面構成

  • ①上部バーアプリ名「RoutineReel」+ Pro バッジ（Pro時は表示のみ・無料時はPaywall起動）+ 設定アイコン
  • ②Heroセクションメインコピー「今日もルーティンを記録しよう。」+ 時間帯別感情コピー +「次に撮影」CTA
  • ③Quick Start横スクロール4プリセット（morning/night/workout/study）150×220pt カード（v1.0.6で画面幅2枚並びの見切れを解消）。タップで 直接 RoutineEditView 起動
  • ④マイルーティン縦並びカード（タップで confirmationDialog 起動）+ 新規ルーティン作成ボタン
  • ⑤最近の動画recentVideos が存在する場合のみ表示。横スクロールサムネ +「すべて見る」
  • ⑥下部バナー広告（無料時のみ）+ 5タブバー + 中央＋ボタン

ルーティンメニュー（confirmationDialog）

マイルーティンカードタップ時のアクションシート 5項目:

  1. 全ステップ続けて撮影 → ShootingView
  2. クリップライブラリを開く → ClipLibraryView
  3. 編集する → RoutineEditView
  4. 複製する → duplicate(routine:) で実行
  5. 削除する（role: .destructive）→ 削除確認 alert → deleteRoutine() で物理ファイル削除 + cascade delete

課金壁とサブスク連携

  • 無料ルーティン上限2件まで（maxFreeRoutines = 2）
  • 3件目作成時PaywallView 自動起動
  • 複製時の課金壁無料上限到達中は Paywall 起動して複製拒否
  • Quick Start タップ時の課金壁同上、Paywall起動
  • バナー広告無料時のみタブバー上に常時表示（高さ50pt）
  • Pro バッジPro時は非インタラクティブ（情報のみ）、無料時は Paywall 起動ボタン

削除処理（deleteRoutine）

ルーティン削除時の物理ファイル + DBレコードの安全な削除順序:

  1. Array(routine.clips) スナップショット化（並行更新防止）
  2. 各 Clip の物理ファイル削除（ClipStorageService.deleteClip）
  3. Array(routine.videos) スナップショット化
  4. 各 Video の物理ファイル削除（VideoStorageService.deleteVideo）
  5. modelContext.delete(routine) で SwiftData cascade 削除

削除確認 alert で紐づくクリップ数・動画数をユーザーに提示。

通知ルーティング

リマインダー通知タップで起動された場合、NotificationRouter.shared.pendingRoutineId を監視し、該当ルーティンの撮影画面（ShootingView）を fullScreenCover で起動する。

  • トリガー.onAppear および .onChange(of: pendingRoutineId)
  • 二重発火防止oldValue != nil ガードで nil リセット時の不要再発火を防止
  • 処理後pendingRoutineId = nil でクリア

マイクロアニメーション

  • Hero フェードインopacity 0→1 + offset 6→0 を easeOut 0.45秒、感情コピーは 0.15秒遅延、CTA は 0.25秒遅延の spring
  • Quick Start カードappear時 spring(0.55, 0.85) でフェード+スライドアップ
  • 中央＋ボタン静的ハロー（pink 0.15）+ 押下時 scale 0.90 spring
  • 「次に撮影」CTA押下時 scale 0.97 spring、ハロー・脈動なし

常時アニメーションは意図的に最小化（GPT評価「アニメーション頑張った感」回避のため）。

ロールバック方式

リリース後に新Home UIで不具合が発生した場合、FeatureFlag運用 経由でユーザー自身が設定画面の「β機能 → 新しいホーム画面を使う」トグルを OFF にすれば、旧 HomeView に即時切替可能。コード変更・再申請は不要。
