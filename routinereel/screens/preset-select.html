プリセット選択（PresetSelectView）

新規ルーティン作成時の入口。プリセットから始めるかカスタムを選ぶ。v1.0で Home UI 統一 + 日本語化。

基本情報

  • 画面IDpreset-select
  • 実装ファイルViews/Home/PresetSelectView.swift
  • 表示方式sheet（モーダル）
  • 起動経路中央＋ボタン / 新規ルーティン作成ボタン
  • 背景色#FAF9F7 (warm white)

画面構成

  1. タイトル: 「ルーティンを選ぶ」（26pt bold）+ サブコピー「プリセットから始めるか、自分で作成できます。」
  2. プリセット4種: morning / night / workout / study（カード形式）
  3. 自分で作成: ダッシュ枠 + accent orange の独立ボタン

raw value 不変ポリシー

  • raw valuemorning / night / workout / study は永続化キー兼 URL スキームパスとして不変
  • 表示文言displayName / ステップ名はユーザー向け文言として日本語化（raw value とは分離）
  • URL スキームroutinereel://preset/<rawValue> で外部から起動可能（例: routinereel://preset/morning）

状態管理（v1.1 で単一 enum 管理に変更）

v1.1 以前は selectedPreset: RoutinePreset? と showEditView: Bool の二変数で管理していたが、初回タップ時に状態が競合して「プリセットタップで登録画面に入る」バグが発生していた。v1.1 で editTarget: EditTarget? の単一 enum 管理に変更し解消。

  • EditTargetenum EditTarget: Identifiable { case preset(RoutinePreset); case custom }
  • sheet 起動editTarget を .sheet(item:) に渡す（nil = 非表示 / non-nil = 表示）
  • 旧メソッドcreateFromPreset(_:) → 廃止
  • 新メソッドpresentEditor(for:)（sheet 起動の意図を明示）

選択フロー（v1.1 以降）

1. ユーザーがプリセット選択（または自分で作成）
2. presentEditor(for: preset) → editTarget = .preset(preset)
3. .sheet(item: $editTarget) が起動
4. RoutineEditView(routine: nil, preset: preset) を表示
5. RoutineEditView.loadData() で preset.displayName と preset.steps を初期値にセット
6. ユーザーが編集して保存 → 新規 Routine 作成

RoutinePreset と統合（v1.0で日本語化）

各プリセット情報は Models/RoutinePreset.swift に集約。PresetSelectView 内でローカルヘルパー（presetEmoji / presetTitleJa）を持っていたが、コードレビューで重複指摘 → 削除し RoutinePreset のプロパティを直接参照する形に整理。

// 旧: PresetSelectView 内のローカル関数
private func presetEmoji(_ preset: RoutinePreset) -> String { ... }
private func presetTitleJa(_ preset: RoutinePreset) -> String { ... }

// 新: RoutinePreset 直下のプロパティ
preset.emoji         // "☀️"
preset.displayName   // "朝5分ルーティン"  (v1.0.7 で検索意図ワードに刷新)
