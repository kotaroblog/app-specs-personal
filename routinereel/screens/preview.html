プレビュー・書き出し（PreviewView）

動画合成後のプレビューと2種類の保存ボタンを提供する画面。ClipGenerateView から NavigationStack に push される。

基本情報

  • 画面IDpreview
  • 実装ファイルViews/Preview/PreviewView.swift
  • 表示方式NavigationStack push（CreateSessionView 管理下。自前 NavigationStack を持たない）
  • 起動経路ClipGenerateView（「動画を作成する」ボタン）→ 動画合成完了後に自動 push
  • 背景色#FAF9F7（warm white）
  • colorScheme.light 固定（warm white 背景のステータスバー文字色対応）

画面構成

  1. トップバー: 「← 戻る」ボタン（左）+ 「プレビュー」タイトル（中央）
  2. 動画プレビュー領域: 9:16 縦型・黒背景・warm white 上に黒カードとして浮かせる。build69 より上詰め配置（余白を下側に集約）
  3. 元音声ミュートトグル: Toggle + speaker アイコン
  4. フィルタセクション: 横スクロールチップ（無料フィルタ + Pro フィルタ）
  5. 書き出しボタン: カメラロール保存（主役・グラデ）+ アプリ内保存（脇役・アウトライン）

動画プレビュー領域（build69 更新）

  • アスペクト比9:16 固定（縦型動画専用）
  • 配置（build69）上詰め（.frame(maxWidth: .infinity, maxHeight: .infinity, alignment: .top)）。以前は上下中央寄せで動画が小さく浮いて見えていたため、上部余白を削減して動画をトップバー直下まで引き上げ、余白を下側（ミュートトグル・フィルタ・ボタン間）に集約
  • cornerRadius16pt（.continuous）
  • 全画面表示動画右上の拡大ボタンタップで FullScreenVideoView（fullScreenCover）を起動。同一 AVPlayer インスタンスを共有し再生位置を維持

動画プレビュー領域の状態分岐

状態表示内容
.composing動画作成中プログレス（%表示 + バー + キャンセルボタン）
.readyAVPlayer 動画再生・タップで再生/一時停止・全画面ボタン
.exported同上（保存済み状態。戻るボタンでアラートなしで閉じる）
.exportingカメラロール保存中プログレス（動画 opacity 0.3 の上にオーバーレイ。キャンセル不可）
.error(message, recovery)エラーメッセージ + リカバリボタン（recompose / retrySave / openSettings の 3 種）
.idleEmptyView（遷移直後の一瞬）

書き出しボタン（build69 更新）

build69 より各ボタンに用途の副説明を追加し、2種類の保存の違いを初見でも理解できるよう改善した。

ボタンスタイル副説明（build69 追加）無料枠消費
カメラロールに保存（主役）pink→orange グラデ・58pt「TikTokやInstagramへの投稿用」消費しない
アプリに保存（脇役）アウトライン・55pt「RoutineReel内であとから再生できます」消費する（無料3本まで）
  • 無効化条件state が .composing / .exporting / .idle / .error 時（isReadyToSave = false）は両ボタン無効化・opacity 0.45
  • アプリ内保存ボタンのラベルPro 時は「アプリに保存」、無料かつ残量あり時は「アプリに保存（残り N 件）」、無料かつ上限到達時は「アプリに保存（Proで無制限）」→ タップで Paywall 起動
  • アプリ内保存完了アラート「アプリに保存しました」タイトル + 「動画はホーム画面の『アプリ内の動画』から確認できます。」メッセージ（build69 より「最近の動画」→「アプリ内の動画」に更新）
  • 二重保存ガードPreviewViewModel 内で保存済み (sessionId, destination) ペアを Set で保持し、同一セッション・同一 destination での重複実行を防止（Analytics の重複送信防止も兼ねる）
  • カメラロール保存完了後親（CreateSessionView）の onSavedToPhotos コールバック経由で ExportCompleteView を push

VideoCompletion 記録

カメラロール保存・アプリ内保存のどちらが先に成功しても VideoCompletionRecorder.recordIfNeeded により1件のみ記録される。

  • 記録タイミング各保存処理の成功直後（saveToPhotoLibrary / saveToApp 内部）
  • 二重計上防止同一 creationSessionId の VideoCompletion が既にあれば挿入をスキップ
  • 影響範囲Hero「今日はもう1本つくれています」（overallHasCompletedVideoToday）/ 累計完成本数（completedVideoCount）が VideoCompletion を参照するため、カメラロールのみ保存のユーザーも達成と判定される

プレビュー破棄確認アラート

  • 発火条件「← 戻る」タップ時に state が .exported 以外の場合
  • タイトル「プレビューを破棄しますか？」
  • メッセージ「作成中の動画は破棄されますが、撮影したクリップはライブラリに残っています。あとから同じクリップで作り直せます。」（クリップが永続化済みであることを明示）
  • ボタン「クリップ選択に戻る」（role: .destructive）/ 「キャンセル」
  • state が .exported の場合アラートなしで即閉じる

フィルタセクション

  • 無料フィルタ全ユーザーが選択可能
  • Pro フィルタlock アイコン表示。タップで PaywallView 起動（trigger: .proFilter）
  • 選択中の effectiveFilterPro 解約後は Pro フィルタが .none に resolved されるため「locked かつ isSelected」の矛盾 UI が構造的に発生しない
  • 右端フェード続きありを暗示する LinearGradient overlay（タップを邪魔しないよう allowsHitTesting: false）

アクセシビリティ

  • カメラロール保存ボタン: .accessibilityLabel("カメラロールに保存。TikTokやInstagramへの投稿用")
  • アプリ内保存ボタン: .accessibilityLabel("\(appSaveButtonLabel)。RoutineReel内であとから再生できます")
  • 動画タップ再生切替: .accessibilityAddTraits(.isButton) + .accessibilityLabel("動画を再生・一時停止")
  • 全画面ボタン: .accessibilityLabel("動画を全画面で表示")
  • 戻るボタン: .accessibilityLabel("プレビューを閉じる")
