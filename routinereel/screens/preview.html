プレビュー・書き出し（PreviewView）

動画合成後のプレビューと2種類の保存ボタンを提供する画面。ClipGenerateView から NavigationStack に push される。画面タイトルは「仕上がりを確認」（旧「プレビュー」）。

基本情報

  • 画面IDpreview
  • 実装ファイルViews/Preview/PreviewView.swift
  • 表示方式NavigationStack push（CreateSessionView 管理下。自前 NavigationStack を持たない）
  • 起動経路ClipGenerateView（「動画を作成する」ボタン）→ 動画合成完了後に自動 push
  • 背景色#FAF9F7（warm white）
  • colorScheme.light 固定（warm white 背景のステータスバー文字色対応）

画面構成

  1. トップバー: 「← 戻る」ボタン（左）+ 「仕上がりを確認」タイトル（中央）
  2. 動画プレビュー領域: 9:16 縦型・黒背景・warm white 上に黒カードとして浮かせる。build69 より上詰め配置（余白を下側に集約）
  3. 動画メタ情報: 総時間・クリップ数（例「0:36・5クリップ」）を合成完了後に表示
  4. 音声トグル: 「撮影音声を残す／音声なしで書き出す」（肯定形。旧「元の音声をミュート」）
  5. フィルタセクション: 横スクロールチップ（無料フィルタ + Pro フィルタ）
  6. 無料ユーザー補足: 保存前に「動画の左上に小さくRoutineReelロゴが入ります」を表示
  7. 書き出しボタン: カメラロール保存（主役・グラデ）+ アプリ内保存（脇役・アウトライン）

動画プレビュー領域（build69 更新）

  • アスペクト比9:16 固定（縦型動画専用）
  • 配置（build69）上詰め（.frame(maxWidth: .infinity, maxHeight: .infinity, alignment: .top)）。以前は上下中央寄せで動画が小さく浮いて見えていたため、上部余白を削減して動画をトップバー直下まで引き上げ、余白を下側（ミュートトグル・フィルタ・ボタン間）に集約
  • cornerRadius16pt（.continuous）
  • 全画面表示動画右上の拡大ボタンタップで FullScreenVideoView（fullScreenCover）を起動。同一 AVPlayer インスタンスを共有し再生位置を維持

動画プレビュー領域の状態分岐

状態表示内容
.composing動画作成中プログレス（%表示 + バー + キャンセルボタン）
.readyAVPlayer 動画再生・タップで再生/一時停止・全画面ボタン

書き出しボタン（build69 更新）

build69 より各ボタンに用途の副説明を追加し、2種類の保存の違いを初見でも理解できるよう改善した。

ボタンスタイル副説明（build69 追加）無料枠消費
カメラロールに保存（主役）pink→orange グラデ・58pt「TikTokやInstagramへの投稿用」消費しない
アプリに保存（脇役）アウトライン・55pt「RoutineReel内であとから再生できます」消費する（無料3本まで）
  • 無効化条件state が .composing / .exporting / .idle / .error 時（isReadyToSave = false）は両ボタン無効化・opacity 0.45
  • アプリ内保存ボタンのラベルPro 時は「アプリに保存」、無料かつ残量あり時は「アプリに保存（残り N 件）」、無料かつ上限到達時は「アプリに保存（Proで無制限）」→ タップで Paywall 起動
  • アプリ内保存完了アラート「アプリに保存しました」タイトル + 「動画はホーム画面の『アプリ内の動画』から確認できます。」メッセージ（build69 より「最近の動画」→「アプリ内の動画」に更新）
  • 二重保存ガードPreviewViewModel 内で保存済み (sessionId, destination) ペアを Set で保持し、同一セッション・同一 destination での重複実行を防止（Analytics の重複送信防止も兼ねる）
  • カメラロール保存完了後親（CreateSessionView）の onSavedToPhotos コールバック経由で ExportCompleteView を push

動画メタ情報（総時間・クリップ数）（abba1b0 追加）

  • 表示例「0:36・5クリップ」
  • 表示条件合成完了後のみ（実測総尺 composedDurationSeconds > 0）。合成前や取得失敗（尺 0）は非表示
  • 尺フォーマットVideoDurationFormatter.string(seconds:)（M:SS）
  • クリップ数usedClipCount（実際に合成へ使用したクリップ数の単一情報源）
  • accessibilityLabel「動画の長さ 0:36、5クリップ」

音声トグル（肯定形に変更）（abba1b0）

トグルを「撮影音声を残すか」の肯定形で表示するよう変更した。内部フラグ muteOriginalAudio はトグルの反転で更新する（isOn = 撮影音声を残す = !muteOriginalAudio）。

トグル状態表示ラベル
ON（音声を残す）「撮影時の音を動画に残す」
OFF（音声なし）「音声なしで書き出す」

実挙動の修正（abba1b0）: 「音声なしで書き出す」を選ぶと保存動画の音声トラックが実際に除去される（保存直前に exportVideoWithoutAudio を実行）。旧実装は文言と実際の保存結果が食い違っていた。詳細は 動画合成仕様参照。

無料ユーザーへのウォーターマーク補足（abba1b0）

無料ユーザーには保存前に「動画の左上に小さくRoutineReelロゴが入ります」を表示し、保存後に透かしへ気づく期待ギャップを防ぐ。全画面アウトロカード廃止に伴い、無料版のブランド露出は左上ウォーターマークのみ。

保存先・工程の状態モデル（f7bb7b0）

保存先（カメラロール／アプリ内）を明示的な状態として管理し、保存中の進捗文言を保存先・工程ごとに正しく出し分けるよう変更した。旧実装は保存先に関わらず常に「カメラロールに保存中」と表示するバグがあった。

  • 保存先ExportDestination: .photoLibrary（カメラロール）／.appStorage（アプリ内）
  • 工程ExportOperation: .preparingAudio(destination:) / .applyingFilter(destination:) / .saving(destination:)
工程・保存先進捗オーバーレイの文言
preparingAudio（音声除去中）「音声なし動画を準備中…」
applyingFilter（フィルタ適用中）「フィルタを適用中…」
saving(.photoLibrary)「カメラロールに保存中…」
saving(.appStorage)「アプリに保存中…」

音声除去・フィルタ・保存は正確な進捗率が取れないため indeterminate 表示。詳細な状態機械・保存先ごとの再試行・戻る操作の分岐は 書き出し・保存フロー（保存先状態管理） を参照。

保存の再試行を失敗した保存先だけに限定（f7bb7b0）

保存失敗時の「保存をやり直す」が、失敗した保存先だけを再試行するよう修正した。旧実装は常にカメラロール保存が実行されるバグがあった。

  • Recovery.retrySave(destination:) が失敗した保存先を保持。retrySave は .error(_, .retrySave(destination)) 状態からのみ実行し、その destination のみ再試行
  • 失敗文言カメラロール失敗「保存に失敗しました」／アプリ内失敗「アプリ内への保存に失敗しました」

戻る操作の分岐（BackAction）（f7bb7b0）

保存中は戻るボタンを無効化し、状態に応じて破棄確認の要否を出し分ける純粋判定 BackAction を導入した。

状態BackAction挙動
保存前処理・保存中（.exporting）blocked戻れない（戻るボタン無効化・「保存中のため戻れません」）
合成中（.composing）confirmCancelCompose確認の上で合成キャンセル
未保存confirmDiscard破棄確認アラートを表示
保存済み or 初期状態dismissImmediatelyそのまま閉じる

保存済み判定は savedDestinations（実際に保存成功した保存先の集合）を単一情報源とする。Analytics 送信済み集合（sentVideoSavedDestinations）とは目的を分け、UI 状態の代用にしない。

アプリ内保存成功後の選択肢提示（f7bb7b0）

アプリ内保存に成功したら、confirmationDialog「アプリに保存しました」で次アクションを提示する。旧実装はアラートで OK を押すだけでプレビューに留まっていた。

  • 続けてカメラロールにも保存そのまま saveToPhotoLibrary を実行し、成功後は ExportCompleteView へ
  • ホームへ戻る再生停止して作成セッション全体を閉じる（onFinishSession）
  • この画面に残るcancel。プレビューに留まる
  • メッセージ「動画はホーム画面の『アプリ内の動画』から確認できます。カメラロールにも保存すると SNS へ投稿できます。」

VideoCompletion 記録

カメラロール保存・アプリ内保存のどちらが先に成功しても VideoCompletionRecorder.recordIfNeeded により1件のみ記録される。

  • 記録タイミング各保存処理の成功直後（saveToPhotoLibrary / saveToApp 内部）
  • 二重計上防止同一 creationSessionId の VideoCompletion が既にあれば挿入をスキップ
  • 影響範囲Hero「今日はもう1本つくれています」（overallHasCompletedVideoToday）/ 累計完成本数（completedVideoCount）が VideoCompletion を参照するため、カメラロールのみ保存のユーザーも達成と判定される

プレビュー破棄確認アラート

  • 発火条件「← 戻る」タップ時に state が .exported 以外の場合
  • タイトル「プレビューを破棄しますか？」
  • メッセージ「作成中の動画は破棄されますが、撮影したクリップはライブラリに残っています。あとから同じクリップで作り直せます。」（クリップが永続化済みであることを明示）
  • ボタン「クリップ選択に戻る」（role: .destructive）/ 「キャンセル」
  • state が .exported の場合アラートなしで即閉じる

フィルタセクション

  • 無料フィルタ全ユーザーが選択可能
  • Pro フィルタlock アイコン表示。タップで PaywallView 起動（trigger: .proFilter）
  • 選択中の effectiveFilterPro 解約後は Pro フィルタが .none に resolved されるため「locked かつ isSelected」の矛盾 UI が構造的に発生しない
  • 右端フェード続きありを暗示する LinearGradient overlay（タップを邪魔しないよう allowsHitTesting: false）

アクセシビリティ

  • カメラロール保存ボタン: .accessibilityLabel("カメラロールに保存。TikTokやInstagramへの投稿用")
  • アプリ内保存ボタン: .accessibilityLabel("\(appSaveButtonLabel)。RoutineReel内であとから再生できます")
  • 動画タップ再生切替: .accessibilityAddTraits(.isButton) + .accessibilityLabel("動画を再生・一時停止")
  • 全画面ボタン: .accessibilityLabel("動画を全画面で表示")
  • 戻るボタン: .accessibilityLabel("プレビューを閉じる")
