撮影画面（ShootingView）

v1.0 軽い見た目寄せ版。録画ロジック非接触で REC / Step progress / カウントダウン / テロップ位置 / ボタン名を統一。

基本情報

  • 画面IDshooting
  • 実装ファイルViews/Shooting/ShootingView.swift
  • 表示方式fullScreenCover（黒地・没入）
  • v1.1予定DesignsRecordPreviewView との本格統合

画面構成

  1. トップバー: 閉じる × / ルーティン名 + ステップ番号 / バランス用 spacer
  2. カメラエリア: AVCaptureSession プレビュー + テロップオーバーレイ + REC + カウントダウン
  3. Step progress バー: カプセル形状ドット + 現在ステップに波紋アニメ + 残り秒数
  4. 下部コントロール: 状態別ボタン（撮り直し/反転 + 録画 + 次へ / 完了 →）

テロップ位置（v1.0で大幅修正・2段階）

中央 → 画面下部 1/4 左下セーフゾーンへ移動。被写体回避 + TikTok/Reels UI（右下のいいね・コメント・シェア）との衝突回避。

  • 配置方法VStack { Spacer(); VStack(時刻 + ステップ名 縦並び・左寄せ) } + .padding(.bottom, SceneLabelLayout.bottomPadding(cardHeight:))
  • ステップ名（メイン）22pt
  • 時刻（サブ・ステップ名の上）15pt・opacity 0.85（v1.0で右下→左下ステップ名の上に統合）
  • 時刻表示AppSettings.timestampFormat（24h / 12h / 非表示（v1.0デフォルト））
  • 完成動画との整合VideoCompositionService 側も renderHeight * 22%・左下配置・フォント係数 0.034 で WYSIWYG 完全一致

v1.0変更前: 右下に時刻 + 左下にステップ名の並列配置 → 右下UI衝突懸念 → 左下に統合（時刻が上、ステップ名が下）

SceneLabelLayout 単一ソース化（2026-06-10）

撮影プレビュー（ShootingView / SingleStepShootingView）と完成動画（VideoCompositionService）の本編テロップを、配置比率・サイズ比率・行折り返し挙動・descent slack 補正まで含めて単一の SceneLabelLayout enum に集約。プレビューと書き出しの WYSIWYG を達成。

  • 実装ファイルViews/Common/SceneLabelLayout.swift
  • 配置比率leadingRatio = 0.05 / bottomRatio = 0.22 / maxWidthRatio = 0.55 / lineSpacingRatio = 0.008
  • サイズ比率stepNameFontSizeRatio = 0.034 / timeFontSizeRatio = 0.024
  • メトリクス補正lineHeightFactor = 1.17 / textHeightFactor = 1.4 / descentSlackFactor ≒ 0.23
  • 行数上限maxLines = 2（プレビュー / 動画書き出し共通）
  • SwiftUI 用ヘルパleadingPadding / bottomPadding / maxWidth / lineSpacing
  • CALayer 用ヘルパlayerOriginX / layerOriginY(fontSize:) / layerTextWidth / layerLineSpacing

撮影プレビュー位置を 22% に統一

  • 旧cardHeight * 7%（プレビューと動画でズレ）
  • 新SceneLabelLayout.bottomPadding(cardHeight:) = cardHeight * 22%
  • 効果完成動画の本編テロップ位置（renderHeight * 22%）と一致 → WYSIWYG 達成

長文ステップ名の折り返し挙動を統一

  • SwiftUI Text.lineLimit(SceneLabelLayout.maxLines) + .truncationMode(.tail)
  • CATextLayerisWrapped = true + truncationMode = .end + frame.height = singleLineHeight × actualLineCount
  • 行数測定actualStepNameLineCount(text:renderSize:fontName:) で NSAttributedString.boundingRect により実行数測定（最大 maxLines にクランプ）

descent slack 補正

  • 課題CATextLayer は top-anchored 描画でグリフ下端が frame 下端より約 fontSize * 0.23 浮く
  • 対応layerOriginY(fontSize:) は base - fontSize * descentSlackFactor で下方向に補正
  • 効果SwiftUI Text の intrinsic 高さと概ね揃い、プレビュー / 動画の見え方がほぼ一致

SingleStepShootingView の追従

  • 底面グラデーション追加ShootingView と同じグラデーションを敷設（22% 化でテロップが暗部に乗る副作用の対応）
  • #Preview マクロ規約準拠で追加

残課題（既知）

撮影フローには ShootingView（シーケンス撮影）と SingleStepShootingView（単発撮影）の 2 系統が残存している。レイアウト面は SceneLabelLayout 集約で揃ったが、撮影ロジック本体の統合は P0-1 / P0-2 として別タスクで実施予定。

REC インジケーター（呼吸する赤）

  • 外側ハロー14pt 赤 opacity 0.4 + blur 4 + DesignsPulse (0.7→1.15, 2.0秒周期)
  • 内側ドット8pt red + DesignsPulse (0.88→1.0, 2.0秒周期)
  • 「REC」文字12pt semibold + tracking 0.5
  • 背景Capsule 黒 opacity 0.32
  • 位置右上 padding-trailing 16 / top 12

カウントダウン

  • フォント128pt ultraLight rounded
  • 色white opacity 0.88
  • トランジションscale 1.4 → 1.0 → 0.6 asymmetric + opacity
  • アニメーションspring(0.45, 0.75)
  • idcount 値で id 付与（切替時に再描画）

Step progress バー（波紋型）

  • 形状Capsule (旧: Circle)
  • サイズ現在 22×4 pt / その他 8×4 pt
  • 色過去: white opacity 0.7 / 現在: accentOrange / 未来: white opacity 0.25
  • 現在ステップの波紋stroke 1.5pt + DesignsRingPulse (scale 1.0→1.18, opacity 0.55→0, 1.8秒周期, 片道)
  • 切替アニメspring(0.35, 0.7)

ボタン名統一（v1.0で日本語化・誤解解消）

状態位置BeforeAfter
ready左flip反転（カメラ反転）
ready右skip次へ（このステップを撮らずに次へ）
recording/countdown中央next step (forward.fill)完了 →（checkmark）

理由: 「skip」「next step」は「飛ばす」誤解を生む。撮影中の主操作は「このステップを終えて次へ」の肯定形が正しい。

下部コントロール詳細

ready 状態（撮影開始前）

  • 左: 撮り直しボタン（前ステップが撮影済みなら）または 反転ボタン
  • 中央: 録画ボタン（赤 56pt + stroke 68pt）
  • 右: 次へボタン（このステップを撮らずに次へ）

recording / countdown 状態（撮影中）

  • 中央のみ: 完了 → ボタン（このステップを確定して次へ）
  • 左右は Spacer で省略

transitioning 状態（ステップ間遷移）

  • 中央: ProgressView

権限処理

  • カメラ必須・拒否時はアラート「設定を開く」
  • マイク必須・拒否時は同上
  • 写真ライブラリ追加のみ・PreviewView 側で要求

BGM 機能の扱い

v1.0 で完全削除。撮影時の BGM 試聴・合成時の BGM 焼き込みは廃止。詳細は BGM機能削除の意思決定記録 参照。

v1.1 で予定される改善

  • DesignsRecordPreviewView との本格統合 — モック背景の生活感（粒子・露出フリッカー・鏡）を実カメラに統合
  • テロップのプリセット別最適化 — 時刻表示を「経過時間」「Day Count」へ切替可能化（workout/study 向け）
  • 撮り直しボタンの位置移動 — 誤タップ防止のため遠い位置へ
  • テロップフォント拡大 — 24pt 以上を検討
