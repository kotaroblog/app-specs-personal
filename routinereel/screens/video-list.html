アプリ内の動画（VideoListView）

build69 より「私の記録」→「アプリ内の動画」に変更。アプリ内に保存した動画のみを表示する画面であることを明示（カメラロール保存のみの動画はここに表示されない）。

基本情報

  • 画面IDvideo-list
  • 実装ファイルViews/Library/VideoListView.swift
  • 表示方式sheet（NavigationStack内）
  • 起動経路ホーム「最近の動画 → すべて見る」/ サムネタップ / 作品タブ
  • ナビタイトルアプリ内の動画（build69 より「私の記録」→「アプリ内の動画」に変更）
  • 背景色#FAF9F7

画面構成

  • 動画一覧2列グリッド（旧3列から変更）・縦並び LazyVGrid
  • 空状態film.stack アイコン + 「アプリ内に保存した動画はありません」（build69 より「まだ記録がありません」→変更）+ 説明「プレビュー画面で『アプリに保存』すると、ここからあとで再生できます。」
  • サムネタップVideoPlayerSheet（黒地・没入再生）
  • contextMenuシェア / 削除

動画セル

  • サムネaspectRatio 9:16・cornerRadius DesignsRadius.xl (22)・designsCardShadow
  • 再生バッジ右下に小さく play.fill 円（黒 opacity 0.35）
  • メタデータサムネ外にカード外配置・最小限の情報（ルーティン名 + 月日のみ）
  • 押下DesignsPressableButtonStyle scale 0.97

VideoPlayerSheet（再生中・黒地維持）

動画コンテンツの視認性を優先するため、再生中のみ純黒背景を採用（Home UI統一の例外）。

  • 上部閉じる × / シェア / 削除（赤 opacity 0.85）
  • 中央VideoPlayer aspectRatio 9:16
  • 下部ルーティン名 + 撮影日時
  • playerAVPlayer + onAppear で auto play / onDisappear で pause
  • colorScheme.preferredColorScheme(.dark) を個別付与（RootView の .light に対する例外。黒地での白テキスト視認性を確保）
  • ミュートToggle VoiceOverミュートトグルボタンに .accessibilityLabel（「ミュート」/ 「ミュート解除」）を付与
  • 動画タップ再生切替.accessibilityAddTraits(.isButton) を付与し VoiceOver でボタンとして認識

動画ファイル欠損時の表示（v1.1.0・UI/UX改善バッチ）

動画ファイルが見つからない場合、従来は無限スピナーまたは黒画面のまま止まっていたが、明示的なエラー表示に変更した。

  • 表示内容「動画ファイルが見つかりません」テキストを画面中央に表示
  • 変更前AVPlayer がファイルなしで生成されローディングスピナーが無限回転 / 黒画面のまま停止
  • トリガーVideoStorageService でファイルURL存在チェックが失敗した場合

削除処理

  1. pendingDelete に Video セット → alert 表示
  2. 「削除」確定で VideoStorageService.deleteVideo()（ファイル + サムネ削除）
  3. modelContext.delete(video) で SwiftData レコード削除

シェア処理

  • ファイル確認FileManager.default.fileExists(atPath:) で存在チェック
  • シートShareSheet（UIActivityViewController）で TikTok/Instagram/AirDrop等へ
