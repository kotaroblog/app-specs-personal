1. アップデート目的とコンセプト

現状

  • 現バージョンv1.2.1 リリース済み
  • 配信地域日本のみ
  • 課題継続率（D1 / D7）の引き上げが急務。習慣形成ループ・期待形成・終了体験が不足している

コンセプト：「静謐ドーパミン」

本コンセプトの核心は「刺激を弱める」ことではない。
"刺激の質を低周波化する"ことである。

❌ ドーパミンを弱くする（＝無刺激化 = 継続理由消失）
⭕ ドーパミンを低周波化する（＝刺激の周波数を下げ、なおかつ脳が欲しがる状態を作る）

Block Blast! が蛍光色・大音量で実装するコンボ演出を、PieceZen の世界観（木・石・自然素材）に合わせ 「静かだが確実に脳が欲しがる」版に置き換える。"静かなのに、やめられない"を成立させるため、 触覚・予兆・期待形成の3要素を中核に据える。

競合のアプローチPieceZen のアプローチ
うるさく・派手にフィードバックする（高周波）低く・深く・確実にフィードバックする（低周波）
蛍光色・大音量コンボ演出木目テクスチャ・低音SE・呼吸グロー
視覚刺激で引き止める触覚・予兆・期待形成で脳を引き止める
"今の快感"だけで成立"次で決まるかも"の期待形成までセット

4. v2.0 機能詳細

4.1 コンボ/連鎖システム強化

項目内容
目的連続ライン消しに倍率を設け、プレイヤーに「次も消したい」という動機を与える
仕様連続ライン消しで倍率アップ（2x → 4x → 8x）。倍率テキストのアニメーション表示
UI仕様"墨が滲む"ような拡散表現でテキスト出現。木目テクスチャ世界観に合うゴールド系。文字だけが派手にならないよう、消去列から光粒子が盤面に吸われる演出を同時発火
既存コード活用既実装の comboCount / comboText / comboCleared() を活用。スコア式調整＋エフェクト層追加
改修ファイルGameEngine.swift, ScoreEvent.swift, PlayFeedbackOverlay.swift
実装難易度S（Small）
工数4〜6h

4.2 パーフェクトクリア演出強化

項目内容
目的盤面全消しを「特別感のある体験」にして達成感を最大化する
仕様盤面全消し検知時、ボーナスを +500pt から +1000pt に調整。0.5秒のスローモーション後に通常速度へ復帰
UI仕様木目テクスチャに合わせた光の粒のパーティクル（20〜30粒）をグリッド全体に広げる。背景に薄い"呼吸グロー"を1サイクル
既存コード活用既実装の isPerfectClearTriggered / PerfectClearOverlay を活用
改修ファイルGameEngine.swift, PlayFeedbackOverlay.swift
実装難易度S（Small）
工数2〜4h

4.3 強化ハプティクス

項目内容
目的触覚フィードバックを「低周波ドーパミン」世界観に統一し、プレイの没入感を高める
仕様（振動パターン） 通常ライン消し: 短い1回振動
コンボ時: 段階的に強くなる連続パターン（2x=弱2回 / 4x=中3回 / 8x=強4回）
パーフェクトクリア: 長めの重厚パターン（残響感を意識）
UI仕様（効果音）コンボ・パーフェクト専用 SE を「ガラス共鳴音・木琴音」など自然素材系に統一。残響を長めに取る
改修ファイルHapticManager.swift
実装難易度S（Small）
工数3〜5h
注意点SE 音源はユーザー側で別途調達予定。実装着手前までに用意する

4.4 ストリークカレンダー + 7日報酬テーブル

項目内容
目的連続起動を視覚化・報酬化して習慣形成ループを作る（1010! 参考）
仕様連続起動日数を「7日カレンダー」で可視化。7日連続ログインで段階報酬を付与
報酬テーブル Day1: 軽い演出（スコア +500）
Day3: スキン「ocean」一時試用（24h）
Day7: スキン「sakura」永久解放（または広告除去 24h）
UI仕様ホーム画面起動時のモーダルに7マスのカレンダーを表示。達成済みマスは木目アイコンで強調
既存コード活用既実装の StreakManager を流用。報酬テーブルとモーダル表示を追加
改修ファイルStreakManager.swift, ContentView.swift, 新規 StreakRewardView.swift
実装難易度M（Medium）
工数8〜12h
重要制約ストリーク消滅ペナルティは設けない（Blockudoku の失敗事例より。カジュアル層離脱を防ぐ）

4.5 Near Miss 演出 NEW v2.0

項目内容
目的「あと1マスで全消し」「あと1列」の状態で"惜しい"感を能動的に演出し、次の一手への期待を生む。人間は「惜しい」で中毒化する
仕様 盤面状態をプレイヤーの操作ごとに評価し、以下のいずれかを検出した時に Near Miss 演出を発火：
① あと1ピース配置でライン消しできるセル列・行が存在
② あと1ライン消去でパーフェクトクリアが成立
③ コンボ継続まであと1手
UI仕様 対象セルに微発光（透明度 0.3 で 1.5秒周期の呼吸アニメ）
対象列・行の外周に淡い磁力線のような細いライン演出
静かなSE（風鈴の余韻のような単発音）を1回再生
ハプティクスは出さない（過剰刺激回避）
新規実装新規 NearMissDetector.swift（盤面評価ロジック）, NearMissOverlay.swift（演出層）。GameEngine.swift に評価フック追加
実装難易度M（Medium）
工数6〜10h
注意点毎ターン評価が走るためパフォーマンス影響を測定。8×8 グリッドなので O(64) 程度に収まる想定

4.6 配置時の「吸着感」 NEW v2.0

項目内容
目的「置けた」感の触覚的快感を作る。配置という最頻アクションに低周波ドーパミンを宿す
仕様 配置確定時のアニメーションを以下のシーケンスで実装：
① ドロップ着地に 0.06秒の遅延（"吸い込まれる"感）
② 配置直後に scale 1.04 → 1.0 のスプリングアニメ（0.15秒）
③ 微振動（UIImpactFeedbackGenerator .light）
④ 低音SE 1発（ウッドノックのような短音）
⑤ 周囲4セルに 0.1秒だけ光拡散
UI仕様シーケンス全体で 0.25秒以内に収め、操作のテンポを損なわない
改修ファイルGameEngine.swift（finalizePlacement のシーケンス変更）, BoardView.swift, HapticManager.swift
実装難易度S（Small）
工数3〜5h
注意点低音SE 1種類が追加で必要（自然素材系）。SE調達リストに追記

4.7 終了画面の再設計 NEW v2.0

項目内容
目的ゲームオーバーを離脱地点ではなく再プレイ起動点に変える。"あと1回"導線を埋め込みD1継続率を直接押し上げる
仕様（表示要素） ① 成長表示: 今回スコア＋過去ベスト超え or 過去N回平均超えを明示
② "あと1回"フック: 「最高スコアまであと◯◯」「あと1コンボで新記録」など、状況に応じた動的コピー
③ streak 維持表示: 「今日プレイ済み（streak ◯日継続中）」または「あと◯時間でstreak途切れます」
④ 1タップ再開ボタン: 大きく中央配置、迷わせない
⑤ リワード広告救済は既存通り維持
⑥ 静かな終了SE（息を吐くような余韻音）
"あと1回"コピー生成ロジック EndOfGameMessageBuilder で以下の優先順位で1メッセージ選択：
a. ベスト記録まで残り 100pt 以内 → 「あと◯◯で新記録」
b. コンボ最大値まであと1〜2 → 「あと◯コンボで自己ベスト」
c. streak 7日達成まで残り3日以内 → 「あと◯日でstreak報酬」
d. 該当なし → 「ナイスプレイ。次の一手を待っている」など中性メッセージ
新規実装新規 EndOfGameMessageBuilder.swift, GameOverView.swift 大幅改修（既存ファイル）
実装難易度M（Medium）
工数8〜12h
注意点"あと1回"コピーの誤発火（誇大表現）を避けるため、しきい値は実データで調整。最初は a/b/c の優先順位を保守的に設定

5. v2.1 機能サマリ（簡易）

5.1 デイリーチャレンジ

  • 日付シードベースで固定パズルを生成し、世界中のプレイヤーが同じお題に挑戦できる
  • クリアでトロフィーバッジを付与（SwiftData に保存）
  • 新規モデル追加のため SwiftData マイグレーション必須
  • 参考: Blockudoku のデイリーチャレンジ機能

5.2 レベル/XP システム

  • プレイごとにスコアに応じて XP を加算
  • レベルアップ時に専用演出（パーティクル + ハプティクス）
  • 新規モデル PlayerProfile を追加（レベル・累計XP・解放スキン一覧）

5.3 スキン拡張（v2.0 改訂で大幅強化）

「色変更だけ」では収集物として弱いという指摘を踏まえ、スキンを"テーマパッケージ"に格上げする。

  • 既定義5テーマ（classic / ocean / sakura / neon / monochrome）それぞれに以下を紐付け：
    • 配色（既存）
    • 専用SE（配置音・ライン消去音・コンボ音）
    • 専用ハプティクス（パターン微調整）
    • 背景粒子エフェクト（sakura=花びら / ocean=泡 / neon=光線 等）
    • UI音（メニュー操作音）
  • 解放トリガー: レベル達成 / 実績達成 / リワード広告視聴のいずれか
  • 解放済みスキンは PlayerProfile.unlockedSkins で管理
  • v2.1 想定工数: 大幅増（各テーマで SE 3〜4種 / 粒子エフェクト1種が必要）。アセット制作がボトルネック

6. v2.2 機能サマリ（簡易）

  • デイリーミッション3種: コンボ5回達成 / 100行消す / パーフェクトクリア1回 等。日替わりで入れ替わる
  • 実績・トロフィー: Game Center 連携。累計スコア・プレイ日数・コンボ回数等で解放
  • 世界ランキング: Game Center リーダーボード。週次ハイスコア + 累計スコアの2種

7. やらないこと（スコープ外）

避けるべき要素理由
ゲーム中のインタースティシャル広告プレイ強制中断によるユーザー体験の著しい悪化
ストリーク消滅ペナルティ（ライフ制）カジュアル層の離脱を招く（Blockudoku の失敗事例）
数日間続くトーナメント（72時間超）プレイ義務感が強くなり長期離脱につながる
ブッシュ・ジェム等のゲームプレイ変質オブジェクト長期ユーザーの離脱・既存ゲームバランスの崩壊
バトルパス形式のメタ進行認知コストが高くカジュアル層に不適合
"刺激を弱くする"方向の演出変更「静謐」と「無刺激」を混同しない。静かであっても脳が欲しがる状態（低周波ドーパミン）を成立させる。単純なボリュームダウン・エフェクト削減は禁じ手
"広告少"を継続理由として打ち出すコピー入口には効くが継続理由にはならない。継続理由は「気持ちいいから続ける」に集約する

8. 技術リスク・確認事項

リスク・確認事項詳細対応方針
グリッドサイズの確定 実装上は 8×8 グリッドで運用中。ストア表記「10×10」とは不一致 8×8 を正とする（確定）。次回ストア提出時にメタデータ（サブタイトル・スクリーンショット文言・説明文）を 8×8 表記に修正する
SE 音源調達 コンボ・パーフェクトクリア・配置吸着・Near Miss・終了画面の自然素材系 SE が未調達 ユーザー側で別途調達予定。実装着手前までに用意する。最低必要数: ①コンボ系3種 ②パーフェクト1種 ③配置吸着（低音ウッドノック）1種 ④Near Miss（風鈴余韻）1種 ⑤終了余韻音1種 = 計7種
Near Miss 検出のパフォーマンス 毎ターン盤面評価が走るため、操作レスポンスへの影響を計測する必要あり 8×8 グリッドで O(64) 程度に収まる想定。配置確定後の非同期評価で対応。Instruments でプロファイリング実施
SwiftData マイグレーション v2.1 以降でモデル追加（PlayerProfile 等）が発生する VersionedSchema + SchemaMigrationPlan の実装を v2.1 開発前に必ず設計
Game Center Capability v2.2 で実績・リーダーボードを追加する際に必要 v2.2 開発前に project.yml に Game Center capability を追加 + Entitlement 設定
AdMob リワード広告の流用 既存のリワード広告実装を v2.0 ストリーク報酬（Day7: 広告除去 24h）・v2.1 スキン解放で流用 既存 AdManager.swift の流用可否を確認。コールバック設計の整合性を保つ
