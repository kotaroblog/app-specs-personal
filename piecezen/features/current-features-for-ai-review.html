Section 1: アプリ概要

GPT-5 などの外部 AI がゼロ知識状態で評価できるよう、ジャンル・世界観・収益構造・配信状況を整理する。

項目内容
アプリ名PieceZen（ストア表示名）/ プロジェクト名: BlockPuzzle
ジャンルカジュアルゲーム / パズル（ブロック配置系）
プラットフォームiOS 18.0+（iPhone 縦向き専用）
配信状況リリース済み（2026年4月）· v1.2.1 · 日本のみ配信
年齢レーティング4+（暴力・購買要素なし、広告は AdMob 標準）
言語日本語のみ
収益構造AdMob（Banner + Interstitial + Rewarded） + IAP 広告除去 ¥480 買い切り
開発体制個人開発 1 名 · 週 20h（Claude Code 活用で実質 3〜5 倍速）
技術スタックSwift 6 / SwiftUI / SwiftData / StoreKit 2 / Google AdMob

世界観・コンセプト

「静かに集中できるパズル体験」をコンセプトとする。競合（Block Blast! 等）が蛍光色・大音量のエフェクトを採用する中、 PieceZen は落ち着いた配色テーマ・システムサウンド活用・白いフラッシュ演出など、 カジュアル層が日常のスキマ時間に使いやすい雰囲気を重視している。 ホーム画面にはゆっくり動くパーティクル背景（ZenParticleBackground）を配置し、 「始める前からリラックスできる」導入体験を意図している。

ゲームの基本構造（ひと言まとめ）

8×8 グリッドに 19 種類のブロックピース（回転対応）をドラッグ＆ドロップで配置し、 横列・縦列が揃うと消えてスコアが入る。トレイの 3 ピースをすべて使い切ると次の 3 ピースが補充される。 どのピースもグリッドに置けなくなった時点でゲームオーバー。回転を使って詰まりを回避するのが差別化要素。

Section 2: 現状ゲーム機能一覧（コード実装ベース）

2-1. コアゲームプレイ

項目仕様根拠ファイル
グリッドサイズ 8×8（64マス固定）。GridPosition.gridSize = 8 GridPosition.swift, GameEngine.swift
ピース種類数 19 種類（BlockShape.allShapes に列挙）。1〜5マス、直線・L字・T字・S/Z字・+字・2×2 正方形を網羅 BlockShape.swift
ピースのマス数内訳 1マス×1、2マス×2、3マス×6、4マス×7、5マス×3（bigLShapeA/B, plusShape） BlockShape.swift
ピース色数 12 色の論理キー（red/blue/green/yellow/orange/purple/cyan/pink/mint/indigo/teal/brown）をテーマにマッピング CellState.swift, ColorTheme.swift
トレイ仕様 常時 3 枠。3 枠すべて使用済みになったタイミングで次の 3 ピースが一括補充。個別補充はない GameEngine.refillTray()
トレイフェアネス 1 セット（3枚）あたり 5 マス以上のピース（bigL/plus）は最大 1 枚。残り 2 枠は小さいピースから選ばれる BlockShape.generateTray()
配置操作 ドラッグ＆ドロップ。ドラッグ中にプレビュー表示（置ける場合は色付き、置けない場合はグレー） GameEngine.updatePreview(), GridView.swift
配置スコア 1 マス配置 = 1 点（マス数分が即時加算） GameEngine.place()
ライン消去ルール 横列（行）または縦列（列）がすべて埋まると消去。同一アクションで複数ライン同時消去あり GameEngine.detectLinesToClear()
回転機能 あり。トレイ上のピースをタップで 90 度時計回り回転。回転状態はトレイ内で保持 GameEngine.rotateBlock(at:)
ゲームオーバー判定 トレイに残るすべてのピースについて「4 回転すべてを試して」グリッドのどこにも置けない場合にゲームオーバー。回転で打開できる場合は継続 GameEngine.canPlace(block:)

2-2. スコアリングシステム

配置スコア: 配置マス数 × 1点（毎配置即時加算） ライン消去スコア: 基本スコア = 20 × ライン数 × max(1, comboCount) 同時消去ボーナス（加算）: DOUBLE（2ライン同時）: +30点 TRIPLE（3ライン同時）: +80点 QUAD（4ライン以上同時）: +200点 CROSS（行+列の同時消去）: +150点 コンボ時（連続消去2回目〜）: ボーナス加算なし（倍率で稼ぐ設計） パーフェクトクリアボーナス: +500点（グリッド全マス消去時）

2-3. コンボシステム

項目仕様
カウント条件 ピース配置 1 回ごとにライン消去が発生したターンを連続で数える。消去ゼロのターンで comboCount リセット
スコア倍率 基本スコアに max(1, comboCount) を乗算。2 コンボなら基本スコア 2 倍、3 コンボなら 3 倍（上限なし）
コンボ数表示 2 コンボ以上でスコアヘッダーに x2, x3... のコンボテキストが 300ms 表示された後フェードアウト
賞賛テキスト 2コンボ→「Nice!」、3コンボ→「Great!」、4コンボ以上→「Amazing!」が 900ms ポップアップ
演出 コンボ段階に応じたハプティクス強度変化 + サウンド（4コンボ以上でより派手な音）。フローティングスコアテキストのフォントサイズ・色も段階で変化

2-4. 同時消去ボーナス演出

ボーナス種別条件賞賛テキスト全画面フラッシュ色フローティングテキスト色
DOUBLE!2ライン同時消去DOUBLE!黄色（opacity 0.18）黄色
TRIPLE!3ライン同時消去TRIPLE!青（opacity 0.22）青
QUAD!4ライン以上同時消去QUAD!紫（opacity 0.28）紫
CROSS!行+列の同時消去（交差）CROSS!ピンク（opacity 0.28）ピンク。交差点に波紋リング+放射パーティクル×16
PERFECT!グリッド全消去PERFECT!虹グラデーション（opacity 0.4）虹グラデーション文字
消去アニメーションは「100ms 白フラッシュ → 300ms フェードアウトでグリッドをクリア」の 2 段階。 消去位置ごとにパーティクル（5〜12 粒）が飛散する。CROSS 演出のみ交差点に CrossExplosion（16粒放射 + リング波紋）が加わる。

2-5. パーフェクトクリア

項目仕様
検知条件ライン消去後にグリッドの全 64 マスが empty になった瞬間（finalizePlacement 内で判定）
ボーナス+500点を即時加算
演出時間1.5 秒間の専用オーバーレイ（虹グラデーション背景 + "✨ PERFECT! ✨" テキスト + "+500 BONUS"）
ハプティクスgameOver()（UINotificationFeedbackGenerator .error）を流用した強い衝撃
サウンドcombo音 + 200ms 後に gameOver音の重ね掛け
シェイクshakeTrigger += 3（画面シェイクアニメーションを 3 回分発火）

2-6. アンドゥ機能

項目仕様
最大使用回数1 ゲームにつき 3 回（GameEngine.undoMaxCount = 3）
無料回数1 回目はリワード広告不要。2〜3 回目以降はリワード広告視聴が必要（undoFreeRemaining で管理）
スナップショット保存タイミングピース配置直前に毎回上書き（1手前のみ保存）
復元内容グリッド状態・トレイの 3 ピース・スコア・comboCount を完全復元
動作条件gameState == .playing かつ undoSnapshot が存在する場合のみ有効
アニメーション進行中のアニメーション Task（消去・配置）をキャンセルしてから easeOut(0.25s) で復元

2-7. 救済機能（リワード広告）

項目仕様
発動タイミングゲームオーバー直後の画面で「広告を見てもう1手」ボタンが表示
表示条件!hasUsedReward && AdManager.shared.isRewardedAdReady（1ゲーム 1 回制限）
救済内容埋まっているセルをランダムに 1 マス消去してゲームを再開
ゲームオーバー再判定救済後に checkGameOver() を再実行。1マス消えただけでは打開できない場合は即座に再ゲームオーバー
使用フラグhasUsedReward = true にして以後の同ゲームでは非表示

2-8. ストリーク（連続プレイ日数）

項目仕様
カウント方法ゲーム開始時に StreakManager.shared.recordPlay() を呼び出し。同日複数回は 1 回とカウント
継続条件前日プレイ済み → +1。2 日以上空いた → 1 にリセット
途切れ検知アプリ起動時（StreakManager.init()）に lastPlayDate が昨日でも今日でもなければ currentStreak を 0 にリセット
永続化UserDefaults（currentStreakDays / lastPlayDate / longestStreakDays）
表示場所ホーム画面の statsRow にストリーク日数を表示
最長記録longestStreak を別途保持
通知連携毎日 21:00 に UNCalendarNotificationTrigger でリマインダー通知（ストリーク維持促進）

記録項目永続化方法表示場所
ハイスコア（全期間） SwiftData HighScoreRecord（score, achievedAt） ホーム画面・ゲーム画面ヘッダー・ゲームオーバー画面

2-10. カラーテーマ

テーマ名表示名配色コンセプトブロック色の特徴
classic クラシック デフォルト。虹色系の鮮やかな配色 SwiftUI 標準色（red/blue/green/yellow 等）+ グラデーション
ocean オーシャン 青〜緑〜水色の落ち着いた海系 全色を HSL で青系統に統一（深いブルー / シアン / ターコイズ 等）
sakura サクラ ピンク〜パープル〜暖色の柔らかい春系 桜ピンク / ラベンダー / コーラル / ピーチ 等
neon ネオン saturation 100% の蛍光サイバー系 全色が彩度最大の蛍光色（ゲームライクな派手さ）
monochrome モノクローム グレースケールのみ 12色すべてをグレー階調（0.25〜0.85）に対応。色覚多様性・ミニマリスト向け
テーマは設定画面から即時切り替え可能（@AppStorage 経由）。 各テーマはメインカラー / lighter / darker の 3 段階グラデーションで立体感を表現。 ブロックの表示色はテーマに依存し、ゲームロジックは BlockColor という論理キーで管理されるため、テーマ変更がゲーム結果に影響しない。

Section 3: ターゲットユーザー像

主ターゲット

  • 年代: 20〜50 代（若者層〜働く世代）
  • シーン: 通勤・休憩・就寝前など「スキマ時間 3〜10 分」の脳トレ用途
  • ニーズ: 「広告が少なくて集中できる」「脳を使う達成感がある」「始めやすくてやめやすい」
  • ゲームリテラシー: カジュアル層（毎日ゲームに 1 時間以上は使わない）

副ターゲット

  • パズルゲームをある程度やり込んで飽きた層（「回転できる」差別化に惹かれる中堅パズルプレイヤー）
  • 色覚配慮が必要なユーザー（モノクロームテーマで対応）
  • BGM / SE をオフにして使いたいユーザー（別アプリの音楽を聴きながらプレイ対応 .ambient モード）

想定しないユーザー

  • オンライン対戦や SNS 連携を求めるコアゲーマー層
  • 長時間・複雑なストーリーを求めるプレイヤー
