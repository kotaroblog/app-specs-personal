6. 処理フロー（maxCombo 永続化）

// GameEngine 内の処理概要

// ゲーム開始時
func startNewGame() {
    maxComboInSession = 0
    // ... 既存初期化処理
}

// コンボ更新時（既存処理に追記）
func updateCombo(_ newCombo: Int) {
    if newCombo > maxComboInSession {
        maxComboInSession = newCombo
    }
}

// ゲーム終了・統計保存時
func saveStatRecord(context: ModelContext) {
    let record = StatRecord()
    record.score = currentScore
    record.playedAt = Date()
    record.maxCombo = maxComboInSession   // ★ v1.3.0 追加
    context.insert(record)
}

7. ビジネスルール

  • 医療効能の断定表現禁止: 「予防効果がある」→「〜と言われています」「〜をサポートします」等に統一
  • アクセシビリティ設定はアプリ再起動後も保持（@AppStorage で永続化）
  • seniorContrast テーマは設定画面のテーマ選択リストに追加するが、旧5テーマは削除しない
  • maxCombo の旧データは 0 扱い（SwiftData デフォルト値。既存ユーザーへの影響なし）
  • 共有テキストの内容はローカル生成のみ（ネットワーク通信なし）

9. リスクと対応

リスク対応
Apple 審査での「医療系訴求」誤判定 「ボケ予防」等の断定表現を避け「〜と言われています」「習慣化サポート」に統一。ガイドライン 1.4 を審査前に再確認
アプリ名変更でストア表示が乱れる 審査通過後に App Store Connect で変更を確定させる（レビュー中は仮表示のまま）
maxCombo マイグレーション失敗 SwiftData の @Model デフォルト値機能を使用。失敗時は VersionedSchema でフォールバック
高コントラストテーマで既存スタイルが崩れる 既存テーマとは独立した ColorTheme.seniorContrast として実装。既存テーマに影響なし

10. 実装メモ

  • dynamicTypeSize の適用は ContentView ルートで行い、全 View に自動伝播させる
  • 曜日ラベル「今日」は Calendar.current.isDateInToday(date) で判定し、当日のみ「今日」表示
  • グラフの週開始は Calendar.current.firstWeekday ではなく月曜固定（Locale に依存しない）
  • 共有ボタンの ShareLink はコンテンツを動的生成するため PreviewProvider にはモックデータを使用
  • maxComboInSession はゲームセッション中のみのメモリ変数。SwiftData への書き込みはゲーム終了時のみ
