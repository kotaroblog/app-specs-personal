ワールド選択画面

保存済みワールドの一覧表示・新規作成・削除を行うエントリー画面

画面ID: VG-WORLD-01

目的・役割

アプリ起動後に最初に表示される画面。プレイヤーがプレイするワールドを選択するか、新規ワールドを作成してゲームプレイ画面へ遷移する。ワールドは SwiftData（WorldStore）に永続化される。

UI 構成

要素種類説明
ナビゲーションタイトルNavigationTitle「ワールド選択」と表示
新規ワールド作成ボタンButton（toolbar）右上「＋」アイコン。タップでワールド作成シートを表示
ワールド一覧List保存済みワールドを最終プレイ日時の降順で並べる
ワールドセルListRowワールド名・シード値・最終プレイ日時を表示。タップでゲームプレイ画面へ遷移
削除スワイプアクションSwipeAction（.destructive）左スワイプで「削除」ボタンを表示。確認アラート後に SwiftData から削除
ワールド作成シートSheetワールド名入力（TextField）・シード値入力（TextField / 整数、省略可）・「作成」ボタン
エンプティビューContentUnavailableViewワールドが 0 件の場合に「まだワールドがありません」と表示

状態

状態表示内容
初期表示（ワールドあり）ワールド一覧を最終プレイ日時降順で表示
初期表示（ワールドなし）ContentUnavailableView「まだワールドがありません。＋ボタンで作成してください」
作成シート表示中ワールド名・シード値入力フォームを表示。作成ボタンは名前が空の場合は無効
削除確認アラート「このワールドを削除しますか？」確認ダイアログ（削除 / キャンセル）

遷移

操作遷移先
ワールドセルをタップゲームプレイ画面（GameView）へ push。選択した WorldRecord を渡す
新規作成シートで「作成」ボタンSwiftData に WorldRecord を保存 → シートを閉じる → 一覧に新規ワールドが追加される
削除確認で「削除」SwiftData から WorldRecord を削除 → 一覧から消える

使用するデータモデル

@Model
final class WorldRecord {
    var id: UUID = UUID()
    var name: String = ""
    var seed: Int = 0
    var lastPlayedAt: Date = Date()
    // チャンクデータは別ファイル（ドキュメントディレクトリ）に保存
}

実装メモ

  • シード値を省略した場合は Int.random(in: Int.min...Int.max) で自動生成する
  • チャンクデータ本体（.chunk ファイル）はドキュメントディレクトリに worldId をキーにして保存する。SwiftData には WorldRecord メタ情報のみ持たせる
  • ワールド削除時は WorldRecord だけでなく関連する .chunk ファイルも削除する
  • 最終プレイ日時はゲームプレイ画面を離れる onDisappear タイミングで更新する
  • 一覧は @Query(sort: \WorldRecord.lastPlayedAt, order: .reverse) で取得する
