データモデル

SwiftData @Model の構造とゲーム内データ設計

概要

永続化は100% SwiftData（デフォルトの SQLite ストア）で完結しており、独自のバイナリファイル I/O は行わない。モデルは WorldRecord・ChunkRecord・ChestRecord の3つで、いずれも worldID（UUID）でワールドごとに分離する。ゲームロジック自体は SceneKit のレンダースレッド上で動くランタイムクラス（Player・Mob 等）が保持し、変化があったときだけ SwiftData モデルへ書き戻す。

WorldRecord（SwiftData @Model）

ワールドのメタ情報・進行状態をまとめて保持する唯一のモデル。プレイヤーの位置・時刻・天候は保存対象外で、毎回スポーン地点から再開する。

プロパティ型説明
idUUIDワールド固有ID。ChunkRecord/ChestRecord の worldID と紐付く
name / seedString / Intワールド名・地形生成シード
createdAt / lastPlayedAtDate作成日時・最終プレイ日時（一覧の並び順に使用）
totalPlayTimeTimeInterval累計プレイ時間
exploredDistanceDouble累積移動距離。目標「探索を広げる」の判定に使用
spawnX / spawnY / spawnZFloatスポーン座標。既定 (8, 62, 8)。ベッド使用で更新される
inventoryData / unlockedToolsDataData / Data所持品・解放済みツールのシリアライズ
selectedToolRawString選択中ツール（既定 "hand"）
hungerInt満腹度（最大20）
hpIntHP（最大20）。死亡画面のまま退出すると 0 が保存され得るため、復元側で 0 以下なら 20 に補正する

ChunkRecord（SwiftData @Model）

プロパティ説明
worldID所属ワールドID
cx, czチャンクグリッド座標（ブロック座標 = cx/cz × 16）
blocksDataBlock.rawValue を 16×64×16=16384 バイトへ直列化した Data。インデックス = x + 16*(y + 64*z)

チャンク（ランタイム側の Chunk クラス）はブロック編集のたびに dirtyChunks へ記録され、(a) チャンクアンロード時、(b) アプリのバックグラウンド移行通知時、(c) 30秒ごとの定期保存、の3経路で WorldStore.saveChunk（専用シリアルキュー上の ModelContext）へ書き込まれる。

ChestRecord（SwiftData @Model）

プロパティ説明
worldID, x, y, z所属ワールドID・チェストのワールド座標
slotsDataチェスト内アイテムのシリアライズ（形式は下記インベントリと同じ）

Player（ランタイムクラス・SwiftData対象外）

final class Player {
    var position: SIMD3<Float> = SIMD3(8, 62, 8)  // 足元基準
    var yaw: Float = 0
    var pitch: Float = -0.3
    var velocityY: Float = 0
    var grounded: Bool = false
    var fly: Bool = false             // クリエイティブのフライ
    var isInWater: Bool = false
    var isThirdPerson: Bool = true    // 既定は三人称視点
    var hp: Int = 20

    // 定数
    let moveSpeed: Float = 6          // m/s
    let gravity: Float = -24          // m/s²
    let jumpSpeed: Float = 9          // m/s
    let eyeHeight: Float = 1.62
    let halfW: Float = 0.3            // AABB半幅
    let halfH: Float = 0.9            // AABB半高さ（全高1.8）
}

位置・時刻・天候・モブは WorldRecord に保存されない（毎回スポーン地点・正午・晴れから再開）。HP・満腹度・所持品・解放ツール・進行フラグのみが永続化対象。

GameInput（UI状態、@MainActor）

SwiftUI 側から参照される @Published の集合。レンダースレッドとの橋渡しは「値が変化したときだけ DispatchQueue.main.async でミラー反映する」規約で統一されている。主なカテゴリ:

カテゴリ代表的なプロパティ
プレイヤー状態playerHP, hunger, selectedBlock, selectedTool, unlockedTools, inventory
UI表示制御showCrafting, showSettings, showChest, showInventory, showFurnace, isDead, deathCause, toastMessage
進行フラグhasPlacedFurnace, hasCraftedTorch, hasPlacedBed, hasSmeltedGlass, hasCraftedGoldenApple, hasCompletedAllObjectives, hasSeenLavaWarning
パフォーマンス計測showPerf, perfFPS, perfMemoryMB, perfChunkCount, perfGenSeconds（設定シートのトグルで表示切替、既定OFF）
設定lookSensitivity, masterVolume, joystickDeadZone, miningButtonBoost, graphicsQuality
視点・演出isThirdPerson（既定true）, damageFlash, breakProgress, isWorldReady, weather

インベントリ / チェストのシリアライズ形式

個数を UInt8（最大255）でクランプしていた旧形式から、上限を撤廃した新形式に移行済み。先頭バイトで新旧を判別する。

形式構造個数上限
旧形式[rawValue: UInt8, count: UInt8] の2バイトペアの並び255
新形式先頭に 0xFF、以降 [rawValue: UInt8, countLo: UInt8, countHi: UInt8] の3バイトペア（UInt16 リトルエンディアン）65535

0xFF は Block の最大 rawValue（33）と衝突しないため、先頭バイトだけで安全に新旧を判別できる。デコードは常に両形式に対応し、エンコードは常に新形式で書く。

マイグレーション方針

SwiftData の @Model は全プロパティにデフォルト値必須。新規プロパティ追加時は必ずデフォルト値を設定する（実績: hp・hasCraftedTorch 等はすべてデフォルト値付きで軽量マイグレーション済み）。新しい Block を追加する場合は既存 rawValue を変更せず末尾に追加する。.modelContainer(for:) はコンテナ生成失敗時にフォールバックがなく起動クラッシュになるため、スキーマ変更は「デフォルト値付きプロパティの追加のみ」に限定する運用とする。
