データモデル

SwiftData @Model の構造とゲーム内データ設計

概要

VoxelGame のデータは 永続化層（SwiftData） と ゲームランタイム層（インメモリクラス） の 2 層に分けて管理する。SwiftData はワールドのメタ情報のみを扱い、チャンクの生ブロックデータはドキュメントディレクトリにバイナリファイルとして保存する。

WorldRecord（SwiftData @Model）

@Model
final class WorldRecord {
    var id: UUID = UUID()
    var name: String = ""
    var seed: Int = 0
    var lastPlayedAt: Date = Date()
}
プロパティ型説明
idUUIDワールド固有ID。チャンクファイルのディレクトリ名に使用
nameStringユーザーが入力したワールド名
seedInt地形生成のシード値。省略時は自動生成
lastPlayedAtDate最終プレイ日時。ワールド選択画面の並び順に使用

Chunk（ランタイムクラス）

class Chunk {
    let cx: Int          // チャンク X 座標（ブロック座標 = cx * 16）
    let cz: Int          // チャンク Z 座標
    var blocks: [Block]  // 16 × 64 × 16 = 16384 要素。インデックス = x + 16*(y + 64*z)
    var isDirty: Bool    // 変更済みフラグ（ファイル保存が必要）
    var node: SCNNode?   // SceneKit ノード参照
}
プロパティ説明
cx, czチャンクグリッド座標。ブロック座標への変換は cx * 16 / cz * 16
blocks16×64×16 のブロック配列（16384 要素）。Block は UInt8 の enum
isDirtyプレイヤーがブロックを変更した際に true にセット。ファイル保存後に false に戻す
node生成済み SCNNode。nil の場合はメッシュ未生成

Player（ランタイムクラス）

class Player {
    var position: SCNVector3 = SCNVector3(0, 20, 0)
    var velocity: SCNVector3 = .init(0, 0, 0)
    var hp: Int = 20          // 最大 20
    var hunger: Float = 20.0  // 最大 20
    var yaw: Float = 0        // 左右視点（ラジアン）
    var pitch: Float = 0      // 上下視点（ラジアン）
    var isFlying: Bool = false
    var spawnPoint: SCNVector3 = SCNVector3(0, 20, 0)
    var inventory: [Block: Int] = [:]
    var unlockedTools: Set<ToolType> = [.hand]
    var selectedTool: ToolType = .hand
}

マイグレーション方針

SwiftData の @Model は全プロパティにデフォルト値必須。新規プロパティ追加時は必ずデフォルト値を設定する。チャンクファイルのバイナリフォーマット変更時はバージョンヘッダー（先頭 1 バイト）でバージョン判定する。
