クラフト

ツールレシピと採掘速度テーブルの定義

機能ID: VG-FEAT-CRAFTING

概要

インベントリに素材が揃った状態でクラフト画面を開くと、対象レシピが解放可能になる。クラフト実行で素材を消費し、対応ツールを解放する。解放されたツールはツールバーに追加され、ブロック採掘速度が上昇する。

関連画面

  • ゲームプレイ画面（クラフト画面シート）

ビジネスルール

  • ツールは一度解放したら永続（ワールドをまたいでも保持する）しない。WorldRecord 単位で管理する
  • 同じツールを複数回クラフトしても素材だけ消費され、効果は変わらない（重複解放の防止は実装では行わずトースト「すでに解放済みです」を表示する）
  • クリエイティブモード時でもクラフト UI は利用可能だが、素材消費がないため意味をなさない（将来は非表示にする）
  • クラフト画面は素材が不足している場合もレシピを表示する（グレーアウト）

外部連携

連携先用途備考
Player.inventory素材の確認・消費クラフト実行時に参照・更新する
Player.unlockedToolsツール解放状態の管理Set<ToolType> に追加する
ブロック採掘ロジック（GameLoop）採掘速度テーブルの参照ToolType × Block → Double（秒）のマッピング

エラー処理

発生条件対応
クラフト実行時に素材が不足している（UI バグ等）クラフトを中断。「素材が不足しています」トーストを表示。Logger.error で記録

実装メモ

  • 採掘速度テーブルは static let として ToolType の extension か専用 struct に定義し、GameLoop から参照する
  • 採掘進行度はゲームループで deltaTime を積算し、閾値を超えたらブロック破壊イベントを発火する
  • 破壊されたブロックのアイテムドロップは Player.inventory[block, default: 0] += 1 で加算する
  • 採掘不可の判定は miningTime(tool:block:) → nil で表現し、GameLoop 側で nil の場合は進行しない
